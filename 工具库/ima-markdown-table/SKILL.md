# IMA Markdown 表格渲染规则

腾讯 IMA（混元大模型知识库）的 Markdown 解析器对表格语法有比标准 Markdown 更严格的要求。违反以下任一条都会导致表格不渲染。

## 7 条规则速查

| # | 规则 | ❌ 错误 | ✅ 正确 |
|---|---|---|---|
| 1 | **引用块与表格之间必须有空行** | `> ...\n\| 表头 \|` | `> ...\n\n\| 表头 \|` |
| 2 | **分隔行 `\|` 两侧禁止空格** | `\|------ \| ------ \|` | `\|------\|------\|` |
| 3 | **禁止 `:--:` 对齐语法** | `\|:--:\|:--:\|` | `\|------\|------\|` |
| 4 | **emoji 颜色分档 → 表后必须有说明** | 只有🟢🟡🔴无注解 | 表后紧跟分档说明 |
| 5 | **表头/数据单元格禁止 `\n` 换行** | `第1次\n05/24`（跨行） | `第1次 05/24`（同行） |
| 6 | **连续表格间标题须有前后空行** | 标题直接贴表格 | `'\n' + label + '\n\n' + '\|...\|'` |
| 7 | **表格行间禁止空行** | 分区标题行前有空行 | 整张表连续无空行 |

---

## 规则详解

### 规则 1：引用块与表格之间必须有空行

**最易踩坑。** `>` 引用行和 `|` 表格行之间只有 1 个 `\n` → IMA 把表格吞进引用块→不渲染。

```python
# ❌ 错误：末尾无 \n
lines.append("> 这是引用块说明")

# ✅ 正确：末尾有 \n（配合 ''.join() 或 '\n'.join() 产生空行）
lines.append("> 这是引用块说明\n")
```

**验证方法**：`cat -An` 查看，引用块和表格头之间应该有 1 行 `$`（空行）。

---

### 规则 2：分隔行 `|` 两侧禁止空格

```python
# ❌ 错误：" | ".join() 产生空格
sep = "|------|----------|" + " | ".join(["------" for _ in cols]) + " |"
# 输出：|------|----------|------ | ------ | ------ |

# ✅ 正确："|".join() 无空格
sep = "|------|----------|" + "|".join(["------" for _ in cols]) + "|"
# 输出：|------|----------|------|------|------|
```

**注意**：此规则仅针对**分隔行**，表头行和数据行可以有空格（`| 时间 | 行为 |` 是正常的）。

---

### 规则 3：禁止 `:--:` 对齐语法

IMA 不支持 Markdown 表格的列对齐语法。全部用纯短横线 `------`。

---

### 规则 4：emoji 颜色分档 → 表后必须有说明

任何表格单元格中使用 🟢🟡🟠🔴🟣🔵⚫ 等 emoji 颜色标记时，表格后必须紧跟颜色分档说明。

```markdown
| 经络 | 平衡% |
|------|-------|
| 肺经 | 🟢95.1% |

> 📌 颜色分档：🟢 ≥95% | 🟡 ≥85% | 🟠 ≥75% | 🔴 <75%
```

**格式不限**：引用块、普通段落、小表格均可，视上下文灵活选择。

---

### 规则 5：表头/数据单元格禁止 `\n` 换行

Markdown 表格要求每行是单个逻辑行。单元格内嵌 `\n` 会导致表头跨多行 → 分隔行不在表头正下方 → IMA 无法识别表格边界。

```python
# ❌ 错误：\n 在单元格内
col_headers = [f'第{i+1}次\n{xlabels[i]}' for i in range(n)]

# ✅ 正确：用空格或 / 分隔
col_headers = [f'第{i+1}次 {xlabels[i]}' for i in range(n)]
```

---

### 规则 6：连续表格间标题须有前后空行

两个表格之间的标题（如 `**② 三次均值**`）没有空行包裹 → 被合并进前一个表格。

```python
# ❌ 错误：label 直接连表格
return label + '\n' + '| 表头 |...'

# ✅ 正确：label 前后各加 \n
return '\n' + label + '\n\n' + '| 表头 |...'
```

## Python 生成 MD 时的标准模板

```python
# 引用块 + 表格（规则1）
lines.append("> 说明文字\n")  # ← 注意末尾 \n
lines.append("")              # ← 显式空行（可选，如果上一行已有 \n 则自动产生）

# 表头（规则5: 无 \n）
header = "| 列1 | 列2 | 列3 |"
lines.append(header)

# 分隔行（规则2+3: 无空格、无对齐）
sep = "|------|------|------|"
lines.append(sep)

# 数据行
for row in data:
    lines.append(f"| {row.a} | {row.b} | {row.c} |")

# emoji 颜色分档说明（规则4）
lines.append("")
lines.append("> 📌 颜色分档：🟢 ≥95% | 🟡 ≥85% | 🔴 <85%")

# 连续表格标题（规则6）
lines.append("")
lines.append("**第二个表格标题**")
lines.append("")
lines.append("| 新表头 |...")
```

---

### 规则 7：表格行间禁止空行

一张表内部任意两行间不能有空行。空行=IMA认为表格结束。分区标题行（如 `| **胸椎** |`）前若有空行→切成两段，后段无分隔行→不渲染。

```markdown
# ❌ 错误：分区标题前有空行
| C7 | 脊柱 - 胸廓区域 | 5.7 | ...
                                ← 空行切断表格！
| **胸椎** | | | ...
| T1 | 心脏 | 6.1 | ...

# ✅ 正确：连续无空行 → 整张表一次渲染
| C7 | 脊柱 - 胸廓区域 | 5.7 | ...
| **胸椎** | | | ...
| T1 | 心脏 | 6.1 | ...
```
