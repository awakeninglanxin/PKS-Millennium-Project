## 1. 选题与开题

### Prompt 1.1: 研究缺口发现
```
我现在正在研究 [{研究方向}] 方向，准备选定一个论文题目。
请帮我：
1. 列出该方向 3-5 个前沿研究缺口（research gap），每个附 1-2 句解释
2. 评估每个缺口的创新性（1-10）和可行性（1-10）
3. 对最有潜力的题目，用一句话给出核心研究问题（research question）
4. 给出选题的初步 IMRaD 结构建议

我的背景信息：
- 研究领域：{研究领域}
- 已掌握方法：{方法列表}
- 目标期刊类型：{CCF-A / SCI一区 / 顶会}
```

### Prompt 1.2: 快速选题验证
```
我想验证 [{论文题目}] 这个选题是否值得做。请：
1. 搜索最近 3 年这个方向的高引论文（≥5 篇），总结它们的共同结论
2. 判断我的题目与现有工作的差异化程度（高/中/低）
3. 如果差异化程度低，给出 2 个调整方向
```

## 2. 文献综述

### Prompt 2.1: 结构化文献综述
```
请帮我系统性地综述 [{研究主题}] 的相关文献。

要求：
1. 按时间线/方法线/结论线三维度组织
2. 每篇文献提取：方法、数据集、核心指标、局限性
3. 生成对比表格（至少 10 篇核心文献）
4. 用一段话总结现有方法的共同局限

文献来源偏好：Semantic Scholar / arXiv / PubMed
输出格式：Markdown 表格 + 结构化段落

请优先引用真实存在的文献。如果不确定某篇文献是否存在，请标注 [需人工验证]。
```

### Prompt 2.2: 单篇文献速读
```
请帮我分析这篇文献：{粘贴标题+摘要}

提取：
1. 核心贡献（一句话）
2. 方法简述（≤100字）
3. 使用的数据集和评估指标
4. 论文的主要局限性
5. 这篇文献对我的研究 [{我的研究方向}] 有什么启示？
```

### Prompt 2.3: 对抗式文献综述
```
我正在写文献综述。请模拟一位持相反观点的审稿人，从至少 3 个角度指出我的综述可能存在的盲点：

{粘贴你的文献综述草稿}

不要客气，直接用审稿人的口吻批评。
```

## 3. 方法论设计

### Prompt 3.1: 实验设计
```
我正在设计论文的实验方法。请帮我完善方法论部分。

研究问题：{研究问题}
数据集：{数据集描述}
baseline 方法：{baseline方法列表}

请：
1. 给出实验设计（变量、对照组、评估指标）
2. 检查是否有统计陷阱（p-hacking、样本量不足、多重比较问题）
3. 用伪代码描述核心算法流程
4. 推荐 3 个合适的评估指标并解释为什么选它们
```

### Prompt 3.2: 消融实验设计
```
我的方法包含以下组件：
{组件1}: {描述}
{组件2}: {描述}
{组件3}: {描述}

请设计一个完整的消融实验（ablation study），逐步去掉每个组件，观察对性能的影响。用表格展示所有消融配置。
```

### Prompt 3.3: 统计陷阱检查
```
请以统计学家+审稿人的视角检查以下实验设计是否有统计问题：
{粘贴实验设计描述}

重点检查：
- p-hacking 风险
- 多重比较是否需要校正（Bonferroni / FDR）
- 样本量是否足够（power analysis）
- 是否有混淆变量未控制
- 结果是否可能来自过拟合而非真正提升
```

## 4. 实验与数据分析

### Prompt 4.1: 可视化方案
```
请帮我分析以下实验数据并生成可视化方案。

数据描述：{数据描述}
对比方法：{方法列表}
主要发现：{发现}

请：
1. 推荐 4 种最适合的数据可视化类型（含使用理由）
2. 给出一张最关键的对比图表的 Python (matplotlib/seaborn) 代码
3. 进行统计显著性检验，告诉我 p 值
4. 列出一个结果表格的 LaTeX 代码
```

### Prompt 4.2: 数据清洗
```
我有一个数据集需要清洗。数据特征：
- 大小：{样本数} 条 × {特征数} 维
- 已知问题：{缺失值/异常值/类别不平衡 等}

请写出 Python 数据处理管线的代码，包括：
1. 缺失值处理策略
2. 异常值检测与处理
3. 归一化/标准化
4. 数据增强（如适用）
```

## 5. 论文撰写（按章节细分）

### Prompt 5.1: 摘要
```
请帮我撰写 {Nature/Science/IEEE} 风格的论文摘要。

论文标题：{标题}
核心贡献：{一句话}
关键结果：{2-3 个数字/发现}

要求：
- ≤ 200 words（Nature 格式）
- 结构：Background → Problem → Method → Results → Conclusion
- 每个句子必须有信息量（不要说 "We conducted experiments" 这种空话）
- 包含具体数字（提升 XX%，p < 0.01）
```

### Prompt 5.2: Introduction
```
请帮我撰写论文的 Introduction。

核心信息：
- 研究领域背景：{背景，2-3 句}
- 现有方法局限：{局限}
- 我们的解决方案：{方案}
- 主要贡献：{3-4 条}

要求：
- 从大到小（broad → narrow），最后一段点明贡献
- 控制 1.5 页以内的长度
- 每条贡献声明必须有对应的实验结果支撑
```

### Prompt 5.3: Method
```
请帮我撰写 Methods 章节。

方法名：{方法名}
核心思路：{思路}

要求：
- 分小节：Overview → {子模块1} → {子模块2} → Training Details
- 每个公式后跟一段自然语言解释
- 提供伪代码
- 列出所有超参数和选取理由
```

### Prompt 5.4: Results
```
请帮我撰写 Results 章节。

实验设置：{描述}
主要实验结果：

对比实验：
{结果表格或描述}

请：
1. 每个段落只讲一个核心发现（1 claim per paragraph）
2. 段落结构：Main Finding → Supporting Evidence → Link to Next
3. 数据用 "XX ± YY (p = ZZ)" 格式
4. 不要在 Results 中讨论"为什么"（留给 Discussion）
```

### Prompt 5.5: Discussion
```
请帮我撰写 Discussion 章节。

主要发现：{3-5 条发现}

要求：
1. 每条发现与现有工作对比（与哪些工作一致/矛盾）
2. 解释"为什么"有这个结果（不只是重复 Results）
3. 坦诚列出 3 个局限性
4. 提出 2 个未来的研究方向
```

### Prompt 5.6: 段落级润色（Claim-Evidence-Link）
```
请帮我重写以下段落的论点结构。

{粘贴原文}

每条论点按这个格式重组：
1. **Claim**：你要说的核心论点是什么？
2. **Evidence**：用什么数据支撑？
3. **Link**：如何连接到下一段/最终结论？

只输出重组后的版本，不要添加新内容。
```

## 6. 图表与可视化

### Prompt 6.1: 图表设计方案
```
我需要为论文生成一张关键图表。

图表类型：{柱状图/折线图/散点图/热力图/示意图}
数据描述：{数据}
目标期刊风格：{Nature / Science / IEEE}

请：
1. 描述整张图的设计方案（panel布局 + 颜色方案）
2. 给出 figure caption 的初稿（遵循目标期刊格式）
3. 用 Python + matplotlib 写出完整绘制代码
```

### Prompt 6.2: 图表自查清单
```
这是我要投稿的图表，请以审稿人的视角检查：

图表标题：{标题}
图表说明：{说明}

检查项：
1. 图注是否独立自明（不依赖正文就能懂）
2. 是否有 Error Bar / 置信区间
3. 颜色是否对色盲友好
4. 坐标轴标签是否清晰
5. 字号是否足够大（投稿印刷时不会糊）
```

## 7. 润色与自查

### Prompt 7.1: 学术语言润色
```
请帮我润色以下学术文本。

[粘贴你的文字]

要求：
1. 保持原意，只改表达方式
2. 去掉口语化表达
3. 统一术语
4. 每个修改标注改动原因
5. 标记任何逻辑跳跃的地方

不要直接重写，用 "原文: ... → 建议: ... 原因: ..." 格式。
```

### Prompt 7.2: 逻辑流检查（Reverse Outline）
```
请对以下论文章节做 reverse outline 分析：

{粘贴章节全文}

为每个段落提取一句话的中心论点。然后判断：
1. 相邻段落间是否有清晰的逻辑连接？
2. 有没有重复的论点？
3. 整体论证顺序是否最优？
```

### Prompt 7.3: 中文论文润色
```
你是中文论文润色专家。请润色以下中文论文章节。

要求：
- 符合 GB/T 7713 学术论文编写规则
- 术语与科技名词规范一致
- 控制"的"字出现频率（连续两个"的"以上需重写）
- 长句拆为短句（每句 ≤ 40 字）
- 数据表述格式统一

[粘贴文本]
```

## 8. 投稿与返修

### Prompt 8.1: Cover Letter
```
请帮我撰写 {Nature/Science/IEEE} 风格的 Cover Letter。

论文标题：{标题}
核心贡献（1句）：{贡献}
目标期刊：{期刊名}

Cover Letter 结构：
1. Dear Editor,
2. 介绍论文核心贡献（2-3 句）
3. 解释为什么适合该期刊（1-2 句）
4. 声明（原创性、无利益冲突、所有作者同意）
```

### Prompt 8.2: 逐条审稿回复策略
```
请帮我分析以下审稿意见并制定回复策略。

审稿人意见：
{粘贴审稿意见}

对每条意见：
1. 给出 1-2 句礼貌的开场回应
2. 说明你做了什么修改
3. 指出修改在论文中的位置

最后总结：哪些意见容易回应，哪些比较棘手？
```

### Prompt 8.3: 返修对比表
```
我的论文收到一轮修改意见。帮我生成一个"修改前后对比表"。

审稿意见：
{粘贴意见}

我做了什么修改：
{粘贴修改内容}

格式：
| 审稿意见 | 修改前 | 修改后 | 修改位置 |
```

## ⚠️ 使用规范

- **每个 Prompt 中的 `{...}` 都需要替换成你自己的具体信息**
- AI 输出不是最终稿——你需要审阅、修改、加入你的学术判断
- 引文必须逐条验证（用 DOI、Semantic Scholar、Google Scholar）
- 过度使用 AI 润色会让整篇文章 AI 味过重——保留你的写作风格
