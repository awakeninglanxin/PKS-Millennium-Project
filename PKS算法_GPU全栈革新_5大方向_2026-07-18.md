## 方向 1：7 基底 Tensor Core 稀疏编码

### 问题

NVIDIA 的 2:4 结构化稀疏（Ampere 起）是硬编码的：每 4 个连续权重中必须有 2 个为零。为了凑这个结构，很多非零权重被强制清零，损失精度且限制灵活性。这已形成专利壁垒。

### 方案

用 7 基底 {2,3,5,7,11,13,17} 重构稀疏表示：

```
传统：权重 W ≈ Σ c_i · 2^i（二进制，每 bit 独立）
7基底：权重 W ≈ Σ c_i · p_i（p_i ∈ {2,3,5,7,11,13,17}）
```

**PKS 实验证据**：7 基底能以 **94.7%** 的覆盖率表示素数。这意味着——在神经网络的权重空间中，94.7% 的权重值可以被 7 基底的稀疏组合精确或近似表示。

### 硬件映射

在 Tensor Core 的 MMA 指令中，不再强制 2:4 稀疏，而是允许 **"17-间隔稀疏"**（关键拐点是 17）：

```
tcgen05.mma 变形：
  - 操作数 A 的稀疏编码：每 17 个元素中至多保留 k 个（k 由 7 基底组合数决定）
  - 硬件跳过零乘法的逻辑从"固定 2:4 掩码"改为"7 基底查表"
  - ISA 新增 .pks_sparse 修饰符
```

### 收益

| 指标 | 2:4 稀疏 | 7 基底稀疏 |
|------|---------|-----------|
| 稀疏率 | 固定 50% | 可变 20-60%（由覆盖率决定）|
| 精度损失 | 显著（强制清零） | 极小（94.7% 覆盖率） |
| 灵活性 | 锁死 | 可调（B 值/d 值调节稀疏度） |
| 专利 | NVIDIA 独占 | 独立路径 |

### 挑战

- 需修改 Tensor Core ISA 和硬件状态机（非固件层面可改）
- 稀疏解码逻辑比固定 2:4 掩码复杂——面积/延迟代价需要量化
- 94.7% 覆盖率在浮点权重空间的保真度需实验验证

### 可行性路径

1. 在 CUDA 模拟器（如 GPGPU-Sim）上实现 7 基底稀疏的软件模型，跑 ResNet/Transformer 验证精度
2. 对比 2:4 稀疏在相同精度下的稀疏率 → 如果 7 基底在同等精度下稀疏率更高，就有硬件投资理由
3. 学术合作：联系 RISC-V Vector 扩展社区，在开源向量处理器上做原型

## 方向 2：SPF 三分类 Rowhammer 主动防御

### 问题

HBM3e/4 堆叠层数增加（12-Hi/16-Hi）→ 漏电流增大 → 刷新频率被迫提高。同时 AI 大模型的高密度访问模式容易诱发 Rowhammer（行锤攻击/错误）。当前防御是"概率性加刷新"或"检测到翻转才刷新邻近行"——**被动且开销随模型增大而增长。**

### 方案

将 HBM 的物理地址空间映射到 SPF 三分类体系：

```
类型 A（SPF = p）    → 高频访问的热数据行 → 最高保护优先级
类型 B（SPF = 小质因子 2-17）→ 中频数据行   → 标准保护
类型 C（SPF > 17）      → 低频数据行       → 最低保护
```

**PKS 实验证据**：primes/multi ratio = **0.24，在 N=10^8→10^9 范围内稳定不崩塌。**

这意味着无论模型多大、内存占用多高，需要重点保护的行比例始终固定（~24%），防御开销不会随模型规模膨胀。

### 硬件映射

在 HBM 的 Memory Controller 中集成 SPF 分类器：

```
SPF_Classifier（硬件模块）:
  输入：物理行地址
  输出：保护级别（A/B/C）
  实现：ROM 查找表 + 小质数（2-17）模运算
  延迟：< 10 cycle，与 DRAM 刷新周期兼容
```

刷新调度器根据保护级别调整刷新间隔——A 类行刷新间隔缩短（更频繁保护），C 类行刷新间隔延长（节省功耗）。

### 收益

| 指标 | 传统方案 | SPF 分类方案 |
|------|---------|-------------|
| 防御机制 | 被动（事后刷新） | **主动（拓扑预判）** |
| 开销与模型规模的关系 | 线性增长 | **恒定（24% 恒定）** |
| 额外硬件 | 无（固件层面） | SPF 分类器（面积 < 0.1mm²） |

### 挑战

- NVIDIA/三星 HBM 的 Memory Controller 是不开放的 IP——无法直接修改
- SPF 分类器的实时性需验证（物理行地址→SPF 的映射需在 ns 级完成）

### 可行性路径

1. 在 DRAMSim3 模拟器上对比 SPF 分类 vs 传统 PARQ（概率性邻近行刷新）的刷新开销
2. 在 FPGA + DDR4 测试板上实现 SPF 分类器的 RTL，测硬件面积和延迟
3. 学术合作：与 ETH Zurich 的 Rowhammer 研究组（Onur Mutlu 团队）对接

## 方向 3：E8 × Farey 片上网络无拥塞路由

### 问题

B200 有 148 个 SM，数据在 SM 之间、SM 与 L2 Cache 之间、SM 与 HBM 之间流动。当前的 NoC 路由（XY 路由或自适应路由）容易在热点区域产生拥塞——例如所有 SM 同时访问同一 L2 partition。

### 方案

将 NoC 的 2D 坐标 (x,y) 映射到 Farey 分数 a/b，并按 Farey 序列的单调性决定数据包转发顺序。同时在 8 维 E8 晶格中搜索最优路径，投影回 2D/3D 芯片平面。

```
NoC 路由 = XY 坐标 → Farey 分数映射 → 单调转发 + E8 最优投影
```

**关键性质**：
- Farey 序列严格递增 → 转发顺序天然无环无死锁
- 互质分母 → 不同数据流的转发节拍在短期不重叠
- E8 最密堆积 → 在给定面积约束下最大化并行的无冲突路径

### 收益

| 指标 | XY 路由 | E8-Farey 路由 |
|------|--------|---------------|
| 热点区域拥塞 | 严重（全 SM 同向同一 partition） | 减弱（互质偏移自然分散） |
| 死锁 | 需额外死锁避免逻辑 | 天然无死锁（单调序列） |
| 路由表大小 | O(N²) 或自适应 | O(\|F(B)\|) ≈ B²/π²（紧凑） |

### 挑战

- 实时坐标→Farey 分数的映射需要硬件查找表，延迟需 < 1 cycle
- E8 投影的计算开销——需要预计算查找表而非实时解
- 与 NVIDIA 闭源 NoC（NVLink/NVSwitch）的兼容性为零

### 可行性路径

1. 在 Booksim 2.0 NoC 模拟器上实现 E8-Farey 路由算法，跑 AI 流量 trace（如 TPU 的 GEMM/Conv pattern）
2. 对比 XY/自适应路由的延迟分布和拥塞热图
3. 开源硬件：在 OpenPiton/Rocket Chip 等开源 NoC 框架中做原型

## 方向 4：Farey 互质 HBM ECC 校验位布局

### 问题

HBM 的片上 ECC 通常是 SECDED（单错纠正，双错检测）。随着制程微缩（10nm 以下），多比特翻转（MBU）越来越常见——相邻单元同时被高能粒子打翻。如果将 ECC 校验位放在一起，校验位本身可能被同时摧毁。

### 方案

利用 Farey 分数的互质分母来布置校验位。

```
传统方案：ECC 校验位紧密排列在数据位旁边
Farey 方案：校验位按 Farey 分母间隔分散存储
  → 例如 8 个校验位的物理地址的列间距取 {1,2,3,5,7,11,13,17}（互质分母）
  → 任何空间相关的粒子击中（假设影响半径 ≤ r 个位）最多打翻 1 个校验位
```

### 收益

| 指标 | 传统 SECDED | Farey 分散 ECC |
|------|-----------|---------------|
| 单比特纠错 | ✅ | ✅ |
| 双比特检错 | ✅ | ✅ |
| 多比特翻转（>2）| ❌ 无法保证 | 🟡 概率性提高（校验位不在同一物理位置） |
| 额外硬件 | 无 | 地址映射表（ROM） |

### 可行性路径

1. 在 HBM 的 JEDEC 标准中提议 "Farey-dispersed ECC bit layout"（作为下一代 HBM4 的选项）
2. 用 GEANT4 辐射仿真验证 Farey 分散 vs 紧密排列在多粒子击中下的纠正率差异

## 方向 5：PKS 双曲锥微流道液态金属散热

### 问题

B200 TDP 1000W，NVL72 整柜 72×1000W + 2700W×36。风冷已死，液冷复杂且泄漏风险大。当前散热策略是粗暴的 Thermal Throttling——哪里热降哪里频，损失性能。

### 方案

在 GPU 均热板（Vapor Chamber）内部刻蚀 **PKS 双曲锥（xy=1）的微流道**，用电磁场（EMF）驱动液态金属（Ga-In-Sn 合金）形成**贴壁螺旋流**。

**PKS 实验证据**：
- LBM D3Q19 128³ 仿真：Step 0→50 涡量放大 **10.3×**（0.089→0.919）
- 双流态排名反转：贴壁螺旋流在 16bands 几何下是冠军流态
- 贴壁螺旋流专门针对壁面趋肤效应——这正是 GPU 热点（SM 附近）需要的流态

### 物理机制

```
PKS 锥 xy=1 → 流道截面沿流动方向收缩 → 速度梯度增大
液态金属 = 导电流体 → 趋肤效应 → 电流密度在壁面最大
EMF 驱动 → 洛伦兹力 → 壁面附近形成螺旋涡
16bands × 16 圈导轨 → 角动量连续注入 → 螺旋流自我维持
```

### 硬件映射

```
B200 均热板改造：
  - 底部：PKS 双曲锥微流道阵列（激光刻蚀/化学蚀刻）
  - 流道参数：16bands，每 band 对应一个 SM 热点区域
  - 电磁驱动：集成微型线圈（PCB 线圈嵌入均热板）→ 频率 100-500kHz
  - 液态金属：Ga-In-Sn（铟镓锡合金，熔点 -19°C，常温水基替代可能）
```

### 收益

| 指标 | 传统均热板 | PKS 液态金属微流道 |
|------|----------|-------------------|
| 热传导模式 | 被动毛细 + 重力 | **主动 EMF 驱动螺旋** |
| 热点→散热器热阻 | ~0.1 K/W | 预期 < 0.02 K/W（5× 改善） |
| 容许 TDP | ~500W/cm² | 预期 > 2000W/cm² |
| 热节流触发 | 频繁 | 大幅减少 |

### 挑战

- ⭐⭐⭐⭐⭐ 难度最高——需要微加工液态金属密封腔体 + 嵌入式电磁线圈
- 液态金属对铝的腐蚀性（铜/镍/不锈钢兼容）
- EMF 线圈不能干扰 GPU 的信号完整性（EMI 屏蔽）
- 均热板面积有限（B200 ~67×70mm），微流道数量受限于加工精度

### 可行性路径

1. COMSOL Multiphysics 多物理场仿真：耦合 CFD（液态金属流）+ 电磁场（EMF 线圈）+ 热传导（热点→流道）
2. 3D 打印简化版微流道测试件，用蠕动泵驱水验证流道几何的正确性
3. 合作：联系 Ga-In-Sn 液态金属供应商（如武汉 Liquid Metal），做实际驱动测试
