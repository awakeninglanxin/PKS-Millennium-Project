## 二、逐个拆解

### 2.1 `fractal dragon.py` — L-系统分形

```python
cmd = 'f'
for _ in range(8):
    cmd = cmd.replace('f', 'f+f-f')
# Angle = 60°, step = 5
```

**数学根因**：每次替换把每条线段替换成三段（f+f-f），8 次迭代后：

$$N_{\text{segments}} = 3^8 = 6561$$

**几何性质**：
- 分形维度：$D = \log 3 / \log 2 \approx 1.585$
- 自相似：任何局部放大后等于全局结构
- 旋转模式：60° 转角在六边形格子上形成对称图案

**为什么是这种形状？** 因为 `f+f-f` 的意思是"走→左转→走→右转→走"，相当于每条线段被替换成一个带折角的路径。60° 转角 + 自相似替换 → **Sierpinski 曲线**（实际不是正宗的 dragon curve，正宗龙曲线规则是 `f → f+g+, g → -f-g`）。

**关键特征**：
- 完全确定性（无随机）
- 全局自相似（分形必备属性）
- 由**替换规则**驱动，而非外部序列

### 2.2 `polygon.py` — 正多边形逐层叠加

```python
for i in range(3, 200):
    circle(i*10, steps=i)  # 半径 i*10，用 i 边形逼近圆
    left(30)               # 每层后转 30°
```

**数学根因**：两层循环：

1. **外层 `i`**：控制多边形的"边数"（3 → 199）
2. **内层 `circle(i*10, steps=i)`**：画半径为 $r=i\times 10$ 的"圆"，但只用 $i$ 条边逼近
3. **旋转 `left(30)`**：每画完一层转 30°

**为什么是这种形状？**

| 参数 | i 小时 | i 大时 |
|:---|:---|:---|
| 多边形边数 | 3（三角形） | 199（近似圆） |
| 半径 r = 10i | 30 | 1990 |
| 相对大小 | 小 | 大 |
| 旋转偏移 | 30° ≠ 360°/整数 → 每层不重叠 |

**关键**：30° 不是 360° 的整除因子 → 每层旋转后和上一层错开 → 12 层后回原点（因为 30°×12=360°）→ 但此时 $i$ 已经变了 → 形成**错位叠加的同心多边形套件**，视觉上就是涡旋。

### 2.3 `prove the law of universe.py` — 数根驱动曼陀罗 🔥🔥🔥

这是最重要、最复杂的文件。核心是两个函数：

#### 算法 A: `make_circle_difference()` — 算术级数

```python
cnt = min_int        # 起始数
while True:
    dpd = outputresult(cnt)    # 数字根 (1-9)
    forward(dpd * coeff * 4)   # 步长 ∝ 数根
    right(dpd * coeff)         # 转角 ∝ 数根
    cnt += step                # cnt 等差递增
    if abs(pos()) < 0.00001:   # 回到原点附近 → 一圈结束
        right(dpd)             # 额外转角 → 下一圈偏移
        break
# 外循环继续，重复多次
```

**数学根因 — 三步分解**：

**第一步：数根的周期结构**

`outputresult()` = 数字根函数。对等差数列 `cnt = min_int + k*step`：

$$\text{dpd}(k) = \text{digital\_root}(a_0 + k \times \text{step})$$

当 `step` 不是 9 的倍数时，$\text{dpd}(k)$ 以周期 9 循环（或更短）。

例如 `min_int=3, step=3`：
```
cnt:  3  6  9  12 15 18 21 24 27 30 33 36 ...
dpd:  3  6  9  3  6  9  3  6  9  3  3  6  ...
```

周期 = 3（因为 3,6,9 循环）。

**第二步：Turtle 的向量累积**

每步 Turtle 前进 `dpd*c`，右转 `dpd*c` 度。这等价于在复平面上：

$$z_{n+1} = z_n + (\text{dpd} \times c) \cdot e^{i\theta_n}$$

其中 $\theta_n = \theta_{n-1} + \text{dpd} \times c \times \frac{\pi}{180}$

**第三步：闭合条件**

当累积转角 $\sum \text{dpd} \times c \approx 360^\circ \times N$ 时，Turtle 回到原点附近 → 一圈闭合。因为数根周期性，这个闭合总是发生。

**关键参数效应**：

| 参数 | 名称 | 小值效果 | 大值效果 |
|:---|:---|:---|:---|
| `step` | 步长 | 数根变化慢 → 多层"花边" | 数根变化快 → 密集的细瓣 |
| `coefficient` | 缩放 | 精细小图 | 粗犷大图 |
| `min_int` | 起始 | 图案的"中心点"数根值 |

**为什么不同参数产生不同曼陀罗？**

因为 `step` 改变的是数字根序列的**周期长度**：

| step 值 | 周期 | 图案特征 |
|:---|:---|:---|
| 1 | 9 | 9 组不同长度段 → 9 瓣花 |
| 2 | 9 | 同 step=1 但起始相位不同 |
| 3 | 3 | 3 组段循环 → 3 瓣花 |
| 4 | 9 | 9 瓣但段长序列不同 |
| 6 | 3 | 3 瓣但角度不同 |
| 7 | 9 | 9 瓣逆向 |
| 9 | 1 | 恒为起始数的数根 → 正多边形！|

**这就是"同一个函数、不同参数产生完全不同的图形"的根因：数字根周期长度决定了花瓣数，步长决定花瓣密度。** 🎯

---

#### 算法 B: `make_circle_ratio()` — 几何级数

```python
cnt = initial * ratio  # cnt 按等比递增
```

与算法 A 的唯一区别：**`cnt` 是等比增长而非等差增长**。

**数学根因**：等比级数 `initial × ratioⁿ` 的数根分布完全不同于等差级数：

- 当 `ratio=2` 时，`cnt` = 2, 4, 8, 16, 32, 64, 128, 256...
- 数根 = 2, 4, 8, 7, 5, 1, 2, 4... — **周期 6 的循环 {1,2,4,8,7,5}**
- 这就是 Rodin 的"加倍电路"（doubler circuit）！

**图案差异**：
- 等差（difference）：周期 3 或 9 → 3 瓣或 9 瓣对称花
- 等比（ratio=2）：周期 6 → 6 瓣 + 缺失 3,6,9 → 不对称曼陀罗，中心有空腔

### 2.4 `fib.py` / `draw_fib()` — Fibonacci 螺旋

`fib.py` 只输出数根，但 `prove the law of universe.py::draw_fib()` 画了 Fibonacci 螺旋：

```python
for num in range(30):
    length = fib(num)   # 1,1,2,3,5,8,13...
    forward(length)
    right(90)

# 然后画四分之一圆
for num in range(30):
    length = fib(num)
    circle(-length, 90)
```

**数学根因**：

1. **直线段**：每条线段 = $F_n$，直角转角 → 形成 **Fibonacci 矩形螺旋**（黄金比例近似从 $F_n/F_{n-1} \to \phi$）
2. **四分之一圆弧**：每条弧线半径 = $F_n$，连接各矩形的对角 → 形成 **Fibonacci 黄金螺旋**

$$\frac{F_{n+1}}{F_n} \xrightarrow{n\to\infty} \phi = 1.618...$$

| 维度 | fractal dragon | polygon | circle_difference | circle_ratio |

## 四、核心洞察：数根 = 曼陀罗的"基因"

**所有蓝馨老师的涡旋图案都可以归约到同一个数学框架**：

```
┌──────────────────────────────────────────┐
│         数字根序列 dr(n)                  │
│              │                           │
│     ┌────────┼────────┐                  │
│     │        │        │                  │
│  等差dr    等比dr   Fibonacci dr         │
│ (周期3/9) (周期6)   (非周期但收敛)        │
│     │        │        │                  │
│     ▼        ▼        ▼                  │
│  fd(dr*c)  fd(dr*c)  fd(F_n)            │
│  rt(dr*c)  rt(dr*c)  rt(90°)            │
│     │        │        │                  │
│  对称花   曼陀罗   黄金螺旋              │
│ (3/9瓣)  (6瓣+空腔) (矩形+圆弧)          │
└──────────────────────────────────────────┘
```

**本质**：Turtle 的 `forward()` 和 `right()` 是两个映射函数：

$$f: \mathbb{N} \mapsto \mathbb{R}^2$$

$$f(n) = \sum_{k=1}^n \text{dr}(k) \cdot e^{i \cdot \sum_{j=1}^k \text{dr}(j)}$$

改变数字根的生成方式（等差/等比/Fibonacci）→ 改变 $dr(k)$ 的序列结构 → 改变复平面上路径的形状。

**这就是 why：同一个 Turtle，不同的数根序列，产生完全不同的曼陀罗。数根序列是图案的"DNA"，Step 决定了它的"周期基因"。**

## 五、更深的连接：为什么这些图案出现在自然界

五种自然模式与四种 Turtle 算法的对应：

| 自然模式 | Turtle 算法 | 共享的数学 |
|:---|:---|:---|
| 鹦鹉螺壳 | Fibonacci 螺旋 | 黄金比例 $\phi$ 对数螺线 |
| 向日葵花盘 | circle_difference(3,3,3) | 数根周期 3 | 
| 雪花 | fractal dragon | 自相似分形 |
| 龙卷风截面 | polygon (多层嵌套) | 多尺度圆周叠加 |
| Enneagram (九点星) | circle_difference(1,1,1) | 模 9 周期完整 |

**这四个文件不是不同的代码——它们是同一个数学框架的四种表述**：数字序列 → 路径积分 → 平面图案。参数的差异只是"基因表达"的差异。
