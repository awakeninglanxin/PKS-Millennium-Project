> **一句话**: 把 896 专家从「扁平平等」重组成「144 本质原型 × 变换副本」，路由先在 144 原型上跑闭式匹配（O(144)），再派发副本。这是对幻方「144 基方 × 25 移位 = 3600」的等价类压缩在 MoE 上的迁移。

---

## 一、为什么是 144？不是拍脑袋

3600 幻方里 144 是**自然涌现**的数：A 拉丁方首行 0 固定 → 4! = 24 种；B 拉丁方首两格 0,1 固定 → 3! = 6 种；24×6 = 144 基方。每个基方通过 Z₅×Z₅ 循环移位（25 个位置）膨胀成 25 个具体方，144×25 = 3600。

类比到 MoE：
- **144 原型** ≈ 144 基方 = 「功能上不可再约的本质专家族」（正交、互不为副本）。
- **变换副本** ≈ 25 循环移位 = 同一原型的「不同实现姿态」（量化版本 / 稀疏度版本 / 不同初始化）。
- **896 专家** ≈ 3600 全方 = 扁平铺开的总专家池。

> ⚠️ **关键差异（必须坦言）**: 幻方里「基方 → 移位方」是**严格群作用**（数学保证一一对应、无重叠）。
> 但 MoE 专家**未必**能干净分解成「原型 × 变换」——专家权重可能是**连续流形**而非离散轨道。
> 所以 144 是**待验证的目标数**，不是先验真理。实验第一步就是确认「896 专家能否聚出 ~144 个清晰原型」。

## 二、整体算法（伪代码）

```python
# ---------- 阶段 0: 离线提取专家原型（训练前/训练后均可） ----------
def extract_prototypes(experts, K=144):
    """
    experts: List[Tensor]  长度 896, 每个是专家权重矩阵(展平)
    返回: prototypes (K个), assignment (896 -> 原型id), replica_of
    """
    # 2.1 把每个专家权重展平为向量, 做谱聚类/标准化
    X = normalize([flatten(e) for e in experts])          # (896, d)
    # 2.2 选 K: 用轮廓系数扫 K=64..256, 取峰值附近
    best_K = argmax_silhouette(X, K_range=range(64, 257, 8))
    labels = spectral_clustering(X, n_clusters=best_K)    # 896 个标签
    # 2.3 每个簇取质心 = 原型
    prototypes = [centroid(X[labels == k]) for k in range(best_K)]
    # 2.4 记录每个专家属于哪个原型(= 该原型的副本)
    assignment = labels
    return prototypes, assignment, best_K

# ---------- 阶段 1: 路由先在原型上闭式匹配 ----------
def route_on_prototypes(router_score, prototypes, top_p=16):
    """
    router_score: (batch, seq, 896) 原始 router logits
    返回: proto_weights (batch, seq, K), proto_ids (top_p 个原型)
    """
    # 把 896 的 score 聚合到 K 原型: 同原型副本取 max(或 mean)
    proto_score = aggregate_by_proto(router_score, assignment)   # (batch, seq, K)
    # 在 K 个原型上跑闭式最优匹配(刀二费用流 / 匈牙利), 选 top_p 原型
    proto_ids, proto_weights = closed_form_match(proto_score, top_p)
    return proto_ids, proto_weights

# ---------- 阶段 2: 派发到具体副本 ----------
def dispatch_to_replicas(proto_ids, proto_weights, assignment, experts_per_proto_budget):
    """
    对每个选中的原型, 在其副本里按激活预算挑具体专家
    """
    chosen_experts = []
    for pid, w in zip(proto_ids, proto_weights):
        replicas = (assignment == pid).nonzero()          # 该原型的副本列表
        # 副本内用原 router_score 排序, 取预算内 top
        local_top = argsort(router_score[replicas])[:experts_per_proto_budget]
        chosen_experts.extend(replicas[local_top])
    return chosen_experts

# ---------- 前向: 用选中的专家算 MoE ----------
y = moe_forward(x, expert_ids=chosen_experts, weights=proto_weights)
```

**复杂度对比**：
| 步骤 | 扁平路由 (K3 现状) | 原型路由 (本文) |
|---|---|---|
| router 候选数 | 896 | 144 (原型) |
| 闭式匹配规模 | O(896) | O(144) ≪ O(896) |
| 副本派发 | 无 | O(副本数) 局部 |
| 等价行为 | 直接选 896 中 top | 先选原型再选副本 |

> 若 best_K ≈ 144，闭式匹配成本降 ~6×；若专家真能分解，路由更稳定（原型正交 → 天然去相关，比 Farey 互素惩罚更干净）。

## 三、评估指标（实验必须报这些）

| 指标 | 说明 | 期望 |
|---|---|---|
| **Silhouette** | 聚类清晰度, 验证「能否聚出 144 原型」 | > 0.3 才说明离散轨道成立 |
| **专精度 (specialization)** | 每个原型激活的 token 类型熵 | 越低越好(原型分工明确) |
| **下游困惑度 / 任务分** | 在验证集上对比扁平路由 | ≤ 扁平路由 (不损精度) |
| **路由方差** | 各原型激活频率的基尼系数 | 低(均衡, 无死原型) |
| **闭式匹配耗时** | 原型路由 vs 扁平路由的路由层 wall-clock | 原型路由应更快 |
| **死原型数** | 训练中从未被激活的原型数 | 0 (否则 K 取大了) |

---

## 四、证伪条件（触发任一条 → 猜想证伪，回退扁平路由）

1. **Silhouette < 0.3** → 专家是连续流形, 聚不出清晰 144 原型 → 等价类压缩在 MoE 不成立。
2. **下游困惑度 > 扁平路由** → 原型聚合丢信息 → 回退。
3. **死原型数 > 10%** → K 过大或聚类错 → 降 K 重试, 仍不行则证伪。
4. **闭式匹配耗时 ≥ 扁平路由** → 聚合+派发开销抵消收益 → 工程上不值。

> 以上任一条触发, 本文猜想降级为「有趣的失败」, 但**不影响** `08` 的 Farey 位置覆盖用法(变量隔离)。

## 五、最小可跑实验路线（里程碑）

```
M0 (0.5天): 从已训 K3 checkpoint 导出 896 专家权重 → 展平 → 跑谱聚类
             → 输出 best_K + silhouette 曲线  (证伪条件 1 先看)

             验证原型路由不损精度 (控制变量, 排除规模干扰)
M2 (3天):   在 128 专家 proxy 上跑完整两阶段路由, 报第三节全部指标
M3 (1周+):  放大到 896, 替换 K3 router 的闭式匹配层, 在 100B token 续训对比

```

**M0 是最便宜也最致命的一步**: 如果 silhouette 直接 < 0.3, 后面 M1~M4 都不用跑了, 猜想当场证伪。这正是幻方思维「先证伪再包装」的落地。
