# Kimi K3 架构深度拆解 — 四大数学底层原理

> **来源**: 元宝对话 https://yb.tencent.com/s/7VHXmmiPqFnw  
> **日期**: 2026-07-21 | **模型**: Kimi K3 (Moonshot AI) | **参数**: 2.8T, 896 专家/16 激活, 1M 上下文

## 一、架构全景

K3 的"超越同档"不是单点魔法，是**四条数学/架构改动叠出来的**。核心公式：

```
K3 = MoE Backbone (896 experts, sparsity 56)
   + KDA (O(n²) → O(n) linear attention)
   + AttnRes (depth-axis softmax residual)
   + Stable LatentMoE (Quantile Balancing + Per-Head Muon)
   + Gated MLA + SiTU (attention/activation tweaks)
```

每个 block 的层模式: **3 KDA + 1 Gated MLA** 循环 (线性注意力扛大部分层，全注意力保表达力)

## 二、四大改动逐项拆解

### 2.1 Kimi Delta Attention (KDA) — O(n²) → O(n)

**问题**: 标准 Transformer 的 QK^T 矩阵是 O(n²)。1M token 上下文是死穴。

**KDA 路线**:
- **Delta 压缩**: 不直接算完整 QK^T，用 Delta 函数对 Q-K 交互做稀疏化压缩
- **线性近似**: 保留关键注意力模式前提下把复杂度压到 O(n)
- **残差补偿**: 压缩丢的高频细节用轻量残差接回来

**结果**: 1M 上下文解码最高 **6.3×** 加速。Chris Wirz 那篇矩阵级推导把 KDA 写成了"递归固定的 state 矩阵"——序列多长 state 都不涨。

### 2.2 Attention Residuals (AttnRes) — 改残差连接

**问题**: 标准 `x_l = x_{l-1} + f_l(x)` 中历史层按固定权重 1 累加，深网络新层信号被稀释。

**AttnRes 做法**: 把"横向 sequence 轴的 attention"转 90° 用到纵向 depth 轴:
- 当前层对前面各层算 softmax 权重，按输入动态挑有用的早期表示
- **Block AttnRes** (陈广宇+张宇): 多层压成块摘要，代价 <2%，训练效率 +25%

**结果**: GPQA +7.5%, 数学 +3.6%, 代码 +3.1%，同等算力 ≈ 1.25× 基线。

### 2.3 Stable LatentMoE + Quantile Balancing

**问题**: 896 专家/16 激活 (稀疏度 56)，传统 aux-loss 路由敏感且需调超参。

**改进**:
- **Quantile Balancing**: expert 分配从 router-score 分位数直接派，消掉 heuristic 平衡超参
- **Per-Head Muon**: 每个 attention head 独立二阶预条件 (K2 MuonClip 升级)
- **QK-Clip**: 对 Q/K 投影做动态裁剪 (τ=100)，防 logit 爆炸

### 2.4 Gated MLA + SiTU

- **Gated MLA**: DeepSeek MLA 变体，KV 压到 ~512 维 latent (KV-cache 压 13×)
- **SiTU** (Sigmoid Tanh Unit): 新激活函数，比 SwiGLU 在超大 MoE 下更稳定

## 三、复杂度总览

| 组件 | 原复杂度 | K3 后 | 改进 |
|:---|:--:|:--:|:--:|
| Attention (标准) | O(n²) | — | — |
| Attention (KDA) | — | O(n) | 6.3× decode 加速 |
| KV Cache (标准) | O(n·d) | — | — |
| KV Cache (MLA) | — | O(n·latent_dim) | 13× 压缩 |
| Residual (标准) | 固定权重 1 | — | — |
| Residual (AttnRes) | — | softmax 加权 | 1.25× 算力等效 |
| MoE routing (传统) | aux-loss heuristic | — | — |
| MoE routing (K3) | — | Quantile 闭式 | 无超参 |

## 四、K3 之后还能改什么（未来五刀）

> 📌 **【来自 3600 幻方算法的修订：每刀配 falsification condition】**
> 原文档三刀（见 `06~08`）收益全是正向预期。幻方实践的纪律是「敢接受 NO」——
> 给每刀加一个**证伪条件**，实验若触发则说明该刀在当前设定下无效，应回退而非硬凑。
> 下列「证伪条件」列即为此补。详见 `09_思维层交叉分析`。

按数学收益/工程可行性排序:

| 优先级 | 改动 | 目标 | 收益 | **证伪条件（触发则回退）** |
|:--:|------|------|:--|---|
| 🥇 | MoE routing 理论闭环 | bias 动态+全局统计+正交约束 → 踢掉 aux-loss | 同等专家数专精度↑ | 若专精度 < Quantile Balancing 基线 → 正交约束过强，回退 |
| 🥈 | Hybrid 插 Mamba-2/DeltaNet | 70-80% 层换线性 recurrence | decode 再翻倍 | 若长程召回 < 纯 KDA → recurrence 占比过高，降比例 |
| 🥉 | PosEnc 从 RoPE 迁到 SSM decay | 扔掉 NTK hack + O(n·d) 成本 | 长上下文更稳 | 若短程任务掉点 > 1% → SSM decay 不适合短程，回退 RoPE |
| 4 | FFN 局部 TTT 化 | 前馈也做成 state 自监督 | 和 MoE 不冲突 | 若训练不稳/显存爆 → 部分层回退标准 FFN |
| 5 | DyT 替 RMSNorm | 训练时省逐 token stat | 深模型累积可观 | 若深层 loss 跳变 > 1% → α 初始化公式错，回退 RMSNorm（见 `06`） |
