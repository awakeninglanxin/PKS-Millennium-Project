## 二、Python→DXF→Rhino9完整工序（24步）

### Phase A: Python生成DXF（代码端）

```
Python程序
  ↓ 输出.DXF文件（含导轨线+截面曲线+中轴线+点阵）
  ↓ 文件名命名：四相文件 × 4个（幻方4相数字对应4个关节数）
```

### Phase B: Rhino9处理（来自DOCX逐步骤）

| 步骤 | 指令 | 参数 | 目的 |
|:---:|:---|:---|:---|
| **B1** | `_RebuildCrvNonUniform` | 目标公差0.06, 点数21 | 重建截面曲线（绿色封闭线） |
| **B2** | 选分段扫掠曲线→重建 | 目标公差0.001, 点数41 | 精细重建导轨线 |
| **B3** | Join后重建 | 目标公差0.06, **反复回车至832点** | 保证每根导轨线控制点一致（相位差π→971点） |
| **B4** | **多轨扫掠** | Rhino9+卓尔谟插件 | 所有导轨线+所有截面线→生成螺旋曲面 |

### Phase C: 曲面处理

| 步骤 | 指令 | 参数 | 目的 |
|:---:|:---|:---|:---|
| C1 | 复制备份+封盖 | — | 保留原曲面 |
| C2 | `_DrowmOffsetSrfLoose` | 距离1.1mm, 实体=是, 松弛=是 | 偏移加厚1.1mm → 壁厚 |
| C3 | 重新导入DXF→保留中轴线 | — | 准备支撑结构 |

### Phase D: 支撑圈（Loft挤出环）

| 步骤 | 操作 |
|:---:|:---|
| D1 | 选红线→Loft生成曲面→自动对齐工作平面 |
| D2 | Shift向两侧挤出0.75mm（总厚1.5mm）**注意最上排红线下挤1.5mm** |
| D3 | 所有loft实体重建→向Z轴两侧挤出1.1mm |

### Phase E: 布尔合并

```
中轴圆柱 + loff支撑环实体 → 布尔合并
    ↓ 布尔交集
螺旋曲面实体
    ↓ 布尔合并
最终实体 = 螺旋管 + 一体支撑
```

### Phase F: 四相处理

> "幻方四相数对应共有四个文件，文件同样运用以上工序走一遍"

| 四相文件 | 关节数 | 实测结果 |
|:---|:---:|:---|
| 文件1 | 幻方数字1 | 🔴 动平衡差、抖动 |
| 文件2 | 幻方数字2 | 🔴 同上 |
| 文件3 | 幻方数字3 | 🔴 同上 |
| **统一简化** | **全用一样关节数** | ✅ 简单、动平衡好 |

## 三、实测经验教训（来自DOCX）

### 3.1 选用哪个锥？

| 锥颜色 | 大小 | 结果 | 原因 |
|:---|:---:|:---|:---|
| **绿色锥** | 适中 | ✅ **推荐** | 高速转稳、出水好 |
| 白色锥 | 太大 | ❌ 淘汰 | 重、螺周期多一圈、支撑变形、管末端不出水 |

### 3.2 四相不同关节数 → 放弃

> "四相不同数字对应代表四螺管关节数量的不可用，原因动平衡太差，质心不在中心，抖的厉害"

**解决方案**: 改成全部用同样数量关节的螺锥。

### 3.3 设计调优

| 问题 | 改进 |
|:---|:---|
| 轴孔径 | 再减小0.1mm（玻璃纤维棍） |
| 白色锥进气圆孔 | 慢转速出水花、高速散架→删除该设计 |
| 未来材料 | 316L+银/金/铜离子, 壁厚1.5mm |
| 管内壁 | mof法镀纳米金/银膜 |

### 3.4 幻方四相 ≠ 四管关节数不同

DOCX明确指出"幻方四相数对应共有四个文件"，但**四相不是四个不同关节数的管**——是对应幻方中不同相位/不同数字的**相同结构的管在圆周上的四个位置**。原始想法用不同关节数是为了区分相位，但实测证明动平衡不行。
