# 蛋形曲线生成算法：数学工序逻辑说明与对比分析

> **核心结论**：项目中 6 种蛋形曲线生成公式，源自 **1 种底层几何**（超双曲锥 xy=1 的平面斜切），对应 **6 套不同代数形式**的参数化策略。公式形态各不相同，但生成的曲线完全一致（偏差 < 0.001）。选型时应在速度、品质、数学精确性之间权衡。

---

## 一、数学逻辑总览

**1 种底层几何 → 6 套不同代数公式 → 同一条蛋形曲线**

```
超双曲锥 xy=1 平面斜切  （唯一底层几何）
    │
    ├─ 6 套不同代数公式（公式形态各不相同，曲线一致）
    │   ├─ A1 显式形式: x = ±√(1/(z₀-y·sinα)² - (y·cosα)²)    [ȳ均匀采样]
    │   ├─ A2 参数形式: r = (z₀±√(...))/(2·cosφ·sinα), x=r·sinφ, y=r·cosφ  [φ均匀采样]
    │   ├─ A3 t-参数化: x = √(1/(z₀+t·sinα)²-(t·cosα)²), y = -t  [解析求根]
    │   ├─ B  Rhino方程: x(t)=a·2sin(t)/(b+√(b²-4k·cos(t))), y(t)=复杂表达式  [Schauberger推导]
    │   ├─ C  r_minus分支: 与A2同源, 仅取r⁻分支  [坐标交换]
    │   └─ D  八度族: 与B同源, 递推参数(k=4ⁿ/6, b=5·2ⁿ/6)  [八度族推广]
    │
    └─ 所有算法验证: 最坏偏差 < 0.001（见§三偏差分析）
```

> 📌 **它们是同一几何对象的不同代数表达**，就像圆的 x²+y²=1, x=cos(t), y=sin(t), x=±√(1-y²) 等不同形式，公式不同但曲线相同。

## 二、算法详述

### A1 显式形式（explicit_form）

| 项目 | 内容 |
|------|------|
| **数学来源** | 超双曲锥 xy=1 平面斜切 |
| **核心公式** | x̄ = ±√( 1/(z₀ - ȳ·sinα)² - (ȳ·cosα)² ) |
| **参数体系** | z₀ = (z₁²+z₂²)/(z₁+z₂), α = arctan(z₁·z₂·(z₂-z₁)/(z₁+z₂)) |
| **采样策略** | 二分法找 ȳ 零点边界 → ȳ 均匀采样 → 逐点代入显式公式解得 x |
| **返回形式** | 右半 x≥0 曲线 + 左半镜像拼接 → 闭合曲线 |

**工序逻辑**：
```
1. 二分法求解 ȳ_min 和 ȳ_max（曲线在 ȳ 方向的有效范围）
2. 在 [ȳ_min, ȳ_max] 上均匀取 n 个 ȳ 值
3. 各点代入显式公式 → 若根内值>0则 √ 得 x，否则 NaN
4. 右侧: x_right = √(inner), 从上到下
5. 左侧: x_left = -x_right(逆序), 从下到上
6. 拼接 → 闭合曲线

【已知缺陷 (已修复 2026-05-28)】二分法边界与显式根不一致 → 以前接缝开裂 0.059
【修复】explicit_form 的 valid = inner > 0 → inner >= 0，边界点 inner=0 不再被误判为 NaN
【当前状态】接缝完美闭合 (0.00e+00)，但两极 (ȳ→ȳ_min/ȳ_max) 采样点仍稀疏
```

**代码位置**：`01_geometry/egg_curve.py` → `EggCurve.explicit_form()` + `get_curve_points_ybar()`

---

### A2 参数形式（parametric_form）

| 项目 | 内容 |
|------|------|
| **数学来源** | 超双曲锥 xy=1 平面斜切（极坐标参数化） |
| **核心公式** | r = (z₀ ± √(z₀² - 4·cosφ·sinα/√(sin²φ+cos²φ·cos²α))) / (2·cosφ·sinα) |
| **参数化** | φ ∈ [0, 2π] 均匀采样 → 计算极径 r → 变换回笛卡尔坐标 |
| **返回形式** | 两条分支：r_plus (锥管部分) + r_minus (平坦山体部分) |

**工序逻辑**：
```
1. φ ∈ [0, 2π] 均匀取 n 个值
2. 避开 cosφ=0 的奇点（φ=π/2, 3π/2）
3. 各 φ 值代入公式计算 r_plus 和 r_minus
4. x = r·sinφ, y = r·cosφ 变换回笛卡尔坐标
5. r_minus 分支经坐标交换（y→x, x→y）→ 闭合蛋形曲线

【特性】纯向量化运算，无逐点循环，速度极快
【特性】首尾点由 φ=0 和 φ=2π 的自然连续性保证 → 完美闭合
```

**代码位置**：`01_geometry/egg_curve.py` → `EggCurve.parametric_form()`

---

### A3 t-参数化（get_curve_points_t）★ 推荐默认

| 项目 | 内容 |
|------|------|
| **数学来源** | 超双曲锥 xy=1 平面斜切（x-坐标参数化） |
| **核心公式** | x = √(1/(z₀+t·sinα)² - (t·cosα)²), y = -t（翻转使尖朝上） |
| **参数体系** | 与 A1/A2 共享同一组 (z₁, z₂, z₀, α) |
| **采样策略** | 解析求解 t 的有效范围 → t 均匀采样 → 逐点验证值>0 → 边界插入 |

**工序逻辑**（最精细）：
```
1. 解析求解 t 的有效范围：
   - 钝端方程: a·t² + b·t - 1 = 0 → 取正根 t_blunt
   - 尖端方程: a·t² + b·t + 1 = 0 → 取内根 t_tip（更靠近 0 的根）

   ⚠️ 关键陷阱：
   二次方程有两个负根，v(t)=1/(z₀+t·sinα)²-(t·cosα)²
   在两根之间为负数（见§七）。只有"内根"才是蛋尖边界。
   ❌ 外根 (-b-√D)/(2a) → hyperbola 另一臂边界，非蛋形！
   ✅ 内根 (-b+√D)/(2a) → 蛋尖真实边界，与 Rhino/B 一致

   其中 a = sinα·cosα, b = z₀·cosα

2. 在 [t_tip, t_blunt] 上均匀取 n 个 t 值

3. 逐点代入：
   value = 1/(z₀+t·sinα)² - (t·cosα)²
   if value > 0: x = √value, 保留该点

4. 显式插入边界点 → 强制闭合：
   x_right = [0(尖端), ..., 0(钝端)]
   x_left  = [0(钝端), ..., 0(尖端)]

5. 翻转 y = -t 使蛋尖朝上 → 拼接闭合曲线

【特性】解析求根确定精确范围 → 100% 全覆盖
【特性】逐点 loop 验证 → 慢（9.4ms/10000点，较B/D慢~26倍）
【特性】t 均匀采样 → 两极区稀疏但无接缝瑕疵
【2026-05-28 修复】取根错误导致尖刺的数学原理见§七
```

**代码位置**：`01_geometry/egg_curve.py` → `EggCurve.get_curve_points_t()` + `_find_t_roots()`

### B Rhino参数方程（x_func / y_func）

| 项目 | 内容 |
|------|------|
| **数学来源** | Schauberger 原始 Rhino 脚本导出参数方程 |
| **核心公式** | x(t) = a·2sin(t) / (b + √(b² - 4k·cos(t))) |
| **参数体系** | 使用 (k, b, m) 替代 (z₁, z₂)，八度蛋: k=2/3, b=5/3, m=2/3 |
| **采样策略** | t ∈ [0, 2π] 均匀采样 → 逐点计算 x(t), y(t) |

**工序逻辑**：
```
1. t ∈ [0, 2π] 均匀取 n 个值
2. 各 t 值代入 x_func 计算水平坐标：
   denom = b + √(b² - 4k·cos(t))
   x = a·2sin(t) / denom

3. 各 t 值代入 y_func 计算垂直坐标（复杂表达式）：
   sqrt_1k2 = √(1+k²)
   sqrt_b2_4k = √(b²-4k)
   term1 = -√(1+k²)·(-√(b²-4k)+√(b²-4k·cos(π))) / (2k)
   term2-term3 = 复杂的 k,b 组合表达式
   y = a·(m·term1 + term2)

【特性】纯向量化，最快之一(0.28ms/10000点)
【特性】t 周期边界连续性 → 完美闭合
```

**代码位置**：`01_geometry/egg_variations.py` → `EggVariationGenerator.x_func()` / `y_func()`

## 七、外根陷阱：A3 t-参数化尖刺Bug的数学原理与修复

> **问题现象**：A3 蛋尖出现异常突起（y≈1.80），正常蛋尖应为 y≈1.20。
> **根因**：`_find_t_roots()` 中二次方程取错了根。
> **修复**：取"内根"而非"外根"。

### 7.1 数学推导

t-参数化的蛋形曲线由下式定义：

```
x(t) = √[ 1/(z₀ + t·sinα)² - (t·cosα)² ]
y(t) = -t  (翻转使尖朝上)
```

边界条件（x=0）:

```
1/(z₀ + t·sinα)² - (t·cosα)² = 0
→ (z₀ + t·sinα)·(t·cosα) = ±1
```

对于尖端（t<0, 蛋形最窄处），展开为：

```
sinα·cosα·t² + z₀·cosα·t + 1 = 0     (尖端方程)
```

### 7.2 外根陷阱：为什么二次方程有两个解？

令 a = sinα·cosα, b = z₀·cosα，则尖端方程为 `a·t² + b·t + 1 = 0`。

判别式 Dₜᵢₚ = b² - 4a > 0，两个负根：

| 根 | 公式 | 值（八度蛋） | 含义 |
|:--:|:----:|:----------:|:-----|
| **外根** t_outer | `(-b - √D)/(2a)` | -1.803 | 超双曲锥**另一臂**的边界 |
| **内根** t_inner | `(-b + √D)/(2a)` | **-1.202** ← 这才是蛋尖！ | 蛋形曲线有效起点 |

### 7.3 关键洞察：v(t) 在两个根之间为负

函数 v(t) = 1/(z₀ + t·sinα)² - (t·cosα)² 的符号分布：

```
     t < -1.803     |  -1.803 < t < -1.202  |  -1.202 < t < 0.601  |  t > 0.601
  ──────────────────┼─────────────────────────┼──────────────────────┼─────────────
   v > 0 (双曲锥)   |    v < 0 (禁区！❌)    |   v > 0 (← 蛋形曲线!) |   v < 0
                   ↑                         ↑
              外根 (非蛋形)             内根 = 蛋尖 ✅
```

两个根之间 v(t) 为负值，说明 t_outer 和 t_inner 之间**没有有效的 x 解**。外根是超双曲锥另一支的解析边界，物理上不参与蛋形曲线。

### 7.4 代码修复（精确到行）

```python
# egg_curve.py → EggCurve._find_t_roots(), 原第183行

# ❌ 旧代码 — 取外根 (hyperbola 另一臂, y≈1.80)
t_tip = (-b - np.sqrt(disc_tip)) / (2 * a)

# ✅ 新代码 — 取内根（靠近0的根，v>0有效区的起点, y≈1.20）
t_tip = (-b + np.sqrt(disc_tip)) / (2 * a)
```

## 八、完整性验证

### 8.1 6 套公式 ≠ 6 种数学逻辑

> 所有 6 种算法共享 **1 种底层几何**（超双曲锥 xy=1 平面斜切），但采用了 **6 套不同的代数形式**。就像圆的 `x²+y²=1`、`x=cos(t)`、`y=sin(t)` 是同一对象的不同表达，公式不同但曲线完全一致。

### 8.2 交叉验证结论

- 6 套独立公式在相同参数下生成完全相同的曲线（最坏偏差 < 0.001）
- 验证了 z₁,z₂ 参数系 ↔ k,b,m 参数系之间的映射关系正确
- 速度排行榜和品质排行榜不完全正相关：最快(B/D)品质最高(10/10)，最慢(A3)品质也可用(8/10)
