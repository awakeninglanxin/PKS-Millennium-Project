## 🔴 坑1：Croft T_30 在大N下独自失效

### 现象
```
N=1000:    ratio = 19.6 ✅
N=10,000:  ratio = 16.9 ✅
N=10,000,000: ratio = 0.09  ❌（比瞎猜还差）
```

### 根因
Croft T_30筛选后的totatives中，**多质因子合数占71%**（如 7×11=77, 7×13=91, 11×13=143...）。

合数方差的物理来源是**和频干涉**：`cos(-t·log(p₁p₂)) = cos(-t·log p₁ - t·log p₂)`，两个素数频率叠加产生额外方差（184.2），碾压素数方差（19.7）。

### ✅ 正确做法
**SPF（最小质因子）线性筛 + 三分类剔除**：

```python
# SPF 线性筛 O(N)，一次性标记所有数的最小质因子
spf = smallest_prime_factor_array(N)

for n in totatives:
    if spf[n] == n:
        class = "素数"          # 24.9%, var=19.7
    elif spf[n] == spf[n // spf[n]]:
        class = "质数幂"        # 4.2%, var=0.6
    else:
        class = "多质因子"      # 70.9%, var=184.2  ← 剔除！
```

**剔除多质因子后**：ratio 从 0.10 跳到 30.64（279倍提升），N=10⁸ 也守住 23.57。

> 📌 **铁律78**：Croft合数必须SPF三分类，筛除多质因子是强制步骤。

## 🟡 坑2：BSD Euler积在 p_max<10⁵ 下收敛不够

### 现象
```
p_max=50K:   33曲线 60% 正确率
p_max=200K:  33曲线 69% 正确率
p_max=1M:    21曲线 66% 正确率（仍是 rank 0 主导）
```

### 根因
Euler 积 `Π (1 - a_p·p^{-s} + p·p^{-2s})^{-1}` 在 s=1 处：
- **Rank 0**: L(E,1) ≈ 0.2~0.5，收敛快（几十个素数就够了）
- **Rank 1**: L(E,1) = 0，需要 ~10⁴ 素数才开始逼近
- **Rank 2**: L(E,1) = L'(E,1) = 0，需要 ~10⁵ 素数才有区分力

### ✅ 正确做法
1. **p_max ≥ 10⁶** + **逐曲线收敛外推**：`L = A + B/log(p)` 拟合 3 个截断点
2. **不要用简单 Dirichlet 部分和**：`Σ a_p/√p` 对 rank≥1 根本不算 L(E,1)
3. **Benchmark 必须按 rank 均衡**：70% rank 0 的集合什么也证明不了

## 🟢 坑3：光谱法做有壁面内流 = 死路

### 现象（四次实验全部衰减）
```
光谱法（96³）:       Growth = 0.56×
光谱+螺旋入口（128³）: Growth = 0.93×
光谱+简单IBM（200³）:  Growth = 0.86×
光谱+IBM Re=160（256³）: Growth ≈ 0.80×
```

### 根因：光谱网格没有真正的"壁面"

光谱法在 Fourier 空间中离散——**壁面只是"模糊的一层"格点**，无法建立真实的**无滑移边界层**。

涡量放大需要三个步骤：
1. 壁面边界层产生涡量（`∂u/∂y` 梯度）
2. 涡量从壁面剥离进入主流
3. 几何收缩压缩涡管 → 涡量拉伸放大

**第1步在光谱网格上就不会发生**。再调参数、加入口力、升分辨率都救不回来。

### ✅ 正确路线
**有限体积法 + Cut-cell 沉浸边界（IBM）+ SDF距离场**：
- 每个壁面格点强制速度归零 → 真实边界层
- Cut-cell 精确处理不规则几何
- SDF 从 STL 文件自动生成（保留真实几何精度）

> 📌 **血泪教训**：当数值方法本身不适合物理问题时，调参是浪费时间。先确认方法论正确，再微调参数。

## ☁️ 坑4：GPU云平台踩坑合集

### 4a. AutoDL RTX 4090 经常抢光
- 按量计费关机就释放 GPU 给别人
- 高峰期（白天/晚上）4090 几乎秒光
- **解决**：换 3090（慢40%但便宜35%）或换机房区域

### 4b. SSH 免密登录不稳
- 每次重建实例 host key 变 → 旧密钥失效
- **解决**：别折腾免密，直接用密码 `ssh -p PORT root@HOST` + 密码明文传

### 4c. 先下载结果再关机
- `shutdown -h now` 执行后 SSH 立刻断 → 没来得及下数据
- **解决**：先 `scp` 下载所有产物，确认落盘后最后发关机指令
- ⚠️ **2026-07-16 触发**：exec_command 跑完 CFD 后直接尝试 sftp.get → SSH 连接已断 → 数据丢失，需重新连接下载

### 4c2. 关机前检查 GPU 是否空闲
- **原则**：GPU 上还有其他任务在跑时**绝对不能关机**
- 下载完数据后先 `nvidia-smi` 确认没有残留进程
- 有其他用户/进程在用 GPU → 不关机，只退出自己的脚本
- 按量计费的 GPU 实例：空闲也会扣费，但有任务时关机 = 数据丢失 + 任务中断

### 4d. CuPy 安装 = 一行命令
- 镜像选 `PyTorch 2.1.0 + CUDA 12.1` 最稳
- `pip install cupy-cuda12x` 就搞定，不需要 PyTorch
- 不要选更高 CUDA 版本（CuPy 兼容性没测试过）

### 4e. 登录验证码必须手动
- AutoDL 登录有滑块 CAPTCHA，自动拖不过去
- **铁律76**：CAPTCHA 人工处理，不尝试自动化

## 📊 坑5：Batch benchmark 的样本失衡

### 现象
33曲线集合中 rank 0 占 70%（23/33），"全猜 rank 0"的笨蛋分类器就能拿 70%。

60-69% 的正确率**比瞎猜还差**。

### ✅ 正确做法
构建 benchmark 前先做分布检查：

| Rank | 最少曲线数 | 来源 |
|:---:|:---:|------|
| 0 | 5 | 小导体高确定性 |
| 1 | 5 | 含已知 L'(E,1) 值 |
| 2 | 3 | 389.a1 + 同类 |
| 3 | 2 | 5077.a1 + 同类 |
