## 二、Farey 噪声调度（T4-T5）

### 2.1 原理

扩散模型的噪声调度 β_t 控制每一步加多少噪声。当前主流：线性 β_t = β_min + t(β_max-β_min)/T，余弦 α_t = cos²(πt/2T)。

**问题**：等距调度在特征空间上产生"共振"——某些频率被反复采样，某些频率被跳过。

**Farey 调度的优势**：Farey 序列 F_N 在 [0,1] 上的任意两个相邻分数的间距 = 1/(q₁q₂)，全部不同。这意味着**每一步的噪声水平都与前一步"尽可能远"**——最大化每一步的信息增益。

### 2.2 实现

```python
def farey_noise_schedule(T, beta_min=1e-4, beta_max=0.02):
    """Farey 序列噪声调度"""
    farey = farey_sequence(T)  # 分母 ≤ T 的全部既约分数
    # 取前 T 个 Farey 分数
    t_values = np.array([f[0]/f[1] for f in farey[:T]])
    # 映射到 β 范围
    betas = beta_min + t_values * (beta_max - beta_min)
    return betas
```

### 2.3 GPU 验证

对比三组噪声调度在 CIFAR-10 上的 FID：
- 线性：FID ≈ 5.2（基准）
- 余弦：FID ≈ 4.8（当前 SOTA）
- **Farey：待测**（预期：在稀疏训练步数下优于线性）

## 三、DP 拐点检测（T2）

### 3.1 原理

T2 多模态采样的核心问题是：**什么时候停止采样？** 当前用 ESS（有效样本量）→ 但 ESS 在多模态场景下缓慢爬升，浪费计算。

DP（Douglas-Peucker）算法的原理是"保留 chordal deviation 最大的点作为拐点"。如果将"发现的 mode 数量"画成一条累积曲线：

```
mode_count(t) → 曲线有拐点：拐点之后每 1000 步只新增 0.1 个 mode
                                                     ↑
                                            DP 检测这个拐点
                                            建议在此终止采样
```

### 3.2 M=15 效率冠军的直接应用

我们在逆M cardioid 上已验证：M=15 的 DP 拐点序列 (40,78,82,...,540) 在 540 处收敛，此后仅需 24 个微调点。**同样的算法直接用于 mode 发现曲线 → 给出最优采样步数。**

## 四、B-smooth 稀疏采样（T3）

### 4.1 原理

T3 高维 score matching 的维度灾难：在 d 维空间中估计 score function ∇log p(x)，方差 ∝ d。

**B-smooth 稀疏采样的思路**：不采样所有 d 维坐标，而是只采样"B-smooth 索引子集"。对 7 基底 {2,3,5,7,11,13,17} 的 smooth 数集合，其密度在自然数中随 index 增长而指数递减——但**覆盖了 94.7% 的素数分布**。这意味着用 smooth 索引采样子集，可以保持大部分结构信息，同时减少 √(d) 的方差因子。

### 4.2 具体实现

```python
# 标准 score matching：采样全部 d 维
indices = np.random.choice(d, size=batch)

# B-smooth 稀疏版：只采样 smooth 索引
smooth_indices = generate_b_smooth([2,3,5,7,11,13,17], d)
indices = np.random.choice(smooth_indices, size=batch)
```

## 五、SPF 负噪声（T5）

### 5.1 原理

扩散模型的负采样——噪声样本——一律从高斯分布抽取。但高斯的统计特性（均值=0、方差=1、独立同分布）是否最优？

**SPF 素数噪声**：从素数分布中采样噪声。素数的统计特性与高斯截然不同——素数间距有长程相关性（Cramér 模型），可能为扩散模型提供不同的"信息梯度"，帮助 score function 学习更丰富的结构。

### 5.2 关键假设

如果素数噪声的 score function 比高斯噪声更难估计 → 用素数训练的扩散模型对高斯的泛化更好（训练更难的先去噪场景 → 测试更易的高斯去噪 = 性能提升）。

**这是一个可测试的假设——GPU 可以直接验证。** 需要跑两组对照实验：高斯噪声训练 vs SPF 噪声训练，在同一个去噪任务上比较 MSE。
