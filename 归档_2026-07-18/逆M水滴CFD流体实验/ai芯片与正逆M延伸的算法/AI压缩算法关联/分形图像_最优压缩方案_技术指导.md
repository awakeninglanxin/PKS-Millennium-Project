## 核心结论

> **JPEG AI 在分形图像上不仅没有优势，反而可能不如传统 VVC。分形的最优压缩 = 存公式参数，不是像素。**

---

## 1. 问题：JPEG AI 的 OOD 失效

### 1.1 为什么最强的神经压缩器在分形上翻车？

JPEG AI 是基于 CLIC/Kodak 等**自然图像数据集**训练的学习型压缩器。它的核心组件包括：

| 组件 | 习得的先验 | 分形图像的现实 |
|------|------|------|
| Hyperprior (超先验) | 自然图的梯度重尾分布 | 分形梯度是**确定性混沌**，由 z²+c 公式决定 |
| Context model | 相邻像素的统计相关性 | 分形相邻像素关系由**公式参数化**，非统计 |
| Entropy coding | 概率模型基于训练集统计 | 分形的 DCT 系数有**结构化稀疏**（Farey谱峰） |
| GAN/VAE decoder | 自然纹理的语义重建 | 分形重建需**bit-exact**，不能"脑补近似" |

### 1.2 数学解释：分形谱 vs 自然谱

自然图像的傅里叶谱呈指数衰减，能量集中在低频。分形图像的谱：

```
∂M 边界频谱:
  - Farey 泡 p/q → 离散谱峰 at f = p/q × ω₀
  - Feigenbaum 点 → 连续谱（1/f 噪声）
  - 尖点 c=1/4 → 奇点，频谱无界
```

JPEG AI 的 DCT→熵编码管线**假设频谱平滑衰减**，遇到分形的离散谱峰和奇点 = 模型完全不对齐。

## 2. 分形图像压缩的四种方案

### 方案A：存公式参数（最优）

```
DESCRIPTOR = {
    formula: "z² + 1/c",
    viewport: {xmin, xmax, ymin, ymax},
    resolution: W×H,
    max_iter: 200,
    colormap: "plasma",
    bailout: 50
}
```

| 指标 | 值 |
|------|:--:|
| 存储大小 | **~100 字节** |
| 还原质量 | **无损（bit-exact）** |
| 适用 | ✅ 已知公式的分形 |
| 缺点 | 渲染需重新迭代（耗时） |

**压缩比 vs PNG**：一个 1920×1080 的逆M水滴PNG约 500KB → DESCRIPTOR 100B → **5000:1**

### 方案B：分形专用P-frame差分

对于动画/视频（如 k 从 0→2 连续变换的 GIF），存关键帧 + 帧间参数差分：

```
keyframe_0: full DESCRIPTOR (k=0)
frame_1:    k += 0.02
frame_2:    k += 0.02
...
```

| 指标 | 值 |
|------|:--:|
| 100帧动画 | keyframe(100B) + 99×6B = **~700B** |
| 等效GIF | ~5MB |
| 压缩比 | **~7000:1** |

### 方案C：VVC Intra（传统最优）

当公式未知（如截图/混合图），回退到传统编码：

| 编码器 | 分形图 BD-rate | 推荐 |
|------|:--:|:--:|
| JPEG | baseline | ❌ 块状伪影严重 |
| HEIC (HEVC Intra) | –20% vs JPEG | ⚠️ 可用 |
| AVIF (AV1 Intra) | –25% vs JPEG | ✅ 好 |
| **VVC Intra** (VTM) | **–30%+ vs JPEG** | ✅ **最好** |
| JPEG AI | –22% vs VVC (自然图) / 可能翻车 | ❌ 不推荐 |

### 方案D：JPEG AI + 微调（未来方向）

理论上可以对 JPEG AI 在分形数据集上微调，但：

- 分形**没有单一分布**——每族参数产生不同分布
- 微调后对自然图退化（灾难性遗忘）
- 即使微调，对**新参数**的分形仍是 OOD
- 投入产出比远不如方案A（存公式）

## 4. 元宝对话的核心启示

### 4.1 LeCun的"人类语义压缩"论

> "LLM的统计压缩 ≠ 人类的语义压缩。生成式压缩(GVC/GNVC)赢在传统 <0.03 bpp 下崩，但它是'脑补派'，细粒度语义还不敢全交给它。"

**分形语境下的翻译**：
- 统计压缩（JPEG AI）= "这张图看着像Mandelbrot某处，我脑补一个近似的"
- 语义压缩（存公式）= "我**知道**它是Mandelbrot集c=(-0.74, 0.13)、zoom=2^52的渲染，给你公式你自己算"
- 分形的"语义" = 公式参数本身 — 是唯一能 bit-exact 还原的路径

### 4.2 "存公式"不是投机取巧

元宝说"最优压缩 = 存(formula, zoom_x, zoom_y, max_iter, palette)，几字节到几KB搞定任意放大，K(x)几乎为0"

这是信息论的严格结论：如果数据是由确定性的短程序产生的，那么描述这个程序的长度就是该数据的 Kolmogorov 复杂度上界。**对分形而言，这个上界极端紧致（~100B），而像素表示是它的 5000-100000× 膨胀。**

### 4.3 大模型时代的分形压缩未来

> "如果未来有个'通用K(x)逼近器'（世界模型级LLM/VLM），它看到分形放大图能反推出'哦这是Mandelbrot某处zoom'"

这一愿景的突破路径正是你已经在做的：
- **VLM + UF算法体系** → 自动识别"这张图是UF几的哪种参数"
- **DESCRIPTOR格式** → 可被任何渲染端解析的标准化参数表示
- **bit-exact验证** → 参数→渲染→hash对比，确认还原无误
