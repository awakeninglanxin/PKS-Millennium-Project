# 逆M水滴渲染算法 — 底层全解

> 补充：原理2「正逆M对偶」只讲了数学变换，本文补全**像素级渲染算法**的实现细节。

## 一、什么是"逆M水滴"

普通 Mandelbrot 集 (正M) 的迭代公式是 `z → z² + c`，得到的是向左开口的"苹果形"主心形。

**逆M水滴**的迭代公式是 `z → z² + 1/c`，得到的是一个**尖端朝上的水滴形**。

因为 `c` 和 `1/c` 互为复反演（complex inversion），正M的心形凹陷（左开口）被反演成水滴的凸起尖端（朝上）。

---

## 二、核心公式链（4步）

### 步骤 1：幂变换 — 正逆M的连续滑动

```
alpha = 1 - 2*s          (s ∈ [0, 1] 是滑块值)
c_eff = c^alpha          (极坐标幂运算)
```

| s 值 | alpha | c_eff | 对应迭代 | 形状 |
|:---:|:---:|------|------|------|
| 0 | +1 | z → z² + c | 正M (开口向左) |
| 0.5 | 0 | z → z² + 1 | 退化圆 |
| 1 | **-1** | z → z² + **1/c** | **逆M水滴 (尖朝上)** |

### 步骤 2：标准 Mandelbrot 迭代

```javascript
z_x = 0, z_y = 0                      // 初始值恒为 0
for i = 0 to maxIter:
    // 一步二次迭代：
    new_zx = zx² - zy² + c_eff.re      // Re(z²) + Re(c_eff)
    new_zy = 2·zx·zy + c_eff.im        // Im(z²) + Im(c_eff)
    zx = new_zx; zy = new_zy
    
    if zx² + zy² > R²:                 // R² = 2500 (逃逸半径)
        break                           // 逃逸 → 外部点上色
```

**关键点**：算法里没有显式做 `cinv()`（复反演）。代码第473-477行直接把 `c^alpha` 算出来放进迭代器——`s=1` 时 `alpha=-1`，`c^(-1)` 自动等价于 `1/c`。不需要分两步。

```javascript
// 这就是"1/C"的实现——藏在幂运算里
const r = Math.hypot(cxv, cyv);           // |c|
if (r < 1e-13){ er=1e6; ei=0; }           // 原点→无穷 (1/0)
else {
  const rp = Math.pow(r, alpha);           // |c|^alpha
  const aa = alpha * Math.atan2(cyv,cxv);  // alpha*arg(c) = -arg(c)
  er = rp*Math.cos(aa);                    // Re(1/c)
  ei = rp*Math.sin(aa);                    // Im(1/c)
}
// 然后: z → z² + (er, ei)  即 z → z² + 1/c
```

### 步骤 3：90° CCW 旋转 — 让水滴尖朝上

正M 坐标系中，主心形沿实轴（Re）向左开口。渲染时交换了坐标轴和方向：

```
像素纵轴 py → Re (且上大下小，所以尖端=小Re在上方)
像素横轴 px → Im (左小右大)
```

实际代码：
```javascript
cxv = reTop - (py/(H-1))*reSpan;    // 纵轴 = Re, 上大→下小
cyv = imLeft + (px/(W-1))*imSpan;   // 横轴 = Im, 左小→右大
```

这个旋转 + Re轴反转恰好让 `z → z² + 1/c` 的水滴尖朝上。

### 步骤 4：双模式着色

```javascript
if (i >= maxIter){
    // 内部 (未逃逸)：轨道陷阱 → 同心圆纹
    v = (log(trap + 1e-9) * 0.55) % 1;
} else {
    // 外部 (已逃逸)：平滑逃逸彩环 (continuous potential)
    const m2 = zx*zx + zy*zy;
    const nu = log(log(m2)/2/ln2) / ln2;
    v = ((i + 1 - nu) * 0.055 * 2.4) % 1;
}
// v → 0~1 映射到 LUT 颜色表
```

| 着色模式 | 公式 | 视觉效果 |
|------|------|------|
| 内部 | `log(最近距原点距离) · 0.55` | 同心圆纹理 |
| 外部 | `平滑迭代计数 ± 重整化` | 彩虹环带 |

## 三、与标准 M 集渲染的 4 个区别

| 维度 | 标准 Mandelbrot | 逆M水滴渲染器 |
|------|------|------|
| 迭代公式 | `z² + c` | `z² + c^alpha` (s=1 → `z² + 1/c`) |
| 坐标映射 | Re=横轴, Im=纵轴 | **90° CCW 旋转** (Re=纵轴, Im=横轴) |
| Re 方向 | 左→右 (开口向左) | 上→下 (尖朝上) |
| 内部着色 | 纯黑 | 轨道陷阱 `log(trap)` 同心圆 |

## 四、逆M水滴的几何根因

```
         复反演 (c → 1/c) 的几何效应：
         
正M心形边界              逆M水滴边界
    ╭──╮                      /\     
   ╱    ╲                    /  \    
  │  ×   │     ──→         │  × │    (× = 原点 0)
   ╲    ╱                    \  /
    ╰──╯                      \/
  
反演把心形的内凹(左侧) 映射到水滴的外凸尖端(顶部)
反演把原点(内部)映射到无穷(外部)
```

**c·(1/c)=1** 这个简单的代数恒等式，在复平面上表现为 ℤ₂ 对合——原点 ↔ 无穷对换，内凹 ↔ 外凸对换。正M 的心形在 `|c|>2` 处逃逸到无穷，逆M 的水滴在 `|c|<1/2` 处（反演回来的大心形内点）逃逸。

## 五、完整渲染流程伪代码

```
function renderInverseM_droplet(width, height, center_re, center_im, zoom):
    alpha = -1                    // 固定为逆M模式 (s=1)
    R2 = 2500                     // 逃逸半径²
    maxIter = (quality > 300) ? 90 : 46
    
    for each pixel (px, py):
        // 1. 坐标映射（90° CCW + Re反转）
        cyv = center_im + (px/width - 0.5) * zoom     // Im = 横轴
        cxv = center_re - (py/height - 0.5) * zoom    // Re = 纵轴, 上大下小
        
        // 2. 幂变换: c → c^alpha (alpha=-1 → 1/c)
        r = hypot(cxv, cyv)
        if r < 1e-13: c_eff = (1e6, 0)     // 原点→无穷
        else:
            rp = pow(r, -1)                // |c|^(-1) = 1/|c|
            aa = -atan2(cyv, cxv)          // -arg(c)
            c_eff = (rp*cos(aa), rp*sin(aa))  // = 1/c
        
        // 3. Mandelbrot 迭代
        zx=0; zy=0; trap=∞
        for i=0 to maxIter:
            (zx, zy) = (zx²-zy²+er, 2*zx*zy+ei)
            m2 = zx²+zy²
            trap = min(trap, m2)
            if m2 > R2: break
        
        // 4. 双模式着色
        if i >= maxIter:
            v = (log(trap) * 0.55) mod 1    // 内部同心圆
        else:
            nu = log2(log(m2)/2)
            v = (i + 1 - nu) * 0.132 mod 1  // 外部平滑彩虹
        
        pixel = LUT[v]   // 颜色查找表
        
    return image
```

## 六、为什么是"水滴"不是其他形状

这跟传统心形的区别来自两个独立因素：

1. **复反演**：`c → 1/c` 把圆映射为直线、把内凹映射为凸起
2. **旋转 90°**：如果不用旋转，逆M是"向右开口的翻转心形"。加上旋转后变成了"朝上的水滴"

两者缺一不可——只做反演不旋转 = 倒心形；只旋转不做反演 = 侧躺心形。合起来才是水滴。

## 七、关键参数速查

| 参数 | 值 | 作用 |
|------|:---:|------|
| `alpha` | -1 (s=1时) | 幂指数，决定正逆M程度 |
| `R²` | 2500 | 逃逸半径²，影响过渡带宽窄 |

| `trap` 系数 | 0.55 | 内部同心圆密度 |
| 外部 v 系数 | 0.055×2.4 | 外环色带密度 |
| 坐标旋转 | 90° CCW + Re反转 | 水滴尖朝上的空间朝向 |

> **总结**：逆M水滴 = 标准M集迭代器 + `c^alpha` 幂变换（合并了1/c反演）+ 90°旋转坐标映射 + 双模式着色。整套算法不超过30行核心代码，但产生了迥异于标准集的视觉效果。根源在于复反演的几何对偶：**c·(1/c)=1** 把分形的心形拓扑映射为水滴拓扑。
