# 健康密码 · 全DNA参考序列 · 音乐探索器

与「DNA·正逆M集·音乐探索器」同引擎，但输入换成**真实的人类参考基因组序列（GRCh38，健康野生型）**，直接读真实碱基→密码子→**真实氨基酸翻译**→音乐。

## 启动
双击 `启动_健康密码DNA音乐探索器.bat`（默认浏览器打开，纯离线）。

---

## ⚠️ 一个重要的科学澄清

老师最初的设想是"健康人 DNA = 去掉所有疾病片段的全序列"。这里必须说明：

1. **`基因片段数据库_1028_最终版.xlsx` 是疾病相关基因库** —— 但这些基因（BRCA1/TP53/CFTR…）本身是**每个人都有的正常基因**，致病的是它们的**突变/变异**，不是基因本身。
2. **不存在"健康基因组 = 全序列减去疾病片段"** —— 所有人拥有同一套基因；健康与否取决于序列上的**变异**。真正的"健康版对照"是**参考基因组（reference genome）**，即野生型/共识序列（无致病变异）。
3. **完整人类基因组 = 31 亿碱基对**（chr1-22+X+Y），约 3.1 GB，**无法嵌入网页、也无法当几十亿音符播放**。

所以本软件用**真实参考基因组序列**在可行规模上呈现"健康密码"：

## 数据来源（真实、可溯源）

| 轨道 | 内容 | 来源 |
|---|---|---|
| **MT 线粒体全基因组** | rCRS 完整 16,569 bp（5,523 密码子） | NCBI `NC_012920.1` / Ensembl `MT:1..16569` |
| **chr1-22 / X / Y** | 各 1 个地标基因的真实 CDS（前 300 密码子） | Ensembl REST，GRCh38，canonical 转录本 CDS |

- 数据均来自 **GRCh38 参考基因组 + rCRS 线粒体参考序列**。
- **⚠️ 重要更正（原"健康野生型"说法过度声称）**：GRCh38 **不是某个健康个体的基因组**，而是由约 20 名匿名捐赠者拼接的**镶嵌式共识参考序列**（约 70% 来自单一捐赠者 RP11，混合非洲-欧洲血统）；它是"坐标系参考"，**不保证无致病变异**（参考序列在若干位点实际携带次要/风险等位基因）。详见同目录 `数据来源与普适性_诚实说明.md`。
- 线粒体 rCRS 来自**单个欧洲个体**（单倍群 H2a2a1），是唯一能完整嵌入的完整参考基因组，故设为默认轨道。
- 核染色体因全长不可能嵌入，用**代表性地标基因 CDS** 作为该染色体的"乐章"。
- rCRS 第 3107 位有个历史占位符 `N`（原始 CRS 遗留），程序自动跳过该密码子。

## 与疾病基因版的区别

| | DNA·正逆M集(疾病库版) | 健康密码(本版) |
|---|---|---|
| 输入 | 基因**名称**(文本) | 真实 DNA **碱基序列** |
| 编码 | 名字位编码→合成碱基 | 直接读真实 A/C/G/T |
| 氨基酸 | 名字派生(非生物学) | **真实遗传密码翻译** |
| 选择 | 1028 个疾病相关基因名 | 25 条染色体/基因轨道 |
| 序列性质 | 病症收集 | 健康野生型参考 |

## 操作
- **轨道下拉**：选染色体/基因（默认线粒体全基因组），选中即演奏
- **起始密码子滑块**：长序列(如线粒体 5523 密码子)分窗浏览，每窗 64 音符
- **◀ ▶ 🎲**：上一条/下一条/随机轨道
- **点击分形图**：听任意点轨道声音
- **s 滑块**：☀正M ↔ 逆M☽（阴阳八度）
- 底部序列条显示真实密码子 + **真实氨基酸(蛋白序列)**

## 音乐学原理
同引擎：密码子→Farey p/q 泡→音高 `f_max/q`；氨基酸极性→音色（疏水/亲水/正电/负电）。详见 `原理1_分形轨道降频发声算法.md` 与 `原理2_底层原理全解.md`。
