## 1. 核心路线：自建频率库 + Agent数据分析

绕过商业黑箱，把NLS变成可控研究工具。硬件已有（1.4-4.9 GHz射频扫描仪），需要:
- 自建器官频率库
- Agent架构做数据处理

## 2. Agent架构设计（LangGraph）

```
用户输入 → Orchestrator Agent
  ├─ Tool 1: 硬件控制(PyVISA扫S11)
  ├─ Tool 2: 预处理(滤波/去噪/归一化)
  ├─ Tool 3: 特征提取(tsfresh)
  ├─ Tool 4: 推理(训练模型→风险概率)
  └─ Tool 5: 报告生成(matplotlib/plotly)
```

## 3. NLS vs MRI 正确定位

- ❌ "NLS超越MRI"是营销话术
- MRI看解剖+软组织对比，NLS/MWI看介电/水含量差异
- 分辨率差1-2个数量级
- ✅ 正确定位：**无辐射快速筛查/监测**，对标MammoWave

## 4. MammoWave参考

- Spec 50%灵敏度：筛查定位(非诊断)，宁可false positive不漏
- design为"complement to mammography"
- EU资助链: H2020 SME(原型验证) → FET(前沿技术) → Horizon CANCER(万人RCT)
- 写related work时cite这条链，reviewer认

## 5. Tiberi重建哲学（关键）

- **不用diffusion、不用U-Net、不用端到端DL反演**
- 用HP(Huygens Principle)做重构——不解逆散射不矩阵求逆
- 快+稳定+可解释
- ML只挂在特征分类(PCA/SVM)，不挂重构
- 每步物理可审→EU regulator喜欢

## 6. 对我们NLS系统的启发

- 我们目前用的是触发传感器(宽带噪声)+反相谐波算法——本质上走的是"经验疗法"路线
- 如果要升级到"可解释+可论文发表"路线:
  1. 保留手环硬件(FD232R)
  2. 建立35频段的器官频率对应库(已有部分B9_MAP)
  3. 用ML/统计替代目前的固定算法: 根据五轮AB测试选最优
  4. 报告生成加入频谱对比图(脊线图已有框架)
