# 泛幻方 (Pandiagonal Magic Square) × 2.5D Torus Chiplet 拓扑

> 来源：MathWorld/Wikipedia Pandiagonal + 元宝对话
> 核心：泛幻方的"折断对角线和守恒"恰好是 Torus 拓扑的负载约束
> 日期：2026-07-19

---

## 一、什么是泛幻方

**泛幻方 (Pandiagonal / Panmagic / Diabolic Square)**：不仅行、列、主对角线和等于幻和，**所有折断对角线（wrapping diagonals）也都等于幻和**。

```
普通幻方:              泛幻方:
  行 ✓                   行 ✓
  列 ✓                   列 ✓
  主对角线 ✓              主对角线 ✓
  折断对角线 ✗            折断对角线 ✓  ← 关键差异
```

"折断对角线"就是 Torus 上的对角线——走出网格边缘后 wrap 到对面继续走。

**最小阶数**：3阶不存在非平凡泛幻方。4阶存在48个。5阶存在3600个。

---

## 二、Torus 拓扑的天然对偶

Torus NoC = mesh + 边缘 wrap-around。Torus 上的"行"和"列"是普通的，但"对角线"在 torus 上是 wrap 的——恰好对应泛幻方的折断对角线。

```
Mesh上的对角线:         Torus上的对角线(wrap):
  \                        \       /
   \                        \     /
    \         vs             \   /
     \                        \ /
      \                        X
       \                      / \
                              /   \
```

**Torus 上 wrap 对角线的条数 = 2n**（每个方向 n 条）。泛幻方保证了所有 2n+2 个方向（行×n + 列×n + 主对角×2 + 副对角×2 + 折断×2n）的负载和都等于幻和。

普通幻方只保证 2n+2 个方向。泛幻方保证了 **4n+2** 个方向。

## 三、5阶泛幻方示例

```
20   8  21  14   2
11   4  17  10  23
 7  25  13   1  19
 3  16   9  22  15
24  12   5  18   6
    幻和 = 65

验证折断对角线:
  20+4+13+22+6 = 65  ✓ (左上→右下 wrap)
  14+23+7+16+5 = 65  ✓
  2+11+25+9+18 = 65  ✓
  ...
  8+14+19+15+9 = 65  ✓ (右上→左下 wrap)
```

**五重对称 (Quincunx)**：泛幻方还在中心格±相邻的 quincunx 模式上和为幻和：

```
中心 13 的 quincunx 之一:
  17+25+13+1+9 = 65
中心 13 的 quincunx 之二:
  21+7+13+19+5 = 65
...
```

总共 100 个 quincunx 模式满足幻和——远超普通幻方的 2n+2 个约束。这在 Torus 拓扑中意味着**局部热点区的所有 quincunx 模式也自动均衡**。

## 四、为什么 Torus 用泛幻方比 Mesh 用普通幻方好？

| | Mesh + 普通幻方 | Torus + 泛幻方 |
|:---|:---|:---|
| 负载约束方向 | 2n+2 | **4n+2** |
| 边缘tile的负载 | 边缘比中心"自由"（对角线不约束） | 边缘和中心一样受约束 |
| Chiplet 2.5D 适配 | 差（chiplet无边缘概念） | 完美（chiplet torus天然wrap） |
| 行/列搬移后保持幻性 | ✗ | ✓（任意行/列平移后仍是泛幻方） |
| 旋转后保持幻性 | ✓ | ✓ |
| 局部热点 quincunx | 无 | ✓（100个 quincunx和守恒） |

**关键**：Chiplet 的 2.5D 互连是环形 torus——物理上就有 wrap-around。用普通幻方的话，chiplet 间的"折断"路径不受保护，负载会堆积在 chiplet 边界。泛幻方恰好解决了这个边界效应。

## 五、泛幻方的 Siamese 扩展参数

Siamese 方法在 p̄ = p+1, q̄ = q-1 时（且 n 为奇数），**自动产生泛幻方**。

```
标准 Siamese（产生普通幻方）：
  p = -1, q = +1, p̄ = +1, q̄ = 0

泛幻方 Siamese（产生泛幻方）：
  p = -1, q = +2, p̄ = +2, q̄ = -1    (n=5时)
```

**约束**：对于奇数 n，存在参数使得 Siamese 产生泛幻方当且仅当 n 不是 3 的倍数。所以 5阶、7阶（7不是3的倍数）都可以，但 3阶和 9阶不行。

**对芯片层级的影响**：
- L0 (3阶): 不存在泛幻方 → 只能用普通幻方。刚好 L0 的 tile 数少，mesh 够用。
- L1 (5阶): 存在泛幻方 → **建议用 torus + 泛幻方**
- L2 (7阶): 存在泛幻方 → **建议用 torus + 泛幻方**
- L3 die 级: chiplet 天然 torus → 用泛幻方

## 六、4阶泛幻方：Most-Perfect 幻方

4阶泛幻方有个独特性质：**每个 2×2 子块的和也等于幻和的一半**。

```
 1  14   7  12
 8  11   2  13      每2×2子块和 = 34 (幻和)
10   5  16   3
15   4   9   6
     每2×2子块和 = 34 ✓
```

**384种不同的 4阶泛幻方** = 16(平移) × 24(参数排列)。

对于 8阶双偶数的 L3 die 级，可以用 4阶泛幻方作基块，Medjig 扩充到 8阶。这保证了 die 级的任意 2×2 tile 子群也负载均衡。

## 七、Torus 上的 quincunx 热均衡

泛幻方在 Torus 上的每个 tile 有 4 个 quincunx 模式（中心 + 四方向邻域）：

```
quincunx 类型1: 上/下/左/右邻 + 中心 = 幻和
quincunx 类型2: 对角线邻 + 中心 = 幻和
quincunx 类型3: wrap上/下/左/右 + 中心 = 幻和
quincunx 类型4: wrap对角线 + 中心 = 幻和
```

**物理含义**：Torus 上任意 tile 为中心的局部 patch（5个 tile），其总负载恒等于幻和。这意味着局部热点被严格约束——**一个 tile 过热必然意味着它的四个邻居更冷**，因为 quincunx 和是固定的。

这在热仿真中会直接体现为：温差<5K（因为热传导 + quincunx 负载守恒的双重约束）。
