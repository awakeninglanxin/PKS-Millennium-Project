# NLS xch患者特异性数据提取 — 方法论

**版本**: v1.0
**日期**: 2026-07-07
**核心方法**: 两人对比法 + 0-6范围过滤 + 多词典翻译
**成果**: 从93.8%标准库噪音中提取92个真实患者差异参数

---

## 一、问题背景

### 1.1 xch文件结构

xch (Meta Exchange 1.7) 是NLS系统的二进制诊断文件，包含200+器官的扫描参数。

**关键问题**: xch中93.8%的参数是**器官标准库(etalon)固定值**，在所有患者间完全相同；只有6.2%是**患者特异性值**。直接使用全部参数会被标准库值淹没。

### 1.2 v4-v8垃圾填充问题

xch文件中8个诊断维度(v1-v8)的v4-v8经常被二进制垃圾值填充：

```
v1 = 频谱相似度    ← 有效
v2 = 熵值          ← 有效
v3 = Fleindler上量表 ← 有效
v4 = Fleindler下量表 ← 常被垃圾填充 (-26495, -1654, -23683...)
v5 = 蓝线(N)       ← 常被垃圾填充
v6 = 红线(S)       ← 常被垃圾填充
v7 = 分离度        ← 常被垃圾填充
v8 = 综合偏离指数   ← 常被垃圾填充
```

## 二、技术工具链

### 2.1 核心解析器: xch_parver.py

**位置**: `D:\AAA我的文件\中健国康 NLS细胞检测\xch_parver.py`

**关键函数**: `_extract_values_near(offset, radius=400)`
- 在参数名后400字节窗口内扫描double和float值
- 问题: 窗口太大导致拾取二进制垃圾

### 2.2 过滤策略: batch_parse.py

**关键过滤**: `abs(v) < 50000`
- 清理大垃圾值 (-26495, -23683等)
- 可视化过滤更严格: `abs(v) < 100`

### 2.3 翻译词典

| 词典来源 | 条目数 | 特点 |
|----------|:---:|------|
| translations.py (NLS绿色破解版) | 2270 | 最全面 |
| add_cn_translation_v2.py | 2194 | 含SimpleChinese补充 |
| 手动补充 | 14 | 覆盖遗漏 |
| **合并总计** | **2649** | **100%覆盖** |

**翻译匹配策略**:
1. 精确匹配
2. 空格规范化匹配
3. 去尾标点匹配
4. SimpleChinese.text补充
5. 乱码检测
6. 未翻译标记

## 三、两人对比法

### 3.1 方法原理

```
蔡某xch  vs  丽梦儿xch
    ↓           ↓
  2809个共有参数对比
    ↓
93.8% (2635个) 完全相同 → 标准库固定值 (etalon)
 6.2% (174个) 有差异   → 患者特异性数据
    ↓
0-6范围过滤
    ↓
92个有效患者特异性差异参数
```

### 3.2 对比结果统计

| 统计项 | 数量 | 占比 |
|--------|:---:|:---:|
| 共有参数 | 2809 | 100% |
| 完全相同 | 2635 | 93.8% |
| 有差异 | 174 | 6.2% |
| 0-6有效值 | 92 | 3.3% |

### 3.3 值成对出现

患者特异性值以(N,S)对的形式出现：

```
示例: 小脑 (模块5300_l)
  蔡某:   [(1.801, 0.593), (1.773, 0.813), (1.828, 0.549), ...]
  丽梦儿: [(1.801, 0.593), (1.773, 0.75),  (1.812, 0.549), ...]
                          ↑差异              ↑差异
```

## 六、操作流程

### 6.1 提取流程

```
Step 1: 解析两人xch文件
  xch_parver.py → 参数列表 (含模块名、俄文名、值列表)

Step 2: 过滤垃圾值
  abs(v) < 50000 → 清理大垃圾
  0 <= v <= 6 → 有效频谱值范围

Step 3: 两人对比
  逐参数对比蔡某vs丽梦儿
  筛选有差异的参数 (174个)

Step 4: 翻译
  合并3个词典 (2649条)
  匹配策略: 精确→规范化→去标点→SimpleChinese→乱码→未翻译

Step 5: 输出
  JSON格式: 92个参数完整对照表
  含: 模块名、俄文、中文、蔡某值、丽梦儿值
```

## 七、注意事项

### 7.1 xch模块匹配陷阱

不同视图可能对应不同xch模块：
- 蔡某右肾前视图 → copy模块
- 蔡某右肾后视图 → 可能对应不同模块 (a=0.663异常)
- "肾脏"截图 → 可能对应肾盂/右肾/肾柱中的任一个

### 7.2 标准库值识别

```
标准库值特征:
  - 两患者完全相同
  - 多个值为0或-0.0
  - 成对出现 (1.581, 0.303) 等

患者特异性值特征:
  - 两患者有差异
  - 非零有效值 (0.1~2.0范围)
  - 差异通常在第4-5个位置
```

### 7.3 翻译匹配注意

- IMA软件Markdown解析器不支持`:--:`对齐语法
- 俄文器官名可能有空格变体 (ПРАВАЯ ПОЧКА vs ПРАВАЯ  ПОЧКА)
- SimpleChinese.text补充词典需要单独加载
