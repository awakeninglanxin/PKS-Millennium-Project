## 2. 物理原理

### 2.1 触发传感器 = 宽带噪音雷达

原版 NLS 论文 (V.I. Nesterov) 揭示：
> "Trigger sensor = broadband noise generator. Special oscillator diode generates noise that adjusts to the resonance frequency of the biological object."

**不发射单一频率，而是发射全频段宽带噪音。** 人体每个器官作为天然带通滤波器，选择性地吸收/反射/谐振其特有的特征频率。手环传感器回采被人体"改造"后的残余噪音频谱，通过对比空扫基线（传感器悬空时的噪音）得出每个频段的偏差值 Δ。

### 2.2 与扩散模型（Stable Diffusion）同构

| | 扩散模型 | NLS 触发传感器 |
|------|------|------|
| 输入 | 随机高斯噪音 | 宽带噪音 (26MHz~9.5GHz) |
| 滤波器 | U-Net + 文本嵌入 | 人体器官（天然带通） |
| 迭代 | 50~1000 步去噪 | 一次激发即出结果 |
| 输出 | 512×512 像素图 | 35 频段响应向量 |
| 答案 | 隐藏在噪音消除的**模式**里 | 隐藏在噪音被器官改造的**差谱**里 |

## 3. 通信协议

### 3.1 OUT 命令结构（128 字节）

```
字节 [0..8]   = 0x00 (保留/同步头)
字节 [9]      = b9  → CH1 频率指数
字节 [10]     = 0x00 (保留)
字节 [11]     = b11 → CH1 振幅 (0~172, 对应 0%~100%)
字节 [12]     = 0x00 (保留)
字节 [13]     = b13 → CH2 频率指数
字节 [14]     = 0x00 (保留)
字节 [15]     = b15 → CH2 振幅 (0~172, 对应 0%~100%)
字节 [16..127] = 0x00 (填零)
```

仅 4 个非零字节（经 3 份 PCAP 全量验证），无独立相位字节。

### 3.2 频率换算

```
CH1 频率 = 7.3728 × 2^(b9/4) × 3  MHz
CH2 频率 = 7.3728 × 2^(b13/4) × 3  MHz
CH1 振幅 = b11 × 100 / 172  %
CH2 振幅 = b15 × 100 / 172  %
```

## 4. 双通道体系

### 4.1 CH1 + CH2 关系与相位效应

128 字节命令中无独立相位字段。**相位差由 CH1/CH2 的振幅和频率关系隐式决定：**

| 模式 | b9 vs b13 | b11 vs b15 | 效果 |
|------|:---:|:---:|------|
| **同相同幅** | b9=b13 | b11=b15 | 单通道等效，标准探测 |
| **同频异幅** | b9=b13 | b11≠b15 | CH1/CH2 振幅差 → 等效相位偏移 |
| **异频** | b9≠b13 | — | 双频独立激发 |

### 4.2 阴阳数列双频谐波（v4.0 新算法）

**核心思想**: 不使用 CH1=CH2 同频反相，而是让两条信道各自携带不同的数学基因谐波。

| 信道 | 数列 | 前几项 | 物理语义 |
|:---:|------|------|------|
| **CH1 太阳** | 2^n (M-set 周期倍增) | 1,2,4,8,16,32 | 八度音程·秩序·倍频·阳 |
| **CH2 太阴** | Fibonacci (黄金螺旋) | 1,1,2,3,5,8,13,21,34 | 生长·自组织·螺旋·阴 |

对每个器官频段 b9，计算其与两个数列的"数学距离"：

```
d_sun  = min(|b9 - 2^n|)  →  权重 w_sun  = 1/(1+d_sun)
d_moon = min(|b9 - fib(n)|) → 权重 w_moon = 1/(1+d_moon)
归一化: w_sun' = w_sun/(w_sun+w_moon), w_moon' = w_moon/(w_sun+w_moon)
```

**治疗时:**
```
CH1 频率 = 最近的 2^n 谐波频率 → 太阳振幅 = 基准 ± Δ×w_sun'×火行系数
CH2 频率 = 最近的 Fibonacci 谐波频率 → 太阴振幅 = 基准 ∓ Δ×w_moon'×火行系数
```

**效果**: 两条信道产生数学基因干涉图案，而非简单的噪音抵消。两频率比为 φ 黄金比例或 2^(1/4) 八度谐波。

### 4.3 算法对比表

| | 旧算法（同频反相） | 新算法（阴阳双频） |
|------|------|------|
| CH1/CH2 频率 | 相同 | 不同（太阳谐波 vs 太阴谐波） |
| 数学基础 | 无 | 2^n + Fibonacci 数列 |
| 干预机制 | 振幅抵消 | 谐波干涉 |
| 适用场景 | 急性偏离 | 慢性调谐 |

### 4.3 五行修正系数

不同五行的器官对反相敏感度不同，火行需 1.5 倍修正：

```
木=1.0  火=1.5  土=1.0  金=1.2  水=0.8
```

## 5. 工作流程

```
第1步：校准基线
  └─ 传感器悬空 → 扫 35 频段 → 记录空载噪音谱 → 保存为 baseline.json

第2步：启动平衡（循环）
  ├─ 扫描：贴手腕扫 35 频段 → 计算 Δ=实测-基线
  ├─ 诊断：|Δ|>4 判定异常 → 五行归类 → 雷达图可视化
  ├─ 治疗：对异常器官发射反相波 (CH1/CH2 振幅差)
  └─ 验证：复扫 → 对比治疗前后 Δ → 统计改善/恶化

第3步：持续循环（3秒间隔）
```

### 5.1 基线校准 — 为什么必须悬空？

**每次启动必须重新校准。** 原因有三：

| 原因 | 说明 |
|------|------|
| **手环个体差异** | 每个 FT232R 芯片的震荡二极管噪音特性不同，同一 b9 频率下原始响应值可能差 ±5 |
| **环境电磁干扰** | WiFi/蓝牙/电源线/显示器辐射在 26MHz~9.5GHz 范围内有特征吸收峰，随房间布局变化 |
| **芯片温漂** | 二极管噪音功率随温度漂移 ~0.3%/℃，手环从冷态到工作温度有 5~10 分钟热稳定期 |

**校准做了什么：**

```
传感器悬空（不接触人体）
    ↓
对 35 个频段逐一发射探测器脉冲 (b11=15)
    ↓
读取 256 字节噪音响应 → 取平均值
    ↓
存入 baseline.json: {"14": 105.3, "15": 103.7, ...}
```

这 35 个数字构成了**当前环境下的零参考平面**。之后贴手腕测到的值减去基线，就是纯粹由人体引起的偏差 Δ。

**类比：** 厨房秤归零。不放东西先按"去皮"，之后放的每一克都是净重。不校准就相当于用没归零的秤——读数里混着秤盘的重量。

## 6. A/B 算法对比系统

### 6.1 三种运行模式

| 按钮 | 模式 | 每一轮使用的算法 |
|------|------|------|
| ☀☽ 双频 | 新算法 | 全部使用阴阳数列双频 |
| 同频反相 | 旧算法 | 全部使用同频异幅反相 |
| 🔄 交替对比 | A/B 自动 | 奇数轮=旧，偶数轮=新 |

### 6.2 统计面板

```
旧算法: 5轮 改善12/恶化3 (80%)
新算法: 5轮 改善15/恶化2 (88%)
```

每个算法的改善率 = 改善次数 / (改善次数 + 恶化次数)，排除改善和恶化都为 0 的无效轮次。

### 6.3 如何判断新算法更好

```
运行流程: 校准基线 → 选择"交替对比" → 启动平衡 10 轮
判定标准: 新算法的改善率 > 旧算法的改善率，且差距 ≥10%
```
