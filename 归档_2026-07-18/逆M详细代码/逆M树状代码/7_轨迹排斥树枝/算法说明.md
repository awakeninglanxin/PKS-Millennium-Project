# UF7 轨迹排斥树枝 — 正M vs 逆M 对照

## 与其他算法的根本区别

UF1-UF6 都是对"逃逸结果"着色（单像素一个颜色值）。
UF7 直接可视化"迭代轨道本身"——每个采样点画一整条 $z_0 \to z_1 \to \dots \to z_n$ 轨迹。

## 对照表

| 对比项 | 正M (Standard) | 逆M (Inverted) |
|:---|:---|:---|
| 复数映射 | `c = pixel` | `c_eff = 1/c` |
| 迭代 | `z = z² + c` 追踪全部 z | `z = z² + c_eff` 追踪全部 z |
| 轨道保存 | 每步 z_n 存入轨迹数组 | **相同** |
| 逆映射 | 无 (轨道已在屏幕空间) | `w = 1/z` 逆映射回屏幕坐标 |
| 旋转 | 无需 | rot90(k=3) 尖朝上 |
| **采样方式** | 稀疏采样 (step=8) | **相同** |
| **渲染方式** | 每条轨道的 z_n 映射为屏幕像素 | **相同** |

### 为什么逆M需要逆映射

正M中轨道 $z_n$ 在 `c = pixel` 空间, 坐标直接对应像素位置。
逆M中轨道 $z_n$ 在 `c_eff = 1/c` 空间, 需要经 $w_n = 1/z_n$ 映射回屏幕坐标再绘制。

### 管线

```python
1. 每 STEP 像素采一个点
2. 对该点做完整 Mandelbrot 迭代
3. 保存每步 z_n
4. # 逆M特有: w = 1/z (逆映射回屏幕)
5. 映射 w.real → x 坐标, w.imag → y 坐标
6. 在图像上画点: 迭代早期深色, 晚期浅色
7. rot90(k=3) → 水滴尖朝上
```

### 排斥分支 (扩展方向)

在轨道拐点植入微扰种子, 递归迭代生成次级枝杈:

```python
if angle_change > threshold:
    branch = z + perturbation
    for k in range(branch_len):
        branch = branch² + c_eff
        draw(branch)
```

### 参数

| 参数 | 值 |
|:---|:---|
| STEP | 8 (采样步长) |
| MAXITER | 120 |
| bailout | 50 |
