# 逆M水滴 — 双渲染策略深度对照

> 两个代码库，同一条迭代公式，但着色策略截然不同  
> **策略A**：DNA_M集音乐探索器（浏览器 Canvas, 实时交互, c^alpha 滑动）  
> **策略B**：UF1b_壳线描边.py（Python+numpy, 离线精绘, XOR棋盘+边界检测）

---

## 零、共同核心：迭代公式完全一致

```python
# 两者共用同一数学引擎
c_eff = c^alpha          # α = -1 时 ≡ 1/c（复反演）
z_0 = 0
for i in range(maxIter):
    z = z² + c_eff
    if |z| > bailout: break
```

**区别不在迭代器，全在"拿到 z、e、pot、ang 之后怎么画"。**

## 一、体系架构对比

| 维度 | 策略A：DNA_M音乐探索器 | 策略B：UF1b_壳线描边 |
|------|------|------|
| **运行环境** | 浏览器 Canvas (JS) | Python + numpy + matplotlib |
| **渲染目标** | 实时交互 + 轨道音乐 | 离线高精度素描 |
| **α 值** | `1-2s`，s∈[0,1] **连续滑动** | **固定 α=-1** (硬编码 1/c) |
| **旋转** | 像素坐标映射（90° CCW） | `np.rot90(..., k=3)` |
| **坐标系** | Re=纵轴(上大下小), Im=横轴 | Re=横轴, Im=纵轴 → rot90后反转 |
| **原点处理** | `r<1e-13` → `er=1e6` | `r<1e-12` → `c_eff=1e6` |
| **bailout** | **2500**（大→宽过渡带） | **128**（小→早逃逸，棋盘格压缩进内部） |
| **maxIter** | 46~90（Canvas 性能折中） | 300（离线不限时） |

## 二、着色策略：两种完全不同的"画笔画法"

### 策略A — 连续势能平滑着色（DNA_M 探索器）

```javascript
// 内部点 → 轨道陷阱同心圆
v = (Math.log(trap + 1e-9) * 0.55) % 1;

// 外部点 → 平滑逃逸彩环 (continuous potential)
const m2 = zx*zx + zy*zy;
const nu = Math.log(Math.log(m2)/2/Math.LN2)/Math.LN2;
v = ((i + 1 - nu) * 0.055 * 2.4) % 1;

// v → LUT 颜色表
pixel = colorTable[v];
```

**特点**：颜色连续渐变，适合实时交互和视觉美观。读者能直接"看"到收敛区域和发散区域。

### 策略B — XOR 二值棋盘 + Sobel 边界描边（UF1b）

```python
# 外部角度量化
rz = floor(er * RAY_DIVISIONS)        # 64 道射线
re = (rz % 2 == 0)                     # 偶数道 = 白格

# 势能量化
pz = floor(pot / POT_STEP)            # /0.12 = 大量同心层
pe = (pz % 2 == 0)                     # 偶数层 = 白格

# XOR 棋盘
cb = (re == pe) & ext                  # 角度⊕势能 = 二维棋盘格

# Sobel 外壳描边
gx = sobel(pot, axis=1)               # 势能梯度
gy = sobel(pot, axis=0)
shell = (gradient > 98.5%分位) & ext   # 高梯度处 = 轮廓线
shell = binary_opening(shell)

# 解析逆心形描边
c_cardioid = 0.5*e^(iθ) - 0.25*e^(2iθ)
outline = -1/c_cardioid                # 逆M边界曲线的解析解
```

**特点**：纯黑白二值，无渐变。棋盘格被反演压缩进水滴内部成三角鳞片状。Sobel 提取的壳线恰好勾出水滴的"骨相"——哪些区域迭代数跳变、哪些区域是边界。解析心形描边叠加在最上层作为数学参考线。

## 三、为什么两种着色策略同时存在——各自的用途

| 需求 | 策略A（Canvas平滑） | 策略B（XOR+壳线） |
|------|:---:|:---:|
| 好看、可交互 | ✅ 主力 | ❌ |
| 数学结构清晰度 | ⚠️ 颜色连续→边界模糊 | ✅ 棋盘格→阴阳分明 |
| 检测分形边界 | ❌ | ✅ Sobel 梯度+解析曲线 |
| 轨道可视化 | ✅ 叠加轨道线 | ❌ |
| 音乐生成 | ✅ 轨道→音高 | ❌ |
| 离线高精度印刷 | ❌ | ✅ 300 DPI |
| 去环（消除同心 bullseye） | ✅ 连续着色天然无环 | ❌ 势能 floor 产生环（UF1系列靠角度边缘检测解决了） |

## 四、UF1b 特有的"壳线描边"——数学原理

### 4.1 为什么 Sobel 能抓到分形边界

逆M 集的 `pot`（连续势能）在**内部=0**（截断），在**外部≈246**（几乎恒定），但在边界处发生**突变**。Sobel 算子正是梯度检测器——`|∇pot|` 在突变处取极大值：

```
外部 pot≈246 ────[突变]──── 内部 pot=0
                     ↑
                Sobel 抓到的高梯度点
```

这就是为什么 `np.percentile(gm[ext], 98.5)` 能精确挑出边界——只有边界上的极少数像素有显著梯度。

### 4.2 generate_outline() — 解析逆心形

```python
def generate_outline(n_pts=6000):
    θ = linspace(eps, 2π-eps, n_pts)
    c = 0.5*exp(iθ) - 0.25*exp(2iθ)    # 正M主心形参数方程
    inv_c = 1/c                           # 复反演→逆M水滴边界
    return -inv_c.imag, inv_c.real        # 坐标轴交换→匹配rot90
```

这个参数方程 `c(t) = 0.5e^{it} - 0.25e^{2it}` 是正M主心形的**精确解析边界**。经过 `1/c` 反演后恰好是逆M水滴的边界。

UF1b 把这条曲线用 `ax.plot()` 叠在最上层，形成了"数学描边 + 数值壳线"的双重边界效果。

## 五、旋转机制差异——容易混淆的点

两者都旋转了 90°，但实现路径完全不同：

```
策略A (Canvas):
  像素遍历时直接交换坐标轴:
    cxv = reTop - py/H * reSpan     ← Re赋值给纵轴
    cyv = imLeft + px/W * imSpan    ← Im赋值给横轴
  → 图像在"出生时"就已经是水滴朝上了

策略B (numpy):
  标准坐标系 Re=横轴, Im=纵轴 渲染完成
  → np.rot90(img, k=3)  逆时针旋转270° = 顺时针90°
  → 图像在"出生后"被整体旋转
```

**结果等价**，但策略A更高效（不需要额外图像旋转步骤），策略B的代码更好理解（坐标变换与渲染分离）。

## 六、核心参数全集对照

| 参数 | 策略A | 策略B (UF1b) | 作用 |
|------|:---:|:---:|------|
| α | `1-2s` 连续 | `-1` 固定 | 正逆M连续滑动 vs 纯粹逆M |
| maxIter | 46~90 | 300 | Canvas性能 vs 离线无限制 |
| bailout | 2500 | 128 | 宽过渡带 vs 紧凑棋盘格 |
| RAY_DIV | — | 64 | 角度量化道数 |
| POT_STEP | — | 0.12 | 势能量化步长 |
| 着色模式 | LUT彩虹 | XOR二值 | 连续渐变 vs 硬二值 |
| 边界提取 | 无 | Sobel+解析心形 | — vs 双重"骨相"描边 |
| 旋转 | 坐标映射 | np.rot90(k=3) | 空间换时间 vs 代码清晰 |
| DPI | 屏幕分辨率 | 300 | 交互 vs 印刷 |

## 七、两种策略的"杂交"可能性

```
策略A的连续着色 × 策略B的壳线叠加
    = 音乐探索器里增加"分形骨骼线"图层
    = 实时听到轨道音的同时，看到 Sobel 勾出的边界

策略B的离线精度 × 策略A的α滑动
    = 批量生成 α=-0.5, -0.7, -0.9, -1.0 的全系列素描
    = 观察水滴如何从正M→圆→逆M连续形变
```

两者共享同一迭代引擎，着色策略可以任意组合——这正是 `UF` 体系 25 种着色变体的设计哲学：引擎不动，画笔随便换。

---

> **一句话总结**：Canvas版画的是"肉"（渐变着色让视觉舒适），Python版画的是"骨"（XOR棋盘+Sobel壳线暴露分形结构）。同一颗心脏（z²+1/c），两副不同的皮囊。
