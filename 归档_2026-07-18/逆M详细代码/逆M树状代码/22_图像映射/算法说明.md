# UF22 图像映射 (Image Mapping) — 正M vs 逆M

## 算法原理

将逃逸时间的归一化值作为纹理坐标, 从外部图像文件中查找对应像素颜色。不同于标准 colorbar (一维查色), 图像映射是**二维纹理采样**——可产生照片级真实感或艺术移植效果。

## 公式

$$u = \frac{\nu}{\nu_{\max}},\quad v = \frac{\arg(z)}{2\pi}$$

将 $(u, v)$ 映射到图像像素坐标 $(x, y)$, 取该像素的颜色作为输出。

也可仅用一维映射: $u = \nu/\nu_{\max}$, 在图像的渐变行上采样。

## 正M vs 逆M 对照

| 对比项 | 正M | 逆M |
|:---|:---|:---|
| 映射 | `c = pixel` | `c_eff = 1/c` |
| 势函数 | `ν = n − log₂(log₂∣z∣)` | **相同** |
| 纹理坐标 | `(u,v) = (ν/ν_max, arg(z)/2π)` | **相同** |
| 颜色来源 | 外部图像文件查色 | **相同** |
| **着色区域** | 集合**外部** (逃逸) | 水滴**内部** (逃逸 = `ext`) |
| **未着色区域** | 集合内部 (黑) | 水滴外部 (黑) |

## 视觉效果

- 图形纹理替换: 将分形结构"披上"图像纹理
- 可产生类似于自然景物(云、木纹、大理石)的质感
- 依赖输入图像的分辨率和色调

## 实现管线

```python
1. 计算逃逸势能 ν 和角度 arg(z)
2. 归一化: u = ν / ν_max, v = arg(z) / (2π)
3. 映射到图像像素: x = int(u * img_w), y = int(v * img_h)
4. RGB = source_image[y, x]
5. 未逃逸区域 → 黑色
```

## 参数

| 参数 | 说明 |
|:---|:---|
| 输入图像 | 任意 RGB 图像 (建议 W×H ≥ 256×256) |
| ν_max | 势能归一化上限 |
| bailout | 50 |
