## 一、问题陈述：芯片设计的"同步诅咒"

### 1.1 现代 AI 芯片的五个物理瓶颈

尺度法则（Scaling Law）推动 AI 芯片走向更大的计算密度，但物理约束正逼近极限。以下是五个最紧迫的问题：

| 瓶颈 | 物理根因 | 当前解决方案 | 方案代价 |
|------|----------|-------------|----------|
| **同步开关噪声（SSN）** | 数百万门同时翻转 → 地弹 > 200mV | 增加去耦电容、多引脚接地 | 面积 +15-25%、封装成本 +30% |
| **PDN 共振** | 周期性电流冲击激发供电网络 LC 谐振 | 多层 PCB、多相 VRM | 层数增加、热耗不减 |
| **互连串扰** | 相邻信号线同时跳变 | 线间距 > 3×W、屏蔽线、差分对 | 布线密度损失 40-60% |
| **时钟树功耗** | 全局时钟驱动数千负载的容性负载同步充放电 | H 树、mesh、缓冲器链 | 时钟功耗占芯片总功耗 20-40% |
| **IR 压降** | 局部区域瞬时电流需求超过供电网络能力 | 网格化 PDN、动态电压调节 | 面积、复杂度 |

**这五个问题的数学本质完全相同**：

> **周期性事件在时间轴上对齐 → 峰值叠加 → 系统必须按最坏情况设计。**

这是数字同步电路设计的"原罪"——时钟本身就是为了使一切同步而存在的，但同步本身创造了物理极限。

### 1.2 展频时钟（SSC）为什么不够

展频时钟（Spread Spectrum Clocking）是当前芯片工业降低 EMI 的主要手段，它在**频率域**将时钟能量分散到一段频带上。TI、Skyworks 的测量显示 SSC 可降低 EMI 峰值 10-17 dB。

但 SSC 有根本局限：

- **SSC 在频率域操作**：调制的是时钟频率，不是负载的翻转相位。所有负载仍然在**同一时间**翻转——SSN 和地弹不受影响。
- **SSC 是概率性的**：它降低 EMI 峰值是统计效果，不能**数学证明**两个事件永不重叠。
- **SSC 引入抖动**：频率调制在时间域表现为时钟抖动，时序设计必须为此留余量。

**本文提出的 Farey 时钟森林（Farey Clock Forest, FCF）是对 SSC 的垂直补充**：

```
SSC:     频率域 → EMI 峰值降低（已有）
FCF:     相位域 → SSN/IR-drop/串扰 同时降低（本文）
```

## 二、数学基础：Farey 序列的物理含义

### 2.1 Farey 序列定义

对任意正整数 n，n 阶 Farey 序列 F(n) 定义为：

```
F(n) = { a/b : 0 ≤ a ≤ b ≤ n, gcd(a,b) = 1 }
```

按升序排列。例如 F(6)：

```
0/1, 1/6, 1/5, 1/4, 1/3, 2/5, 1/2, 3/5, 2/3, 3/4, 4/5, 5/6, 1/1
```

|F(n)| ≈ 3n²/π²（当 n → ∞）。F(200) 约有 12,233 个分数。

### 2.2 关键性质：相邻分数的互质最小差

**性质 1（邻居间距）**：若 a/b 和 c/d 是 F(n) 中相邻分数，则：

```
|c/d - a/b| = 1/(b·d)
```

且 bc - ad = 1（互质条件）。

**性质 2（最大间距）**：F(n) 中任意相邻分数的最大间距 = 1/n（出现在 0/1 到 1/n 之间）。

### 2.3 映射到芯片时钟：Farey 相位的三个物理含义

将 Farey 分数 p/q 视作归一化时钟周期中的相位偏移 θ = 2π·p/q，则：

**物理含义 1：负载翻转永不重叠的数学保证**

设有 N 个负载，分别分配 Farey 分数 φ₁, φ₂, ..., φ_N。如果所有分母 ≤ B，则任意两个负载的最小相位间隔为：

```
Δφ_min ≥ 1/B²
```

对 B = 200（F(200) = 12,233 个可用分数），Δφ_min ≥ 1/40,000 = 25 ppm。在 2.5 GHz 时钟（T = 400 ps）下：

```
最小时间分离 = 400 ps / 40,000 = 10 fs
```

而单个标准单元的门延迟约为 5-15 ps。这意味着**相位分离细到比单个门的开关时间还小两个数量级**——从数字电路的角度看，翻转边沿几乎是连续分布的。

**物理含义 2：互质分母 = 谐波永不对齐**

两个频率 f₁ = f₀·(p₁/q₁) 和 f₂ = f₀·(p₂/q₂) 的谐波对齐周期为：

```
T_align = LCM(q₁, q₂) / f₀
```

由于 gcd(p₁,q₁) = gcd(p₂,q₂) = 1，且 Farey 序列确保 q₁ 和 q₂ 可以取到任意互质对，**在任意有限时间窗内，不存在所有谐波同时对齐的时刻**。这是组合数论的直接结论，无需实验验证。

**物理含义 3：Farey 分数天然对应时钟分频电路**

任何有理数频率 a/b 都可以用整数 PLL + 分频器实现：

```
PLL 倍频 × a → 除法器 ÷ b → 输出频率 f₀·(a/b)
```

这意味着 Farey 时钟森林不需要全新的电路拓扑——它可以用现有的 PLL/DLL 基础结构实现。

### 2.4 何时 Farey 比均匀分相更优

在 128 个负载的仿真中，Farey 分相与均匀分相效果几乎相同（峰值差异 < 0.4%）。但这一结论不能外推到芯片级场景。

**Farey 的互质优势随负载数量变化**：

| 负载数量 | 均匀分相的优势 | Farey 的额外优势 | 原因 |
|----------|:-------------:|:----------------:|------|
| < 100 | 完全够用 | 0 | 间距足够大 |
| 100-1000 | 够用 | 接近 0 | 12,233 > N，等间距取子集即可 |
| 1000-10,000 | 开始不足 | **开始体现** | 等间距趋近连续，无法保证互质 |
| 10,000+ | 趋近 Farey | **显著** | 等间距本质上退化为均匀分布，而 Farey 保持互质结构 |
| 100,000+（全芯片） | 完全不够 | **决定性** | 均匀分相的"相邻负载等间距"在百万级负载下意味着必然出现谐波对齐 |

**核心论点**：芯片级负载数是百万级的（每个标准单元一个负载），而均匀分相在负载数 > 可用相位精度时必然产生重合。Farey 的互质性质确保即使重合不可避免，重合的周期（LCM(q_i,q_j)）也极长，在实际的电源/热时间尺度上相当于永不对齐。

## 三、Farey 时钟森林：从数学到硅片

### 3.1 传统时钟树 vs Farey 时钟森林

```
传统 H 树 CTS（Clock Tree Synthesis）：
  目标： skew → 0（所有叶子同时翻转）
  代价： 峰值电流 = N × I_single
        PDN 必须按 N×I_single 设计
        时钟树功耗 ≈ N·C_load·V²·f

Farey 时钟森林（Farey Clock Forest, FCF）：
  目标： skew(i,j) = |a_i/b_i - a_j/b_j| = 1/(b_i·b_j)  （互质偏移）
  代价： 峰值电流 ≈ I_single × (负载数/Farey密度)
        PDN 按平均电流设计即可
        时钟树功耗不变，但缓冲器可缩小 15-30%
  收益： 片上电容需求 ↓ 5-10×
        地弹振幅 ↓ 5-10×
        IR 压降 ↓ 3-5×
        时钟缓冲器面积 ↓ 15-30%
```

### 3.2 Farey 时钟森林的架构

```
                    主时钟 CLK (f₀)
                         |
            ┌────────────┼────────────┐
            |            |            |
         PLL₁          PLL₂    ...  PLL_k
      (f₀·a₁/b₁)  (f₀·a₂/b₂)    (f₀·a_k/b_k)
            |            |            |
       子树₁ (H树)   子树₂ (H树)  子树_k (H树)
      ┌───┼───┐    ┌───┼───┐    ┌───┼───┐
      叶子...叶子  叶子...叶子  叶子...叶子

每组叶子内部：传统零偏斜（同相同步）
组间：Farey 分数相位偏移（互质去同步）
```

**关键设计原则**：
1. **子树内部保持零偏斜**——简化局部时序收敛，子树内的负载数量控制在 100-500（均匀分相已足够）
2. **组间由 PLL 阵列注入 Farey 偏移**——每个 PLL 输出频率 f₀·(a_i/b_i)，其中 a_i/b_i 取 Farey 分数
3. **PLL 数量 k = min(N_loads, |F(B)|)**——对于百万级负载，k ≈ 10⁴-10⁵ 个 PLL 组，每个组内 10-100 个负载

### 3.3 片上实现要素

| 组件 | 功能 | 现有技术基础 | Farey 独有要求 |
|------|------|-------------|---------------|
| PLL 阵列 | 生成 k 个 Farey 频率 | 已有（多相 PLL） | 频率对必须取 Farey 分数 |
| DLL 微调 | 补偿 PVT 漂移 | 已有 | 互质约束下的相位校准 |
| 相位检测器 | 监测实际相位偏移 | 已有 | 检测 FCF 特有的谐波对齐事件 |
| 动态重配置 | 运行时调整 Farey 参数 | 已有（DVFS 框架） | 动态更新 Farey 分母集合 |

### 3.4 PVT 漂移对 Farey 保证的影响

工艺-电压-温度（PVT）漂移是 Farey 时钟森林面临的最大物理挑战。互质相位的数学保证建立在精确频率比的基础上——如果 PLL 输出的实际频率偏差超过相位间隔，互质性质将被破坏。

**量化分析**：

Farey 最小相位间隔 Δφ_min = 1/B²，对应时间间隔 Δt = T/B²。

现代 PLL 的抖动典型值：周期抖动（period jitter）= 1-5 ps RMS。

对 B = 200，T = 400 ps（2.5GHz）：
```
Δt = 400 ps / 40,000 = 10 fs（远小于 1 ps 的 PLL 抖动！）
```

**这直接暴露了一个硬约束**：B = 200 时要求的 10 fs 精度在当前硅工艺上不可实现。这意味着**需要校准电路**来自动补偿漂移。

**实用方案：片上相位校准环**

```
Farey 目标相位: φ_target(i) = 2π·a_i/b_i
实际相位:       φ_actual(i) = φ_target(i) + ε_i(t)  [ε ≈ 1-5 ps]

校准策略：
  1. 片上相位检测器（PD）持续测量相邻 PLL 的相位差
  2. 若 |ε_i - ε_j| > Δφ_min/2 → 触发 DLL 微调
  3. DLL 调整范围 ±50 ps，精度 < 1 ps
```

B = 200 的 10 fs 要求过于严苛。**实用 B 值**应在 30-50 范围内：

| B | |F(B)| | Δφ_min | Δt @ 2.5GHz | PLL 抖动是否覆盖 |
|---|------|--------|------------|--------------|
| 30 | ~278 | 1/900 | 444 fs | ❌ 不覆盖 |
| 50 | ~774 | 1/2500 | 160 fs | ❌ 不覆盖 |
| 100 | ~3045 | 1/10000 | 40 fs | ❌ 不覆盖 |
| 200 | ~12233 | 1/40000 | 10 fs | ❌ 严重不覆盖 |

**修正后的设计参数**：考虑到 PVT，实际方案应采用多级 Farey：

```
第 1 级：B₁ = 10（100 个相位组，Δφ_min = 1/100 = 4 ps @ 2.5GHz ✅ 可保证）
第 2 级：在每个相位组内部，用 B₂ = 30 做次级 Farey 分相
第 3 级：叶子级，每个次级组内 50-100 个负载，传统零偏斜或均匀分相
```

这种分层架构将 PVT 敏感度隔离在第 1 级的粗粒度相位（PLL 级），第 2-3 级在本地时钟域内用 DLL 精细分相。

## 四、E8 × Farey 统一框架：物理分离 × 时间分离

### 4.1 E8 晶格的基础

E8 是 8 维空间中最密的球堆积。它的根系统由 240 个 8 维向量构成，满足：

```
晶格点之间的最小距离 d_min = √2
堆积密度 = π⁴/384 ≈ 0.254
```

E8 解决的核心问题：**在 8 维空间中，如何放置尽可能多的球，使它们两两不重叠。**

### 4.2 两个"分离"问题的同构

| 维度 | E8 解决的问题 | Farey 解决的问题 |
|------|-------------|-----------------|
| 空间 | 8 维欧氏空间 | 1 维时间轴 |
| 元素 | 球（晶格点） | 翻转边沿（相位点） |
| 约束 | 球不重叠（d ≥ √2） | 边沿不对齐（Δφ ≥ 1/B²） |
| 最优性度量 | 堆积密度最大化 | 互质保证最大化 |
| 数学基础 | 李群 E8 的根系 | Farey 序列的数论性质 |

**关键洞察**：E8 和 Farey 是同一个问题在空间域和时间域的镜像。芯片的自动布线如果要彻底解决串扰 + SSN 的联合问题，必须**同时**优化空间分离（E8 约束线间距）和时间分离（Farey 约束翻转偏移）。

### 4.3 统一框架：Farey-E8 联合布线算法

```
输入：
  - 网表（N 条互连线，M 个标准单元）
  - 芯片面积约束
  - 时序约束（目标频率 f₀）

输出：
  - 物理布线（3D 坐标）→ E8 编码的线间距约束
  - 相位分配（Farey 分数）→ 每条线的翻转偏移
  - 时钟树拓扑 → Farey 时钟森林

约束：
  (1) E8 约束：对每条线 l_i，其路径上的间距 ≥ d_min(l_i, l_j)
      当 l_i 和 l_j 在同一时间翻转时，d_min 取最大约束
  (2) Farey 约束：对每对信号 (l_i, l_j)，若物理间距 < d_threshold，
      则必须分配互质分母，即 q_i ⊥ q_j
  (3) 时序约束：每条路径的 setup/hold 满足 Farey 偏斜下的 slack ≥ 0

目标函数：
  minimize: α·(峰值电流) + β·(互连线总长) + γ·(时钟树面积)
  subject to: (1), (2), (3)
```

### 4.4 E8 在 3D 布线中的实际映射

虽然 E8 是 8 维结构，但在芯片的 3D 布线（x, y, metal layer）中，可以取其 3 维投影或截面。核心思想是：

> **E8 晶格的 3D 投影保持了一定程度的最密堆积性质，可以在给定的金属层约束下最大化线间距。**

具体做法：

1. 将芯片的布线空间量化为 3D 晶格
2. 在 E8 晶格中搜索与目标间距最接近的截面
3. 用该截面的晶格向量作为自动布线的方向约束
4. 配合 Farey 时钟森林：物理上靠近的线在时间上互质分离

## 五、应用场景详解

### 场景 1：HBM 宽位总线

**当前状态**：HBM3 每通道 1024 位。所有位同时翻转 → 同步开关输出（SSO）地弹 > 200mV。

**Farey 改进**：1024 根数据线分为 16 组，每组 64 根。组间由 Farey 分数分配发射延迟，组内保持同步。

```
Farey 分组：B = 50，取 16 个互质分母 Farey 分数
每组延时：τ_i = T·p_i/q_i （分布在 0.2T - 0.8T 之间）

等效 SSO 数量：1024 → ~100（去同步因子 ≈ 0.1）
PDN 目标阻抗可放松 5-10×
```

**需要增加的电路**：每组一个可编程延迟线（DLL），延迟控制精度 < 10 ps。

**对带宽的影响**：每笔数据的绝对延迟增加 ≤ 1 个 UI，但在流水线中该延迟被隐藏——有效带宽不变。

### 场景 2：AI 加速器脉动阵列

**目标**：TPUv4 的 MXU 有 128×128 = 16,384 个 MAC 单元。

**Farey 改进**：16,384 个 MAC 按 Farey 分数分到 512 个相位组（B=80，|F(80)| ≈ 1,945，取 512 个等间距子集），每组 32 个 MAC 同步。

**收益**：
- 峰值 di/dt 降低约 512/16384 ≈ 1/32
- 片上电容需求可减少 50-80%
- 省下的面积可分配给更多 MAC 或更大 SRAM

**与脉动阵列的天然适配**：脉动阵列的数据本就按"波浪式"流过阵列，每个 MAC 的计算天然有相位偏移。Farey 非均匀偏移不是破坏流水线，而是让偏移从"数据决定的"变为"数论决定的"，在统计上更均匀。

### 场景 3：3D 堆叠芯片的跨层相位管理

**背景**：3D IC（如 HBM + Logic die）通过 TSV（硅通孔）连接各层。当堆叠层数超过 4-8 层时，跨层同步翻转引起的地弹/串扰放大。

**Farey 改进**：不同层的时钟分配不同的分母 B_k——底层用大的 B（细粒度分相），顶层用小的 B（粗粒度）。

```
Layer 0 (Logic):  B=200, 12233 相位组
Layer 1 (Cache):  B=100, 3045 相位组
Layer 2 (HBM):    B=50,  774 相位组
Layer 3 (HBM):    B=30,  278 相位组
```

层间 Farey 分级 — 计算密集层需要最精密的分相，存储层可以粗一些。

### 场景 4：光子互连芯片的波分复用 × Farey 时分复用

**延伸畅想**：硅光子芯片用不同波长的光在多条波导中同时传输。如果不同波长的**调制器**也按 Farey 分数做相位偏移，可以在波长域（WDM）基础上叠加时间域的去同步。这是 Farey + 光子学的交叉。

波分复用（WDM）正交 + Farey 时分复用正交 → 两个正交维度的联合去串扰。对下一代光互连 AI 芯片（Lightmatter/Celestial AI 等公司的架构），这是理论上有趣的补充方向。

### 场景 5：NVIDIA Blackwell B200 双 Die 架构（⚠️ 2026-07-18 新增，基于元宝 AI 深度推演）

#### 5.1 B200 为何改写 Farey 规则

B200 引入了三个 RTX 4090 所没有的硬件变量，迫使 Farey 模型从"128 SM 一锅分相"迁移到多层级时钟域：

```
B200 = 2×GB100 die (CoWoS-L) × 每 die 74 active SM = 整卡 148 SM
     + NV-HBI 10 TB/s cross-die 链路（300+ cycle 延迟鸿沟）
     + 8×HBM3e PHY（独立 DFI 时钟域）
     + 2SM-pair 协作 MMA（tcgen05.mma.cta_group::2 指令级锁步）
```

**根本修正：负载模型从"1SM=1 相位槽"变为"1TPC=1 相位槽"**

Blackwell 的 `tcgen05.mma.cta_group::2` 指令使 pair 内两 SM 的 Tensor Core 由同一条指令触发——**指令级锁步**，非调度近似同步。因此：

```
修正前：148 SM → 148 个独立相位槽
修正后：74 TPC（Texture Processing Cluster，每 TPC=2 SM）→ 74 个相位槽
       pair 内 2 SM 零偏（硬件已保证锁步）
       pair 间做 Farey 偏移
```

每 die 74 active SM → 37 TPC/die → 整卡 74 TPC。取 F(12)=49 里的 37 个互质对，挂到 TPC 级 PLL 分频。这是 B200 硬件拓扑给 Farey 的天然对齐——分组粒度跟硬件协作单元正好吻合，无需软件强行配对。

#### 5.2 B200 三层 Farey 分相

| 层级 | 硬件实体 | 数量 | Farey 参数 | 说明 |
|------|---------|------|-----------|------|
| L1: TPC 级 | TPC（含 2SM-pair） | **74**（37/die） | B₁=12, \|F(12)\|=49 | pair 间互质偏移，pair 内零偏 |
| L2: NV-HBI 级 | cross-die coherence 链路 | 1/die-pair | 两 die 的 L2 credit 返回节拍偏移 | ⚠️ 非 PLL 频率调整，是 credit 节拍 |
| L3: HBM PHY 级 | HBM3e 内存接口 | 8 PHY | B₃=8 | DFI 时钟域去同步 |

**L2 层的关键修正**（原白皮书写"Die A/B PLL 取 Farey 对"是过度简化的）：

NV-HBI 是 coherence 链路，CDC FIFO + credit 机制会平均掉 SM 级翻转边沿。Farey 在此层的真实形态是：**调两 die 的 L2 credit 返回节拍偏移**（而非 PLL 频率），使用 SemiAnalysis 微测的 300+ cycle 作为标定标尺（非调节量）。

标定实验：
```
C_eff(HBI) ≈ I_peak · τ_burst / ΔV_droop
→ 用 forced cross-die L2 burst（MoE token routing all-to-all 模式）
→ 板级 Rogowski 线圈测 die_A vs die_B 电流波形相位差
→ NCU NV-HBI 事件计数器辅助验证
```

#### 5.3 B200 时钟频率对 PVT 分析的影响

B200 SXM 实际运行频率约 **1.8-2.0 GHz**（非本文举例的 2.5GHz 消费级频率）。这意味着：

| 影响 | 量值 | 方向 |
|------|------|:--:|
| Δt（最小时间分离）| 比 2.5GHz 时大 25-40% | ✅ PVT 压力缓解 |
| B=50 的 Δt 从 160fs → ~220fs | 仍远小于 PLL 1ps 抖动 | 🔴 仍不覆盖 |
| B=10 的 Δt 从 4ps → ~5.5ps | 与 PLL 抖动同量级 | 🟡 边界 |

**结论不变**：分层 Farey（B₁=10/30/50）仍是实用方案，B200 的较低频率反而轻微缓解了 PVT 矛盾。

#### 5.4 峰值形状变化：B200 为什么跟 4090 不同

```
4090：128 SM 各自由 warp scheduler 独立发射 → 翻转边沿准随机分布 → 均匀分相几乎抹平峰
B200：74 pair × 每 pair 两 SM 锁步点火 → 每 pair 电流尖峰 ~2× 集中度 → 均匀分相开始漏拍
```

**量化**：Farey 相对均匀的优势在 B200 上从 4090 的 <0.4% 拉大到估 **1-3%**（仍是小数字，但方向是"100-1000 档 Farey≈均匀"边界的首次松动）。

#### 5.5 Rack 级杠杆：GB200 NVL72 才是 Farey 互质优势的真正战场

单 B200 卡 148 SM 本身远未达到"100,000+ 负载"阈值。但 GB200 NVL72 整柜场景：

```
72 颗 B200 = 144 die
+ 72×NVLink 5.0 (1.8 TB/s)
+ 8×HBM3e×72 = 576 HBM PHY
= 电源域负载数 10⁵+
```

Rack 级三层 Farey：

| 层 | 对象 | 数量 | Farey 级 |
|----|------|------|---------|
| L1 | TPC | 74×72=5328 | TPC 级（B=12） |
| L2 | NV-HBI + NVLink 5.0 | 144 die | die 间（B₂=12/card） |
| L3 | 整 rack 72 卡 | 72 | rack 级 F(72) 互质对 |

Rack 级能效比杠杆：
- di/dt 降 20-30% → PSU 效率曲线在高 η 区间运行
- 整 rack ~144-288 kW 下，省 **3-10 kW** 级
- **这是 Farey 互质性质超越均匀分相的首个可测物理场景**

**诚实判据**：单 B200 卡上 Farey 的 SM 级收益 <1-3%，P0 验证如果只测单卡可能回到 4090 的 <0.4% 噪声。必须用 **MoE all-to-all + cross-die 高占比 kernel（如 Mixtral-8x7B / DeepSeek-MoE）在 NVL72 整柜上测**，才能激活三层 Farey 的全部杠杆。

### 6.2 Farey-STA 的关键算法

传统 STA 假设同一时钟域内 skew 在 ps 级。Farey-STA 需要处理数百 ps 的相位偏移。

**核心思想**：Farey 偏斜不是随机噪声——它是确定的数论结构。因此，最差路径不是遍历所有 (i,j) 对（O(N²)），而是在 Farey 序列中找到分母使相邻间距最大的分数对——这恰好是 0/1 到 1/B，间距 1/B。

**关键松弛条件**：

```
setup:   T_data + T_setup ≤ T_cycle + T_skew
         → 松弛 = T_cycle + |φ_i - φ_j| - T_data - T_setup

hold:    T_data ≥ T_hold + T_skew
         → 松弛 = T_data - |φ_i - φ_j| - T_hold
```

Farey 偏斜直接增加了一侧的时序预算。如果一条数据路径本身就短，就可以安全地给它分配更大的 Farey 偏斜。这就是 **Farey 偏斜分配问题**——一个组合优化问题，但受 Farey 序列结构约束，有高效近似解法。

## 七、落地路径与阶段规划

### Phase 0：FPGA 原型验证（6 个月，1 人）

**目标**：在 Xilinx/AMD FPGA 上实现最小可行的 Farey 时钟森林原型。

**做法**：
- 利用 FPGA 已有的 MMCM/PLL 生成 8-16 个 Farey 频率
- Vivado `create_clock -waveform` + `set_clock_groups -asynchronous` 实现相位组隔离
- 用片内 ADC 监测各组的实际电流波形
- 验证：Farey 分相组的总峰值电流是否低于同步组

**成功标准**：峰值电流降低 ≥ 50%（此为系统级降低，不同于仿真的单脉冲叠加模型）。

**失败标准**：FPGA 的固定 PLL 资源不够生成所需频率比，或 PVT 漂移使相位保证完全失效。

### Phase 1：ASIC 测试芯片（18 个月，3-5 人团队）

**目标**：在 28nm/65nm 成熟工艺上流片验证。

**芯片规格**：
- 测试结构：4 组各 1024 个标准单元的时钟负载
  - 组 A：传统 H 树（基准）
  - 组 B：Farey 时钟森林 B=30
  - 组 C：Farey 时钟森林 B=50
  - 组 D：均匀分相
- 片上测量：集成电流传感器 + 温度传感器
- 外部测量：板级 Rogowski 线圈 + 高速示波器

**成功标准**：组 B/C 的峰值 di/dt 比组 A 降低 ≥ 80%；组 C 的时钟抖动导致的时序违例 < 1%。

### Phase 2：EDA 工具链原型（24 个月，与 Phase 1 并行）

**目标**：开发 Farey-CTS + Farey-STA 的学术原型。

**基准测试**：
- ISCAS-89 benchmark circuits
- OpenROAD flow 集成
- 与商业工具（Synopsys/Cadence）的结果对比

### Phase 3：AI 加速器参考设计（36 个月）

**目标**：将 Farey 时钟森林集成到完整的 AI 加速器 RTL 中。

**合作目标**：学术合作（MIT/Stanford/Berkeley 的 VLSI 组）或开源硬件社区（CHIPS Alliance）。

### Phase 4：商业化（48 个月+）

Farey EDA 工具的商业模式可能是：
- 作为 Synopsys/Cadence 的插件/功能模块
- 或被收购集成到他们的 CTS/STA 流程中

## 八、诚实评估：可行性矩阵

| 维度 | 评估 | 依据 |
|------|:----:|------|
| 数学正确性 | ✅ 确定 | 组合数论定理，非经验假设 |
| 128 SM GPU 场景优势 | ❌ 几乎没有 | 仿真：Farey ≈ 均匀（差异 < 0.4%） |
| 芯片级（>10⁴）场景优势 | 🟡 理论成立，未验证 | 互质性质随负载数增长的优势未被硅验证 |
| PVT 漂移对 Farey 的破坏 | 🔴 严重（B > 50时） | B=200 需 10fs 精度 → 当前 PLL 无法满足 |
| 分层 Farey 降级方案 | 🟢 可行 | B=10 级间距 4ps → 当前 PLL 可达 |
| EDA 工具链兼容性 | 🟡 中 | 需开发新算法，但可嵌入现有流程 |
| 流片验证成本 | 🔴 高 | 最小测试芯片 ~$100K-500K（28nm MPW） |
| 学术合作可行性 | 🟢 高 | 数论+EDA 交叉方向，有论文发表空间 |
| 商业可行性（短期） | 🔴 低 | 芯片行业极度保守，新时钟方案需 5-10 年 |
| FPGA 原型路径 | 🟢 可行 | 1 人 6 个月，成本低 |
| 与 PKS 项目的协同 | 🟡 间接 | 验证了千禧难题数学的工程价值，但不直接推动 PKS 主目标 |

**总体判断**：Farey 时钟森林是一个**长期（5-10 年）的芯片设计范式革新**，而非短期可落地的工程方案。短期最现实的路径是 FPGA 原型验证，证明原理后寻求学术合作。

## 附录：与 PKS 千禧难题项目的关系

### A.1 定位：千禧难题"反向输出"的第三条线

PKS 项目有三条从千禧难题数学到工程应用的路径：

```
PKS 双曲锥 xy=1
  ├── 线 1：NS 正则性（直接目标）─── 锥内涡量放大 → 证明或否定 blowup
  ├── 线 2：液态金属散热（应用输出）── 16bands 趋肤效应 → AI 芯片散热
  └── 线 3：Farey 时钟森林（数学输出）── 数论 → 芯片布线底层革新
```

Farey 时钟森林属于第三条线——它不是直接解决千禧难题，而是千禧难题研究过程中"溢出"的数论工具对另一个领域的革命性应用。这种"有用的意外"本身就是基础研究价值的体现。

### A.2 五大锚点的修正关系

截至 2026-07-17，五大核心锚点的落地价值排名（已修正）：

| 排名 | 锚点 | 评分 | 状态 |
|------|------|:----:|------|
| 🥇 1 | SPF 三分类筛除 | 8.0/10 | N=10^9 验证通过 |
| 🥈 2 | 液态金属 × PKS 漏斗 | 7.5/10 | LBM 涡量 10.3× 但 NaN |

### A.3 本文的诚实声明

本文讨论的 Farey 时钟森林在以下层面是**理论/仿真阶段**的：

| 已验证（仿真） | 未验证 |
|---------------|--------|

| | E8 × Farey 联合布线的实际物理收益 |
| | 任何硬件实测的 di/dt 降低数据 |
