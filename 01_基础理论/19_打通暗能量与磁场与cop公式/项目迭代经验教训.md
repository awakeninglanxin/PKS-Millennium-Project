## 二、关键Bug与修复

### Bug #1：fill_between 透明度假象（严重）

**现象**：图4 W_Hf填充矩形"从B≈1.25开始"，而非从B=0。

**根因**：`fill_between` 设置 `alpha=0.25`，粉色(#E684B6)叠加白底后合成色≈(249,224,237)，与白底(255,255,255)仅差2%，肉眼不可见。只有B-H曲线（也是品红）附近的叠加区看起来"有填充"。

**修复**：弃用 `fill_between`，改用 `matplotlib.patches.Rectangle` 显式矩形，alpha提升至0.48 + 边框线。

**教训**：
- Canvas渲染时，alpha<0.3在白色背景上几乎不可见
- 透明填充必须有边框辅助定位
- 先用PIL逐像素验证渲染结果

---

### Bug #2：轴与参数等比缩放——视觉抵消（严重）

**现象**：调B_S时图3/图4曲线完全不动；调H_X时H_S标注线跟随移动。

**根因**：
```javascript
// 错：坐标变换的分母和曲线数据使用同一个参数
const toX = (hv) => pad.l + (hv / hx) * pw;  // hx既是轴终点又是曲线参数
const toY = (bv) => pad.t + (1 - bv / bs) * ph;  // bs既是轴终点又是曲线数据
```
改H_X→x轴缩放→H_S标注位置漂移（看起来和改H_S一样）。改B_S→y轴缩放+曲线高度同步变化→曲线相对位置不变。

**修复**：改用固定参考轴
```javascript
const refHmax = Math.max(hx * 1.08, 1.5);  // 轴仅轻微跟随参数
const refBmax = Math.max(bs * 1.25, 2.5);
```

**后续优化**：y轴加固定偏移 `bs*1.8+5` 破坏等比性 → B_S变化可视。

**教训**：
- 数据值**和**轴缩放**使用同一参数**是数据可视化常见陷阱
- 确保至少有一个自由度——要么数据动轴不动，要么轴动数据不动
- +5固定偏移是破坏比例锁定的简单有效手段

---

### Bug #3：Canvas曲线绘制循环被误删（致命）

**现象**：图3反色后B-H曲线和μ曲线完全消失。

**根因**：替换颜色代码时，`ctx.beginPath()` 和 `for` 循环被整块替换，只剩 `ctx.strokeStyle` 设置语句，没有实际绘制。

**修复**：恢复完整绘制循环——500点路径 + beginPath/moveTo/lineTo/stroke。

**教训**：
- 大块代码替换前，先用 `grep -n` 确认替换范围内的所有关键语句
- Canvas绘制三要素缺一不可：strokeStyle + beginPath/路径 + stroke
- 养成替换后立即做diff验证的习惯

---

### Bug #4：gainMax被参考曲线锁定——白点永远钉在顶部

**现象**：图5右子图增益曲线的白点只在x轴移动，y不动。

**根因**：
```javascript
// gainMax = max(每条参考曲线的增益) = 9,999,999
gainMax = max(10000/0.001 - 1, 10000/0.01 - 1, ...) = 9,999,999
// 当前工作点增益 = 10/1.5 - 1 = 5.67
// y = 1 - 5.67/9,999,999 ≈ 1.0  → 钉在顶端
```

**修复**：gainMax改为以当前工作点为中心缩放
```javascript
const curGain = hx/hs - 1;
const gainMax = Math.max(curGain * 3 + 10, 15);
```

**教训**：
- 动态图表中，参考基准线/轴的缩放**必须**以当前数据范围为中心
- 不要让固定的参考曲线主导轴范围——工作点会被"压扁"
- `max(所有参考值)` 在数值跨度巨大时会导致小值区域完全不可见

---

### Bug #5：Slider双绑导致RAF中断

**现象**：侧栏参数拖动后图表不联动更新。

**根因**：`DOMContentLoaded` + `load` 两个事件都调用 `setupSliders()`，第一次绑定后第二次清除 `oninput` 属性但 `addEventListener` 已叠加。某些情况下第二次绑定时机不对导致事件丢失。RAF动画循环中5个draw函数任一报错→循环断裂→图表冻结。

**修复**：
- 去掉DOMContentLoaded重复绑定，只保留load→init单一链路
- 动画循环每个draw函数包裹 `try{}catch(e){}`，单图失败不影响整体
- `this.value` 改为显式 `s.value` 避免闭包作用域歧义

**教训**：
- `requestAnimationFrame` 循环中**必须**try-catch每个绘制步骤
- 双层事件绑定是隐性炸弹——一个`addEventListener`看不住
- 浮窗DOM移出grid布局后，`this`引用可能变化，用显式引用更安全

---

### Bug #6：μ轴固定范围溢出

**现象**：设 H_S=0.001 时 μ_linear = B_S/H_S = 1500，远超固定 μ_max=3.0 → μ曲线飞到画框外。

**修复**：`muMax = Math.max(muLinear * 1.3, 2)` 动态跟随。

**教训**：参数范围扩大后，**所有**派生变量的显示范围都必须同步扩展。

## 三、架构决策记录

### ADR-1：matplotlib → Canvas

**决策**：弃用Python matplotlib生成PNG，全部改用JavaScript Canvas实时渲染。

**理由**：
- 交互式滑条需要毫秒级响应，Python子进程启动延迟不可接受
- Canvas支持60fps动画（电路图T_on/T_off状态切换）
- 单HTML文件零依赖部署

**代价**：需要手写所有坐标映射、刻度标注、曲线采样。

---

### ADR-2：固定浮窗 vs Grid布局

**决策**：参数面板从CSS Grid双列改为`position:fixed`浮窗。

**理由**：
- 页面滚动时参数始终可见
- 折叠按钮释放屏幕空间给图表
- 与MWO电路仿真v8.2保持一致的交互模式

**代价**：需要手动管理`margin-left`偏移和`transition`动画，div配对复杂度增加。

---

### ADR-3：对数滑条 → 线性宽步长

**决策**：频率/H_S/H_X从对数滑条回退到线性宽步长。

**理由**：
- 用户反馈"非线性"体感差
- HTML range的视觉滑块位置与实际值脱节
- 宽范围线性滑条（如0.001~10, step=0.01）通过放大步长解决颗粒度问题

**教训**：对数滑条在纸上看起来很优雅，但对非技术用户的交互体验是灾难。

---

### ADR-4：模拟数据 vs 真实仿真引擎

**决策**：simulateFlyback()使用简化物理模型，不作完整SPICE仿真。

**理由**：
- 需要在浏览器60fps下运行
- 用户调整参数时需要即时反馈
- 完整SPICE仿真（Newton-Raphson迭代）单步>100ms不可接受

**代价**：波形精度有限，不能替代真正的LTspice验证。已明确标注"简化模型"。

## 四、编码铁律（本项目特定）

1. **Canvas颜色变更**：永远用 `grep` 先查全部引用，再逐行替换，最后验证 `ctx.beginPath()` 和 `for` 循环完整

2. **轴缩放公式**：`toX()` 和 `toY()` 的分母**不能**与曲线数据使用同一个参数

3. **动态图表参考范围**：`max()` 基准线必须以当前数据为中心，不用固定参考值

4. **RAF循环**：每个draw函数必须 `try{}catch(e){}`

5. **事件绑定**：只用一次`load`事件，不用`DOMContentLoaded`重复绑定

6. **Slider格式**：宽范围用线性+大step，不用对数滑条

7. **HTML结构变更后**：立即跑 `python3 -c "html.count('<div')==html.count('</div>')"` 验证配对

### 从专业仪器到手机传感器

在构建交互式仿真过程中，也调研了如何实际测量三参数：

| 层级 | 方案 | 成本 | 精度 | 适用 |
|------|------|------|------|------|
| 专业 | B-H回线分析仪(IWATSU/Magnet-Physik) | ¥5~30万 | ±1% | 精确测量 |
| 科研 | VSM/SQUID磁强计 | ¥50~200万 | 最高 | 材料表征 |
| 工程 | 示波器+电流探头 | ¥0.5~3万 | ±10% | 电路级验证 |
| 极简 | 手机+SS49E霍尔+phyphox | <¥10 | ±20% | 定性判饱和 |

**最廉价方案**：SS49E线性霍尔(¥2)配合手机示波器App(phyphox)，检测磁芯表面B场随电流变化→找到线性增长转平坦的拐点→即为H_S。成本不到¥10，能肉眼看到饱和现象，适合教学验证。

**教训**：手机内置磁力计(±200μT)无法直接测B_S(0.1~10T)，量级差5~7个数量级。低成本方案适用于**定性判断**而非**定量测量**。
