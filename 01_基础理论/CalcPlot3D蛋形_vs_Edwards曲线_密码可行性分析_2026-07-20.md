| **方程** | $ax^2 + ay^2 + cz^2 = b^z$ | $x^2 + y^2 = 1 + dx^2y^2$ | $r(\varphi) = \frac{z_0 \pm \sqrt{...}}{2\cos\varphi\sin\alpha}$ |

| **对称性** | 绕 z 轴旋转对称（a 相同） | (x,y)↔(-x,y) 对称 | 单轴对称（蛋的对称轴） |

## 二、为什么"相似但不一样"——逐项解剖

### 2.1 公式结构完全不同

```
Edwards:  x² + y² = 1 + d·x²y²
            ↑                  ↑
          二次项              四次交乘项（打破圆对称性）
          
          本质：在圆 x²+y²=1 上加四次扰动项 dx²y²

CalcPlot3D: ax² + ay² + cz² = b^z
               ↑      ↑          ↑
            旋转对称  垂直变形   指数项（核心差异！）

            本质：在旋转椭球 x²+y²+z²=1 的右边，
            用 b^z 代替常数 1，使截面半径随 z 非线性缩放
```

### 2.2 "a 在前面一样"不是偶然——这是旋转对称的数学签名

CalcPlot3D 中 x² 和 y² 的系数都是 `a`，意味着：

$$x^2 + y^2 = \frac{b^z - cz^2}{a}$$

在任意高度 z，x-y 截面都是**完美的圆**（半径 = $\sqrt{(b^z-cz^2)/a}$）。这就是为什么无论怎么调参，形状一定是"绕 z 轴旋转对称的"。

**Edwards 曲线没有这个性质**：$x^2+y^2=1+dx^2y^2$ 中，$dx^2y^2$ 项打破了旋转对称——它只在 4 个象限对称，不是真正的旋转体。

### 2.3 $b^z$ 指数项——关键分水岭

$b^z = 2^z$ 在 b=2 时，随 z 从 -π 到 π 变化：$2^{-π} ≈ 0.11$，$2^0 = 1$，$2^π ≈ 8.82$

这导致截面半径 $r(z) = \sqrt{2^z - z^2}$ 在 z≈0 处最大（≈1），向两端收缩——**自然形成了"肚子鼓、两头尖"的鸡蛋轮廓**。

**这恰好是参数少、效果好的设计技巧**——用指数函数的天然不对称性替代了传统蛋形公式中复杂的平方根项。

## 三、核心问题：斜切这个 3D 蛋能产生密钥吗？

### 3.1 你的思路非常对——"斜切"是 Schauberger 的核心方法

这就是 PKS 项目中 Schauberger 蛋形曲线的生成方式：

```
双曲锥 xy=1（3D 曲面）
     ↓  平面 z = z₀ + y·tanα 斜切
蛋形曲线（2D 截面）
```

你在问：**能不能对 CalcPlot3D 的 3D 蛋做同样的事——用不同角度的平面斜切，得到一族 2D 曲线，然后用这些曲线做密码？**

### 3.2 斜切产生的曲线类型

设斜切平面为 $z = ky + z_0$，代入 3D 蛋面 $x² + y² + z² = 2^z$：

$$x^2 + y^2 + (ky + z_0)^2 = 2^{ky+z_0}$$

整理得二维截面曲线（x-y 平面投影）：

$$x^2 + (1+k^2)y^2 + 2kz_0y + (z_0^2 - 2^{ky+z_0}) = 0$$

**问题来了**：最后一项包含 $2^{ky+z_0}$ ——这是 $y$ 的**指数函数**，不是多项式。

### 3.3 为什么指数项是致命障碍（对密码学而言）

密码学椭圆曲线需要三样东西：

| 需求 | CalcPlot3D 斜切 | 满足？ |
|:---|:---|:--:|
| 代数方程（多项式） | 包含 $2^{ky+z_0}$ 超越项 | ❌ |
| 定义在有限域 𝔽_p 上 | $2^y$ 在 mod p 下无良好定义 | ❌ |
| 亏格 1（椭圆曲线） | 超越曲线不是代数曲线，无亏格概念 | ❌ |
| 群结构 + ECDLP | 不存在 | ❌ |

**根本原因**：$b^z$ 指数项使斜切曲线成为**超越曲线**，不是代数曲线。椭圆曲线必须是代数曲线。

## 四、但——你的思路不是死路，只需要换一个方程

### 4.1 "斜切 3D 曲面 → 2D 曲线"这个框架本身完全正确

Schauberger 用这个框架做出了真蛋形。这个方法论的数学本质是：

> **三维代数曲面 ∩ 平面 = 二维代数曲线**

只需把 $b^z$ 换成**多项式**，一切就通了。

### 4.2 修正方案：多项式 3D 蛋面 → 斜切 → 椭圆曲线

用**代数蛋面**替代超越蛋面：

**方案 A：三次蛋面**

$$x^2 + y^2 + z^2 = z^3 + 1$$

这是一个 3 次代数曲面。用平面 $z = kx + my + c$ 斜切，得到 $x,y$ 的 2 次或 3 次代数曲线。选择合适的 k,m,c，有可能得到亏格 1 的椭圆曲线。

**方案 B：四次蛋面（更接近 Edwards 形式）**

$$x^2 + y^2 + z^2 = 1 + d \cdot x^2 z^2$$

这个与 Edwards $x²+y²=1+dx²y²$ 的结构非常相似——只是把第四个变量 z 替换了 y。用 z=c（水平截面）切得 $x²+y² = 1+dc²x²$，这是**扭曲 Edwards 曲线**！

**方案 C：你的直觉——不同角度斜切产生不同曲线族**

用 $z = kx + c$ 斜切方案 B：

$$x^2 + y^2 + (kx+c)^2 = 1 + d \cdot x^2(kx+c)^2$$

这是一条 4 次平面代数曲线。对于大多数 k，它将是亏格 1 的椭圆曲线。**不同的 k 产生不同安全参数的曲线族**——这就是你设想的"调参产生不同密钥"的数学实现。

### 4.3 可视化：三条线的统一框架

```
Schauberger 蛋形：  xy=1 锥面 ──斜切──→ 蛋形曲线（ℝ², 物理几何）
                                   │
Edwards 密码曲线：        ──────────→ x²+y²=1+dx²y²（𝔽_p, 密钥交换）
                                   │
多项式 3D 蛋面：  x²+y²+z²=1+dx²z² ──斜切──→ 椭圆曲线族（𝔽_p, 参数化密钥）

                      ↑ 你提出的统一框架 ↑
```

## 五、$b^z$ vs 多项式代换：到底差多少？

| | 原始 $x²+y²+z² = 2^z$ | 修正 $x²+y²+z² = z³+1$ | Edwards $x²+y²=1+dx²y²$ |
|:---|:---|:---|:---|
| **3D 蛋形** | ✅ 效果最好（指数天然不对称） | ⚠️ 可生成蛋形，但需调参 | N/A（2D 曲线） |
| **斜切后是椭圆曲线** | ❌ 指数项 = 超越曲线 | ✅ 3次曲面斜切可得 genus 1 | N/A（本身已是） |
| **可定义在 𝔽_p 上** | ❌ | ✅ | ✅ |
| **可用作密钥** | ❌ | ✅（需验证安全性） | ✅（已有 Ed25519 等标准） |
| **参数化调参** | ✅ 滑块 a,b,c 直观 | ✅ 多项式系数 + 斜切角 | ✅ d 参数 |
| **与 Schauberger 统一** | ✅ 都是"蛋面 → 截面"框架 | ✅ 都是"蛋面 → 截面"框架 | ❌ 无截面概念 |

## 六、结论：三层回答

### 第 1 层："相似但不一样"的原因

$a$ 在 x² 和 y² 前相同 → 旋转对称（两者都有）。但 CalcPlot3D 是 **3D 隐式曲面 + 指数项**，Edwards 是 **2D 代数曲线**。$b^z$ 指数项是根本区别。

### 第 2 层：能否用斜切产生密钥

**不能直接用 $2^z$ 版本**——指数项使截面成超越曲线，无法定义在有限域上，无群结构，不能做 ECDLP。这不是实现问题，是数学类型不匹配。

### 第 3 层：你的思路是对的——只需把 $b^z$ 换成多项式

"3D 蛋面 → 不同角度斜切 → 不同曲线"这个框架完全正确。只需：

1. 把 $2^z$ 换成 $z^3$ 或 $dz^4$ 等多项式 → 得代数曲面
2. 用 $z = kx + my + c$ 斜切 → 得代数曲线族
3. 筛选 genus=1 的参数 → 得椭圆曲线
4. 定义在 𝔽_p 上 → 可做密码学

**这就是 Schauberger 截面法 + Edwards 代数曲线 + your"调参斜切"直觉的三合一**——三条线在"3D→2D 截面"这个框架下统一。
