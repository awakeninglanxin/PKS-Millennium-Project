# AI 图像质量评估算法原理

> 从 AI 生成图像的"瑕疵检测"到"美感评分"，一篇讲清楚背后到底发生了什么。

---

## 一、AI 怎么"看"出图有问题？

### 1.1 判别模型 vs 生成模型

AI 审美的底层是两个模型家族：

- **生成模型（Generator）**：画图的，比如 Stable Diffusion / FLUX / Midjourney
- **判别模型（Discriminator）**：评图的，用来判断"这张图好不好"

在扩散模型训练时，两者是**对抗训练**的——生成器不断画，判别器不断挑毛病，直到生成器画的图骗过判别器为止。

判别模型的学习方式：

```
输入：一张图 → 卷积神经网络(CNN)逐层提取特征 → 输出：得分(0-1)
                      │
                      ├── 低层：边缘、纹理、噪声
                      ├── 中层：形状、物体部件（手、脸、翅膀）
                      └── 高层：语义合理性（两只手、五官位置、对称性）
```

### 1.2 检测常见问题的机制

| 问题 | 判别模型检测原理 |
|------|-----------------|
| 手指多余/残缺 | CNN中层学习到"人体结构先验知识"——手应该有5根手指，骨骼拓扑固定。违反即低分 |
| 脸部扭曲 | 高层语义理解"人脸应有对称五官"，不对称或比例失衡触发警告 |
| 翅膀不对称 | 物体部件的空间位置关系偏离训练集均值 |
| 身体畸变 | 解剖学拓扑约束（四肢数量、关节位置、身体比例） |
| 模糊/噪点过多 | 频域分析（傅里叶变换），高频成分异常 |

### 1.3 关键概念：拓扑约束

AI 判别器学到的不是"黄金比"，而是**统计拓扑**。具体来说：

- 人类的手部解剖有**固定拓扑**：手掌1块 + 5根手指，关节数、骨骼连接方式都是固定的
- 训练集（如 LAION-5B）包含了数百万张正确的人体图像
- 判别器通过学习这些统计规律，形成了"人体应该是这样的"先验知识
- 当生成图违反了这个先验（多出一根手指），判别器的损失函数就会产生很大的梯度信号

**这不是几何比例的黄金分割，而是概率分布的偏离**。

## 二、"美感"的数学模型

### 2.1 什么是"美丽"？

AI 没有主观审美，它的美感的底层逻辑是：

```
美 = 高概率 + 低混乱
```

扩散模型的训练本质上是在拟合一张"美图的概率分布"。当生成图的特征接近训练集中的高频模式时，就是"美的"。

### 2.2 具体指标

#### CLIP Score

- 使用 OpenAI 的 CLIP 模型（图像-文本对对齐）
- 将 prompt 嵌入到向量空间
- 将图像嵌入到同一个向量空间
- **相似度 = cos(文本向量, 图像向量)** → 这就是"这张图是否符合prompt描述"的分数

#### FID (Fréchet Inception Distance)

- 用预训练 Inception 网络的特征分布来衡量
- 计算两组特征的 Fréchet 距离（ Wasserstein-2 距离）
- 公式：

$$FID = \|\mu_g - \mu_r\|^2 + \text{Tr}(C_g + C_r - 2\sqrt{C_g C_r})$$

其中 μ 是均值向量，C 是协方差矩阵，g 是生成数据，r 是真实数据。

#### NIQE / BRISQUE

- 无参考质量评估
- 基于自然场景统计（NSS）模型
- 检测模糊、噪声、伪影

### 2.3 人脸质量评估

- **ArcFace / FaceNet**：检测人脸是否清晰、对称、符合人脸数据库统计分布
- **Landmark Detection**（如 Dlib / MediaPipe）：检测2D/3D人脸关键点
  - 检测对称性：左眼、右眼、嘴角等的坐标关于中线是否对称
  - 对称度 = |左坐标 - 右坐标| / 2 → 阈值判定

## 三、黄金比、白银比在 AI 中有用吗？

### 3.1 简短回答：**几乎没有**

黄金比 φ ≈ 1.618 在 AI 图像生成中：

- **不是训练数据的直接约束**：扩散模型没有显式学习黄金比例
- **间接影响**：训练数据中大量摄影作品/艺术品恰好符合黄金比构图，模型因此"学到"了这种统计偏好
- **不是主动检测指标**：判别模型不会拿尺子去量 φ 值

### 3.2 黄金比只在某些特定工具中使用

| 场景 | 是否使用黄金比 | 说明 |
|------|---------------|------|
| Stable Diffusion 训练 | ❌ 否 | 依靠数据驱动的隐式学习 |
| Midjourney prompt 工程 | ❌ 否 | 不主动检测 |
| Adobe Photoshop 自动裁剪 | ⚠️ 有限 | 仅作为构�选项之一 |
| 人工评审标准 | ⚠️ 可能 | 设计师可能参考 |

## 四、AI 质量评估的实际流程

在我们的慈海九宫格项目中，使用的是一套简化版的多维度评分体系：

### 4.1 评估维度

| 维度 | 权重 | 检测方法 |
|------|------|---------|
| **清晰度** | 25% | 频域分析 + Laplacian 方差 |
| **解剖合理性** | 30% | CNN + 人体拓扑约束 |
| **面部对称性** | 20% | 关键点检测 + 镜像对比 |
| **配色和谐度** | 15% | HSV 空间分布熵 |
| **Prompt 一致性** | 10% | CLIP Score |

### 4.2 检测流水线

```
生成图 → 面部检测 → 关键点提取 → 对称性评分
                  ↘ 肢体检测 → 拓扑检查 → 完整性评分
                   ↘ 频域分析 → 清晰度评分
                    ↘ CLIP比对 → Prompt一致性评分
                     ↘ 综合打分 → 不合格标记 → 重新生成
```

### 4.3 常见问题分类

| 问题 | 占比 | 典型修复手段 |
|------|------|-------------|
| 手部畸形 | ~35% | 增加 prompt "FIVE FINGERS" |
| 面部不对称 | ~25% | 增加 prompt "symmetrical face" |
| 肢体缺失 | ~15% | 增加 prompt "complete body" |
| 模糊/低清 | ~12% | 增加分辨率/超分 |
| 颜色失真 | ~8% | 调整 CFG scale |
| 其他 | ~5% | — |

## 五、总结

1. **AI 的"审美"是统计学习的结果，不是几何比例的运用**
2. **判别模型通过训练数据中学到的"正常人体拓扑"来检测问题**
3. **黄金比等比例只在人类设计师的直觉中存在，AI 不用这些作为评分标准**
4. **真正起作用的是 CLIP Score、FID、面部关键点检测、频域分析等算法**
5. **每次"重修"质量差的图，本质上是通过改变 prompt 和 seed，让生成器走出"坏分布"的概率陷阱**
