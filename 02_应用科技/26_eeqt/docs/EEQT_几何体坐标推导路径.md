# EEQT 几何体坐标推导路径

## 核心原则
1. **去猜测化**：基于已知数学定义直接推导，不猜测坐标
2. **简化逻辑**：使用最简数学表达式生成顶点
3. **显式归一化**：顶点坐标可直接生成，scale在渲染层调整

## 黄金比例常量
```javascript
const PHI = (1 + Math.sqrt(5)) / 2;  // ≈ 1.618033988749895
const PHI_INV = 1 / PHI;             // ≈ 0.6180339887498949
```

## 1. 正二十面体 (Icosahedron) - 12顶点

### 标准顶点定义
正二十面体的顶点由三组互相垂直的黄金矩形交点构成：
```
(0, ±1, ±PHI) 及其循环排列
(±PHI, ±1, 0)
(±1, 0, ±PHI)
```

### 归一化推导
顶点 `v = (0, 1, PHI)` 的模长：
```
|v| = √(0² + 1² + PHI²) = √(1 + PHI²)
```
由于 `PHI² = PHI + 1`，所以：
```
|v| = √(PHI + 2)
```
归一化因子：
```
s = 1 / √(PHI + 2)
```
最终顶点坐标：
```
(0, s, PHI·s) 等
```

### 代码实现
```python
def make_icosahedron():
    s = 1 / math.sqrt(PHI * PHI + 1)  # 1/√(1+PHI²) = 1/√(PHI+2)
    verts = []
    for s1 in [-1, 1]:
        for s2 in [-1, 1]:
            verts.append([0, s1 * s, PHI * s * s2])
            verts.append([s1 * s, 0, PHI * s * s2])
            verts.append([s1 * s, PHI * s * s2, 0])
    return verts
```

## 2. 正十二面体 (Dodecahedron) - 20顶点

### 立方体角顶点 (8个)
位于单位立方体的角点上：
```
(±1/√3, ±1/√3, ±1/√3)
```

### 环顶点 (12个)
在三个坐标平面上的环：
```
(0, ±Rr, ±PHI·Rr)
(±Rr, ±PHI·Rr, 0)
(±PHI·Rr, 0, ±Rr)
```
其中 `Rr` 是环半径归一化因子。

### 归一化推导
环顶点 `v = (0, Rr, PHI·Rr)` 的模长：
```
|v| = Rr·√(1 + PHI²) = Rr·√(PHI+2)
```
要求 `|v| = 1`，所以：
```
Rr = 1 / √(PHI+2)
```
这与二十面体的 `s` 相同。

### 代码实现
```python
def make_dodecahedron():
    Rc = 1 / math.sqrt(3)        # 立方体角归一化
    Rr = 1 / math.sqrt(PHI*PHI + 1)  # 环顶点归一化 = s
    
    verts = []
    # 立方体角
    for s1 in [-1, 1]:
        for s2 in [-1, 1]:
            for s3 in [-1, 1]:
                verts.append([s1*Rc, s2*Rc, s3*Rc])
    # 环顶点
    for s1 in [-1, 1]:
        for s2 in [-1, 1]:
            verts.append([0, s1*Rr, PHI*s2*Rr])
            verts.append([s1*Rr, PHI*s2*Rr, 0])
            verts.append([PHI*s1*Rr, 0, s2*Rr])
    return verts
```

## 3. C60 富勒烯 (60顶点)

### 四组顶点定义

**A组 (8顶点)**：
```
(±1, ±1, ±(2·PHI+1)) / Ra
```
归一化半径：
```
Ra = √(1+1+(2·PHI+1)²)
```

**B组 (12顶点)**：
```
(0, ±(PHI+2), ±PHI) / Rb 及其循环排列
```
归一化半径：
```
Rb = √((PHI+2)² + PHI²)
```

**C组 (24顶点)**：
```
(±PHI, ±2, ±(PHI+1)) / Rc 及其循环排列
```
归一化半径：
```
Rc = √(PHI² + 4 + (PHI+1)²)
```

**D组 (16顶点)**：
```
(±1, ±(PHI+1), ±(PHI+2)) / Rd 及其循环排列
```
归一化半径：
```
Rd = √(1 + (PHI+1)² + (PHI+2)²)
```

### 关键修正
- **D组顶点数**：2×8=16（非3×8=24）
- **各组独立归一化**：每组有自己的归一化半径

## 4. 半球体坐标 (Pentagon/Hexagon Hemisphere)

### 生成原理
1. 从完整C60顶点中筛选 `z ≥ 0` 的顶点
2. 确保所有顶点在单位球面上
3. 分为五边形面和六边形面两个半球

### 验证要点
```python
# 所有顶点 z ≥ 0
assert all(v[2] >= 0 for v in hemisphere_verts)

# 所有顶点在单位球面上
for v in hemisphere_verts:
    assert abs(math.sqrt(v[0]**2+v[1]**2+v[2]**2) - 1.0) < 1e-8
```

## 5. 缩放逻辑

### 核心原则
**边界圆固定不变，只缩放几何体顶点**

### 实现方案
1. **边界圆**：固定像素半径 `BASE_SCALE = Math.min(W,H)*0.46`
2. **几何体缩放**：在投影公式层面缩放x,y坐标
   ```javascript
   // 立体投影带缩放
   function projStereoScaled(r, s) {
       const den = 1 + r[2];
       return [r[0]*s/den, r[1]*s/den];
   }
   ```
3. **缩放因子构成**：
   ```
   totalScaleFactor = autoScaleFactor × userScaleMultiplier
   ```
   - `autoScaleFactor`：C60/半球自动修正因子
   - `userScaleMultiplier`：用户手动缩放滑条 (0.5~2.0)

### 自动修正因子
| 几何体 | stereo投影 | 原因 |
|--------|------------|------|
| C60完整 | 1.0 | z范围[-0.9485,+0.9485]，赤道已在边界圆 |
| Pentagon半球 | 1.1716 | z∈[0.1716,0.9392]，最大投影半径=1/(1+0.1716)=0.8535 |
| Hexagon半球 | 1.0720 | z∈[0.0720,0.9150]，最大投影半径=1/(1+0.0720)=0.9328 |

## 6. 对齐模式 (Alignment Modes)

### 实现方式
通过绕Y轴旋转原始顶点实现三种对齐模式：
```python
def rot_y(v, angle):
    a = math.radians(angle)
    ca, sa = math.cos(a), math.sin(a)
    return [v[0]*ca + v[2]*sa, v[1], -v[0]*sa + v[2]*ca]
```

### 旋转角度
| 几何体 | 模式0 | 模式1 | 模式2 |
|--------|-------|-------|-------|
| 二十面体 | 0° | 72° | 144° |
| 十二面体 | 0° | 72° | 144° |
| C60 | 0° | 36° | 72° |
| 四面体 | 0° | 60° | 120° |

## 7. 推导检查清单

### 验证步骤
1. ✅ 使用标准数学定义，不猜测坐标
2. ✅ 验证模长归一化：所有顶点 |v| ≈ 1.0 (±1e-8)
3. ✅ 验证顶点数：与理论值一致
4. ✅ 验证对称性：适当符号排列
5. ✅ 验证半球条件：z ≥ 0 或 z ≤ 0

### 常见错误避免
1. ❌ 不要混淆PHI和PHI_INV
2. ❌ 不要忘记归一化因子
3. ❌ 不要使用未经验证的旋转角度
4. ❌ 不要硬编码坐标值，使用公式生成

## 9. 最优 ε 预设（SOLID_EPS）

### 物理意义
ε（epsilon）是EEQT中的量子跳变参数，控制方向性强度：
- **ε ≈ 0**：强方向性，点几乎直接跳到detector，分形最清晰
- **ε ≈ 1**：弱方向性，接近均匀随机分布，分形模糊

### 正确预设值（必须严格遵守）
```javascript
const SOLID_EPS={
  'tetrahedron':800,      // ε=0.8
  'octahedron':100,       // ε=0.1 - 关键！低epsilon=清晰分形
  'cube':800,             // ε=0.8
  'merkaba':800,          // ε=0.8
  'icosahedron':900,       // ε=0.9 - 较高epsilon适合更多顶点

  'c60_full':800,         // ε=0.8
  'compound_32':800,       // ε=0.8

};
```
