## 1. 核心物理与数学框架

### 1.1 事件增强量子理论（EEQT）

EEQT 将量子测量建模为**量子态在 Bloch 球面上的混沌游走**，测量事件对应"跳变到某个检测器方向"。

Bloch 球面上任意纯态：
$$\vec{r} = (x, y, z), \quad x^2 + y^2 + z^2 = 1$$

### 1.2 转移映射（核心动力学公式）

给定当前 Bloch 向量 **r**，选中检测器方向 **nᵢ**，参数 ε，则新位置 **r'** 为：

$$\boxed{
\vec{r}' = \frac{(1 - \varepsilon^2)\,\vec{r} + 2\varepsilon\,(1 + \varepsilon\,\vec{n}_i \cdot \vec{r})\,\vec{n}_i}
               {1 + \varepsilon^2 + 2\varepsilon\,\vec{n}_i \cdot \vec{r}}
}$$

代码实现：
```javascript
function T(r, n, eps) {
  const e2 = eps * eps;
  const ndr = dot(n, r);
  const num = add(scl(r, 1 - e2), scl(n, 2 * eps * (1 + eps * ndr)));
  const den = 1 + e2 + 2 * eps * ndr;
  return [num[0] / den, num[1] / den, num[2] / den];
}
```

### 1.3 检测器选择概率（混沌博弈核心）

选中第 i 个检测器的概率（与 dot(r, nᵢ) 成正比）：

$$\boxed{
p_i = \frac{1 + \varepsilon^2 + 2\varepsilon\,\vec{n}_i \cdot \vec{r}}{N\,(1 + \varepsilon^2)}
}$$

其中 N = 检测器总数。ε = 0 时退化为均匀分布；ε → ∞ 时趋向最近的检测器。

### 1.4 混沌博弈算法流程

```
1. 初始化 Bloch 向量 r = (0.1, 0.1, 0.98)
2. Burn-in 阶段：执行 1,000 步（正二十面体/C60: 3,000 步）
3. 主循环（points 次）:
   a. 计算每个检测器的选择概率 pᵢ
   b. 轮盘赌随机选中一个检测器 i
   c. 应用转移映射 r ← T(r, nᵢ, ε)
   d. 将 r 投影到 2D 平面，记录坐标
4. 用 ImageData 将所有点绘制到画布
```

## 3. 投影方法（3 种）

### 3.1 球极平面投影（Stereographic Projection）

```
公式：
  sx = x / (1 + z)
  sy = y / (1 + z)
  其中 r = (x, y, z)，|r| = 1

特点：
  • 保角（角度不变）
  • 圆→圆：球面上的圆投影后仍为圆
  • 单位球面上的北极点 (z=1) → 无穷远点
  • 赤道 (z=0) → 单位圆
```

> **代码实现**：REF_plus_hemi_v3 和 EEQT_fixed_c60 均使用此投影作为默认

### 3.2 兰勃特等面积投影（Lambert Equal-Area Projection）

```
公式：
  L = √((1 - z) / 2)
  sx = (x / √(x² + y²)) × L
  sy = (y / √(x² + y²)) × L

特点：
  • 面积保持（每个投影面积与球面面积成正比）
  • 适合统计分析（如 Hausdorff 维数计算）
  • 赤道处压缩最小
```

### 3.3 正交投影（Orthographic Projection）

```
公式：
  sx = x
  sy = y
  （直接丢弃 z 坐标）

特点：
  • 从 +z 方向俯视 Bloch 球
  • 不保角、不保面积
  • 只能看到球面"前面"半球
  • 最直观但信息损失最大
```

## 5. Hausdorff 维数计算

### 5.1 方法：盒计数法（Box-Counting）

对投影后的 2D 点云，使用不同网格尺度 εᵢ 计数被占据的格子数 N(εᵢ)：

```
尺度序列：ε = [0.5, 0.25, 0.125, 0.0625, 0.03125, 0.015625]

算法：
  1. 将所有点 (x, y) 映射到网格：box_id = (⌊x/ε⌋, ⌊y/ε⌋)
  2. 统计不同 box_id 的数量 → N(εᵢ)
  3. 在 log-log 坐标下线性回归：log N(ε) = D_H · log(1/ε) + const
  4. 斜率即为 Hausdorff 维数 D_H
```

### 5.2 Hausdorff 维数范围

```
D_H ∈ [0, 2]  ← 2D 投影的 Hausdorff 维数上界为 2
D_H ≈ 1.0     ← 分形充满一维曲线
D_H ≈ 1.5     ← 布朗运动典型值
D_H ≈ 2.0     ← 填满整个二维平面
```

### 5.3 投影影响

不同投影方式会影响 Hausdorff 维数：
- **Stereographic**：球极投影拉伸赤道附近区域，测量值偏高
- **Lambert**：等面积投影最忠实反映分形维数
- **Orthographic**：只考虑可见半球，可能低估

## 8. 关键数学验证

### 8.1 Bloch 向量归一化

每次转移后必须重新归一化，确保 |r| = 1：

```javascript
const norm = (a) => {
  const L = Math.sqrt(dot(a, a));
  return L < 1e-15 ? [0, 0, 1] : [a[0]/L, a[1]/L, a[2]/L];
};
```

### 8.2 转移公式的几何意义

```
分子：(1-ε²)r + 2ε(1+ε·cos θ)n
分母：1 + ε² + 2ε·cos θ

其中 θ = arccos(n·r)，即 nᵢ 与 r 的夹角
• cos θ → 1（对齐）：分母大，分子主导 r 本身（跳动小）
• cos θ → -1（对立）：分母小，分子主导 nᵢ（跳向检测器）
```

### 8.3 概率归一化验证

$$\sum_{i=1}^{N} p_i = \frac{\sum_i (1 + \varepsilon^2 + 2\varepsilon\,n_i \cdot r)}{N(1+\varepsilon^2)} = \frac{N(1+\varepsilon^2) + 2\varepsilon\,\sum_i n_i \cdot r}{N(1+\varepsilon^2)}$$

若 Σnᵢ = 0（如四面体、立方体、梅尔卡巴），则概率归一化为 1 ✓

```javascript
for(s1,s2,s3 ∈ {±1}){
  v.push([s1, s2*(φ+1), s3*(φ+2)]);  // 第1个排列 ✓
  v.push([s1*(φ+1), s2*(φ+3), s3]);  // 第2个排列 ✓
  // 不要第3个排列（第3个排列会产生重复顶点）
}
```

### 10.3 半球体缩放修正（核心Bug） ✅

**症状**：C60 Stereographic 图形明显太大，半球体视觉偏小。

**根本原因**（逐坐标数学验证，2026-04-25）：

立体投影公式 `s = 1/(1+z)`：

```
z = -1  →  s = ∞   (南顶点→无穷远，不在有限范围)
z =  0  →  s = 1    ← 边界圆由赤道决定，永远是 s=1
z = +1  →  s = 0.5  (北极→原点)
```

**C60 的关键错误**：赤道 z=0 → s=1 = boundary circle。
对 C60 乘以 1.55× 后，赤道被推到 s=1.55 **超出边界圆之外**，所以"太大"。

**半球体也错误**：z 全为正值（无负 z），最低 z 决定最大 s：
- Pentagon: z_min=0.1716 → max_s=0.8535 → 需 scaleFactor=1.1716×
- Hexagon:  z_min=0.0720 → max_s=0.9328 → 需 scaleFactor=1.0720×

| 版本 | 公式 | C60 stereo | PentHemi | HexHemi | 效果 |
|------|------|-----------|----------|---------|------|
| 原始错误 | `ND===60 ? 1.55 : 1` | ❌ 1.55×→赤道出界 | ❌ 1.0×→太小 | ❌ 1.0×→太小 | 三者全错 |
| v2修正 | `(ND===60\|\|isHemi)?1.55:1` | ❌ 赤道仍在1.55× | ✅ 1.55× | ✅ 1.55× | C60 仍错 |
| **最终修正** | `各几何体独立因子` | ✅ **1.0×** | ✅ **1.1716×** | ✅ **1.0720×** | 全部正确 |

```javascript
// 修正前（第279行）：
const scale = Math.min(W,H)*0.46*(ND===60&&projType!=='ortho'?1.55:1);

// 修正后：
const isHemi = (type === 'pentagon_hemi' || type === 'hexagon_hemi');
const scale = Math.min(W,H)*0.46*((ND===60 || isHemi) && projType!=='ortho' ? 1.55 : 1);
```

**数学原理**：半球体 z ≥ 0，投影公式中分母 `(1+z) ∈ [1.07, 1.94]`，远大于 C60 的 `[0.05, 1.95]`，
导致投影被压缩。乘以 1.55× 补偿压缩效应，使半球体分形视觉上与 C60 等比例。
