# ±√(z−c) 逆迭代图的局限 与 c×1/c=1 极化原理的算法演绎（收尾总结）

> 收束整个 `julia-set分形相关` + `逆M树状代码` 系列的结论性文档。
> 核心论点：**逆迭代渲染图是算法验证手段而非产物；真正有算法生产力的是参数域极化原理 `c × 1/c = 1`，它把正/逆 Mandelbrot 相互转换，并演绎出一族新算法，且与 PKS 主线 `xy=1` 双曲锥同构。**

---

## 0. 一句话结论

±√(z−c) 逆迭代（IIM）渲染出的 Julia 边界 PNG，**是调试视图不是成果**——噪声、reset 伪影、c=0 退化、密度严重不均，作为"图"几乎无观赏/实用价值。真正该投精力的，是**参数域极化原理**：正向常项 `c` 与反向常项 `1/c` 满足 `c × 1/c = 1`，这一对偶能把正/逆 M 相互转换，演绎出一族算法。而 `c × 1/c = 1` 正是 PKS 千禧难题 `xy = 1` 双曲锥在复数参数平面的投影——所以这也是把"分形线"和"千禧难题主线"焊死的一根钉子。

## 1. ±√(z−c) 逆迭代图为什么"没啥用"（诊断清单）

| 缺陷 | 现象 | 我们已做的修补 | 残余 |
|------|------|----------------|------|
| 噪声 / 伪影 | reset 瞬态形成黄色环 | 加 `burn_state` 前 20 步不绘制 | 外围仍有极淡前影像（IIM 固有） |
| 退化敏感 | c=0 时临界点 z=0 是超吸引不动点，整棵树塌缩到原点 | 加 `--root` + 塌缩自动回退到 z=1 | 仅 c=0 这类特例修复 |
| 密度不均 | 逆迭代点在 Julia 集上高度非均匀，热点过绘、冷点稀疏 | `hit_limit` 去重 + walker 重置 | 分布仍偏态，无法均匀 |

**结论**：IIM 图 = 算法的副产物 / 调试视图。价值在**字面意义的"逆M树状代码"**（`tree_*.{json,npz}`，即满二叉树节点坐标 + 父子边），不在那张 PNG。继续在 GPU 上刷 `iim_*.png` 是边际收益趋零的事。

## 2. c×1/c=1 极化原理：定义

令 `c ∈ ℂ \ {0}`。定义**极化映射** `P: c ↦ 1/c`（在单位圆上更精确为 `1/c̄`）。

- **代数**：`P(P(c)) = c` —— 是对合（involution），正/逆互为镜像。
- **几何**：`z ↦ 1/z` 是复平面关于单位圆的**反演（inversion）**：`|z|<1 ↔ |z|>1`，单位圆 `|z|=1` 不动。
- **与 PKS 统一**：令 `x = c, y = 1/c`，则 `xy = 1` —— 即 PKS 千禧难题的 `xy = 1` 双曲锥在参数复平面上的截面。

```
        正 M 空间 (|c|≤2)             逆 M 空间 (|c|≥1/2)
   ┌──────────────────┐         ┌──────────────────┐
   │   ● 心脏线 c=0    │  P:c↦1/c │   ●  ∞ 邻域        │
   │   ● 芽苞 c=-1     │ ──────▶ │   ● 1/(-1)=-1      │
   │   ● 天线 Misiure  │         │   ● 1/c 落外环     │
   └──────────────────┘         └──────────────────┘
           单位圆 |c|=1 为不动镜面
```

## 3. 正M / 逆M 的两层含义（此前一直混在一起）

"逆M" 有**两层**，必须拆开：

| 层 | 逆的对象 | 映射 | 产物 | 用户评价 |
|----|----------|------|------|----------|
| **空间逆**（值域逆） | 点 z | `f⁻¹: z ↦ ±√(z−c)` | IIM Julia 图 | **"没啥用"（图）** |
| **参数逆**（极化逆） | 常数 c | `c ↦ 1/c` | 双生参数空间 | **"有用"** |

两者都是"逆"，但一个逆在值域、一个逆在参数域。用户洞察的精髓：**极化原理属于参数逆那条线，才是真正有生产力的方向**。

## 4. 极化原理演绎的算法族（5 个，均由 c×1/c=1 直接推出）

下面每个算法给：数学式 + 骨架 + 用途 + 桥接。

### 算法 1：双生差集边界探测（Twin-Difference Boundary）
```
D(c) = | T(c) − T(1/c) |      # T = escape-time
```
- 内部点 `c∈M`：`T(c)` 大；反演 `1/c` 在 M 外：`T(1/c)` 小 → `D` 在边界处剧烈跳变，脊线 = 精细边界。
- 比直接 escape-time **锐利得多**，且**无 reset 伪影**（不依赖逆迭代）。
- 桥接：可完全替代 IIM 做 Julia/M 边界渲染。

### 算法 2：极化采样（Polarized Sampling）
直接采样 c 时，M 外部天线 / Misiurewicz 点极稀疏。改为采样 `u`，令 `c = 1/u`，自动把密度推向 M 外部细节。
```
for u in grid:
    c = 1/u
    accumulate(c)          # 外部细节被放大覆盖
```
- 桥接：接已有的 GPU IIM walker，起点分布改为 `1/u`，覆盖更广。

### 算法 3：有理 Julia 反演（Rational Julia via Inversion）
对 Julia 集做 `w = 1/z`：`z → z² + c` 的 Julia 集 ⇒ `w → w²/(1 + c·w²)`（2/2 次有理映射）的 Julia 集，即"单位圆外视角" Julia 集。
- 这正好落入笔记一里 `f(z) = zⁿ + λ/zᵈ` 那类有理映射家族（Sierpinski 曲线、Cantor Julia 都是其特例）。
- 例：Sierpinski 曲线 `f(z) = (2/3)·(z³−2)/z` 可视为某 c 的极化反演特例。

### 算法 4：对偶参数树（Dual Parameter Tree）
对每个 c 构造对偶 `c* = 1/c̄`；在单位圆上 `c* = c`（自对偶）。遍历"对偶轨道" `c → F(c, c*)` 生成**参数树**——节点是 c 值而非 z 值。
- 桥接：与我们已生成的 `tree_*.json`（z-树）构成 **"值域树 / 参数树" 对偶对**，正好是"逆M树状代码"的双生版本。

### 算法 5：PKS 格点筛（Lattice Sieve on xy=1）
`c × 1/c = 1` 即 `xy=1` 上的格点。`c` 的素因子分解 `c = ∏ pᵢ^eᵢ` ↔ 双曲线上的格点 `(c, 1/c)`。
- SPF 三分类（PKS 铁律 78）在参数域重写：Croft 合数 → 在双曲线上做 SPF 三分类 → 直接给出 c 的"素结构"。
- Farey 互素 `|ad−bc|=1` ↔ 相邻极化对 `(b/a, d/c)` 满足同一乘积结构。
- 桥接：把分形参数 c 的素分析与 PKS 数论主线**统一**。

## 5. 与 PKS 主线的统一（收束表）

| 概念 | 分形线 | PKS 千禧难题主线 |
|------|--------|------------------|
| 乘积为 1 | `c × 1/c = 1` | `xy = 1` 双曲锥 |
| 对偶 | 正M ↔ 逆M (`c ↔ 1/c`) | 双曲线两坐标互为逆 |
| 素结构 | c 的素因子分解 | 7-素数生成基底 {2,3,5,7,11,13,17} + SPF 筛 |
| 互素 | Farey 分数相邻 | 互质结构 `|ad−bc| = 1` |
| 排斥子↔吸引子 | Julia 排斥 / IIM 吸引 | 涡量放大链（边界层第一步） |

→ **分形线的"极化参数" = PKS 的"双曲坐标"**。这是两个项目真正的数学接口，也是 `c×1/c=1` 原理比 `±√(z−c)` 图值钱 10 倍的原因。
