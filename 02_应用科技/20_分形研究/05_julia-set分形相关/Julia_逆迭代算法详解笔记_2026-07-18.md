## 0. 一句话核心

> **Julia 集是映射 f 的"动力排斥子"（repeller）；因此对 f 的逆映射做迭代，Julia 集反而成为"吸引子"。** 这就是所有逆迭代算法的根本思想——从任意起点反复取逆像，点会"落"到 Julia 集上。

这与本目录主题「逆M树状代码」直接对应：**逆迭代天生是树状结构**——每个点分裂出 2 个逆像（二岔树），迭代 depth 层得到 2^depth 个叶节点。本文是"逆M树状"最干净的理论蓝本。

---

## 1. 理论基础：Fatou 与 Julia 集

### 1.1 动力系统框架
- 研究映射 $f:\mathbb{C}\to\mathbb{C}$（通常取有理函数）。
- 状态空间 = 复平面 $\mathbb{C}$；迭代一次 = 流逝一个单位时间。
- 初值 $z_0$，第 $n$ 步位置 $f^n(z_0)$，轨道（orbit）定义为 $\{f^n(z_0)\}_{n\ge0}$。

### 1.2 Fatou 集与 Julia 集（互补定义）
| 概念 | 定义 | 动力学性质 |
|------|------|-----------|
| **Fatou 集** $F$ | 使迭代族 $\{f^n\}$ 构成**正规族**的最大开集 | 动力学"温顺"：邻近点长期行为相似 |
| **Julia 集** $J$ | $F$ 的补集（$J=\mathbb{C}\setminus F$） | 动力学**混沌**：任意接近的两点可有截然不同的长期行为 |

### 1.3 经典示例：$f(z)=z^2$ → $J=$ 单位圆
- 单位圆内：$f^n(z_0)\to 0$（吸引到不动点 0）
- 单位圆外：$f^n(z_0)\to\infty$
- 单位圆上：写 $z=e^{\alpha\pi i}$
  - $\alpha$ 有理 → 轨道有限点集
  - $\alpha$ 无理 → 轨道在单位圆上**稠密**（Beardon 1991, sec.1.3）
- 结论：**任意接近的两点，长期行为可完全不同** → 这就是混沌。

### 1.4 二次函数的普适性（关键）
> 任何二次函数的动力学，都可由某个 $f_c(z)=z^2+c$ 完全展现（Carleson & Gamelin 1993, p.123）——即任一二次 Julia 集都与某个 $J_c$ **同胚**。

因此研究 $f_c(z)=z^2+c$ 的 Julia 集（$J_c$）就足够一般。

### 1.5 逆迭代的理论支点（本文灵魂）
- $J$ = $f$ 的**排斥周期点**之闭包（Beardon 1991, Thm.6.9.2）。
- 所以 $J$ 是 $f$ 的动力排斥子 → 必然是 $f^{-1}$ 的吸引子。
- 由此构造第一类 Julia 集生成算法。

## 2. 简单确定性逆迭代（Simple Deterministic）

### 2.1 两个逆映射
$f_c(z)=z^2+c$ 有两个逆：
$$
f_1^{-1}(z)=+\sqrt{z-c},\qquad f_2^{-1}(z)=-\sqrt{z-c}
$$

### 2.2 算法流程
从起点（如 $\{1\}$）出发，反复套用 `invImage`：
- 1 个点 → 2 个点 → 4 个点 → … → 每迭代一次点数翻倍，且都**更靠近** $J_c$。

Mathematica 伪码（原文）：
```mathematica
invImage[complexPoints_] :=
  Flatten[ ({1, -1} Sqrt[#1 - c] & ) /@ complexPoints ]

points = ({Re[#], Im[#]} & ) /@ Nest[invImage, {1}, depth];
ListPlot[points, AspectRatio -> Automatic, Axes -> False,
         PlotStyle -> {AbsolutePointSize[0.4]}]
```
- 封装函数：`JuliaSimple[c, depth]`（第二个参数是**深度**而非总点数）。
- $c=0$ 时 $J_c$ 为单位圆，depth=10 即可画出。

### 2.3 致命缺陷
某些 Julia 集区域"吸引力"强、某些弱 → 图像**细节不均**（部分密、部分疏）。朴素解法是加大 depth，但常需 depth>50 → $2^{50+}$ 个点，不可行。

---

## 3. 改进算法：`JuliaModified`（去重 + 分辨率量化）

### 3.1 核心思想
- 引入 **Resolution（分辨率）** 参数 `res`（默认 200）：把逆像**量化**到网格，使"靠得很近的点"被视为同一点。
- 维护全局已绘点集 `pointsSoFar`；每次迭代只保留**新点**（去重），列表长度不再按 $2^{\text{depth}}$ 爆炸。

### 3.2 量化逆像
```mathematica
res = 200;
invImage[points_] :=
  Flatten[ (Floor[{1, -1} res Sqrt[#1 - c]]/res &) /@ points ];
```

### 3.3 去重迭代
```mathematica
reducedInvImage[points_] := Module[{newPoints},
  newPoints = Complement[invImage[points], pointsSoFar];
  pointsSoFar = Union[newPoints, pointsSoFar];
  newPoints]

pointsSoFar = Nest[invImage, {1}, 5];        (* 预热 *)
FixedPoint[reducedInvImage, pointsSoFar];   (* 跑到无新点为止 *)
```

### 3.4 效果对比（文中实测）
以 $c=0.68i$ 这个较复杂 Julia 集为例：
- `JuliaSimple` 要达到同等细节需 depth ≈ 77+5 = **82** → 列表 $2^{82}$ 个点（天文数字）。
- `JuliaModified` 仅用约 **20,399** 个点就画出**更多**细节。
- `Resolution` 控制粗细：40=快速粗糙，200=精细。

> **与本项目的关联**：`pointsSoFar` + `Complement` 去重，本质是**树遍历时的 visited 集合（已访问剪枝）**。逆迭代树长出新枝才记录，重复节点不入队——这正是"树状代码"要做的事。

## 4. 推广到任意有理函数（n 岔树）

### 4.1 三次多项式示例 $f(z)=z^3+z+0.6i$
- 用 `NSolve[f[z]==#, z]` 求所有逆像 → 装进 `funcs`。
- `invImage[points_] := Flatten[(Through[funcs[#]] &)/@ points, 1]`
- 此时每个点分 **3 支**（三岔树）。封装函数 `Julia`。

### 4.2 有理函数的有界性修正（重要）
- 多项式的 Julia 集有界，但**有理函数的 Julia 集未必有界**。
- 因此 `reducedImage` 需额外丢弃 $|z|>\text{bound}$ 的点（选项 `Bound`，默认 4）：
```mathematica
bound = 4;
reducedImage[points_] := Module[{newPoints},
  newPoints = Complement[image[points], pointsSoFar];
  newPoints = Select[newPoints, N[Abs[#]] <= bound &];
  pointsSoFar = Union[newPoints, pointsSoFar];
  newPoints]
```
- 示例：用 Newton 法求 $z^3=1$ 根 → $f(z)=z-\dfrac{z^3-1}{3z^2}$ 的 Julia 集。

> **分支因子 = 映射次数**：二次→2 岔，三次→3 岔，有理 $z^n+\lambda/z^d$→$n+d$ 岔。逆M树状代码的"树"的**度（arity）**由此决定。

## 5. 随机逆迭代（Stochastic，类混沌游戏）

### 5.1 基本随机算法
- 每步随机选 `inv1` 或 `inv2` 施加到前一点；丢弃前 10 个 transient 点；绘图。
- 封装：`JuliaStochastic[c, numPoints]`。
- 问题：与 `JuliaSimple` 同样的**分布不均**。

### 5.2 偏置修正：`OneBias` 选项
- `OneBias ∈ (0,1)` = 选 `inv1` 的概率（选 `inv2` 概率 = $1-\text{OneBias}$）。
- **关键判据**：关于其不动点**更压缩（contractive）**的那个逆，应赋**更低概率**。
- 压缩度 = 该逆在不动点处的**导数绝对值** $|f'|$。

以 $c=-1$ 实测：
```
inv1 不动点 = 1.61803,  |D[inv1]| = 0.309017   ← 更压缩
inv2 不动点 = -0.618034, |D[inv2]| = 0.809017
→ 应取 OneBias < 0.5（文例取 0.28 明显改善）
```
- 局限：很多 Julia 集的细节超过随机算法任何偏置所能生成（如 $c=0.68i$ 最优偏置≈0.23 仍不如改进确定性算法）。文中**放弃**随机路线。

## 6. 与 Mandelbrot 集的关系（选 c 的指南）
- Mandelbrot 集 = 所有使 $J_c$ **连通**的 $c$ 值集合。
- 文内用 `Compile` + `FixedPointList[#^2+c&, c, 100, SameTest->(Abs[#2]>2&)]` 生成 Mandelbrot 图（改自 Dickau 1997）。
- **实用经验**：靠近 Mandelbrot 边界的 $c$ 常产生最有趣的 Julia 集。

## 7. 可移植到「逆M树状代码」的要点清单

| 本文概念 | 逆M树状代码对应 | 备注 |
|----------|----------------|------|
| 逆映射 $f_1^{-1},f_2^{-1}=\pm\sqrt{z-c}$ | 树的 2 个子节点 | 分支因子 = 多项式次数 |
| `Nest[invImage, {1}, depth]` | 深度优先/广度优先生成 depth 层树 | $2^{\text{depth}}$ 叶 |
| `pointsSoFar` + `Complement` 去重 | visited 集合剪枝 | 防指数爆炸的关键 |
| `Resolution` 量化网格 | 空间离散化精度 | 控制粗细与点数 |
| `Bound` 有界截断 | 树节点距离/模长上限 | 有理函数必备 |
| `OneBias` 导数压缩度偏置 | 树分支概率加权 | 可让"稀疏区"多采样 |
| 推广到 $z^3+z+0.6i$ | n 岔树（n>2） | 通用化框架 |

**结论**：本文给出了逆迭代生成 Julia 集的**完整、可运行、可推广**方案。其"逆像分叉 + 去重剪枝 + 分辨率量化"三件套，正是逆M树状代码应当直接采用的算法骨架；分支因子由映射次数决定（二次=二叉树，三次=三叉树），有界截断保证有理映射不发散。
