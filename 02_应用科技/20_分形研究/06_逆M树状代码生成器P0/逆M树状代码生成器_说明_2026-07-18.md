# 逆M树状代码生成器 P0 —— 运行说明与算法映射

> 落地路径 P0（最短路径）：把 fams 论文的 **Eq12（±分叉）+ Eq13（2^N 倍增）+ McClure 去重** 写成可运行代码，
> 直接对齐「逆M树状代码」项目。零硬件依赖（纯算法），GPU（Seetacloud RTX 4080）用于把规模跑大。

---

## 1. 三件套 → 代码逐行对应

| 论文要素 | 数学表达 | 本生成器实现 | 产物 |
|----------|----------|--------------|------|
| **Eq12** 逆 Julia 二值分叉 | `z_{n+1} = ±√(z_n − c)` | `p = √\|w\|·e^{i·arg(w)/2}`；`z = branch ? +p : −p` | 每个节点 → 2 子节点（二叉树） |
| **Eq13** 规模倍增 | `M₂ = M₁ × 2ᴺ` | BFS 逐层展开，每层节点数 ×2 | 深度 N 时共 `2ᴺ` 个叶 |
| **去重** McClure `pointsSoFar` | 访问集剪枝（均匀填充） | IIM 网格版：像素命中超 `hit_limit` → walker 重置到逃逸半径，并重新进入 **burn-in** | Julia 边界被均匀覆盖，无 reset 瞬态伪影 |

根节点取 **临界点 `z=0`**（`z²+c` 的临界点 `f'(z)=2z=0 ⇒ z=0`），从临界点的逆迭代 Preimage 树正是 Julia 集的骨架。

### 关于 burn-in（重置瞬态修复）
原始 IIM 在 walker 被 reset 回逃逸半径（\|z\|=2）后，前若干次逆迭代仍落在远离 Julia 集的过渡区域，
会在渲染图外围形成黄色光滑环。本生成器为每个 walker 维护 `burn_state` 计数器：
- reset 后计数器 = `burn_in`（默认 20），此阶段仍继续逆迭代但**不绘制**；
- 计数器归零后才重新参与命中累加。
从而保证只有 Julia 集边界附近的真实点被渲染。

## 2. 两种输出模式

### A. 精确满二叉树（字面「逆M树状代码」）
- 函数 `build_exact_tree(c, depth)`：CPU、结构化。
- 深度验证：depth=4 ⇒ 节点数 `1+2+4+8+16 = 31` ✓（已验证）。
- 输出：
  - `tree_*.npz`：全节点紧凑存储（`re, im, parent, dlev, br, c, depth`）—— 大规模友好，可直接 `np.load` 喂给 Canvas/树可视化。
  - `tree_*.json`：深度 ≤12 的可读节点（`{id, parent, depth, re, im, branch}`）—— 供前端/人工检查树结构。
  - `tree_*.png`：小深度（≤12）树状结构画线图（parent→child），直观展示「分叉即代码」。

### B. GPU 逆迭代采样（IIM 点云 + 高对比渲染）
- 函数 `iim_gpu(c, n_points, n_steps, ...)`：CuPy 向量化，8M walker × 200 步 ≈ 数秒。
- 输出：
  - `iim_*.npz`：命中缓冲 `buf[H,W]`（点云密度图）+ 元数据。
  - `iim_*.png`：**高对比** Julia 边界渲染（亮前景 `#F2D973/#335BFF` + 深底 `#06060F` + `gamma=0.45` 提亮暗部）—— 吸取上次黑底看不清的教训。

## 5. 与「逆M树状代码」项目的桥接

| 逆迭代概念 | 逆M树状代码对应 |
|------------|------------------|
| `±√(z−c)` 二岔 | 树的 2 个子节点 |
| `2ᴺ` 增长 | 深度 N 的满二叉树树叶数 |
| `pointsSoFar` 去重 | 树的 visited 剪枝（避免重复子树） |
| 分支因子 = 多项式次数 | 二次映射 ⇒ 二叉树；三次 ⇒ 三叉树（可扩展） |

下一步可直接把 `tree_*.npz` 的 `parent/br` 阵列接入你已有的 Canvas 旭日图渲染管线，
实现「逆M树状代码」的实时可交互可视化。
