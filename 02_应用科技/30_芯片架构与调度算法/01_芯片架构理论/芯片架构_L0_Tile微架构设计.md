## 一、L0 Tile = 芯片的最小自平衡单元

每个 tile 是一个微型自治系统，包含算术、热管理、时钟分配、负载交换四个子系统：

```
┌──────────────────────────────────┐
│           L0 Tile (ALU)          │
│                                  │
│  ┌─────────┐   ┌─────────────┐  │
│  │ ALU核心  │   │ PWM发生器    │  │
│  │ P/N对称  │   │ φ=可配 D=可配 │  │
│  │ 传输门   │   └──────┬──────┘  │
│  └────┬────┘          │         │
│       │          ┌─────▼──────┐  │
│       │          │ Farey ROM  │  │
│       │          │ 9×8 bit    │  │
│       │          └────────────┘  │
│  ┌────▼─────────────────────┐   │
│  │    本地 TEC 驱动器        │   │
│  │    AC-Peltier 换向        │   │
│  └──────────────────────────┘   │
│  ┌──────────────────────────┐   │
│  │  借核接口 (load交换)       │   │
│  │  ↔ 东/西/南/北邻tile      │   │
│  └──────────────────────────┘   │
└──────────────────────────────────┘
```

## 二、ALU核心：P/N对称传输门

传统CMOS逻辑只用N管下拉、P管上拉，但电流路径不对称→局部热点。

**P/N对称传输门**：每个逻辑单元同时有N管和P管路径，奇数拍N管主导、偶数拍P管主导。

```
传统反相器:              P/N对称:
   VDD                      VDD
    │                        │
  P管(常通)              P管(PWM选通)
    │                        │
    ├── OUT                  ├── OUT
    │                        │
  N管(常通)              N管(PWM选通)
    │                        │
   GND                      GND
```

**效果**：单边电流密度减半→焦耳热∝J²→单边热降至1/4。代价：面积+40%，速度−10%（传输门串联电阻）。

## 三、PWM发生器 + Farey相位ROM

### 3.1 Farey F₃相位分配

```python
# F₃ Farey序列: 0/1, 1/3, 1/2, 2/3, 1/1
# 为9个tile分配相位（加微小偏移避免重复）

F3 = [(0,1), (1,3), (1,2), (2,3), (1,1)]
T_cycle = 1000  # ns

phases = []
for i in range(9):
    num, den = F3[i % 5]
    base_phase = (num / den) * T_cycle
    offset = (i // 5) * (T_cycle / 150)  # ≈6.67 ns偏移
    phases.append((base_phase + offset) % T_cycle)
```

### 3.2 占空比D矩阵（行列和守恒）

3阶幻方：
```
M = [ [8, 1, 6],
      [3, 5, 7],
      [4, 9, 2] ]
幻和 = 15
```

线性公式 D[r][c] = a·M[r][c] + b：
- D_min=0.03, D_max=0.35
- 解得 a = −0.04, b = 0.38（归一化后）

```
D = [ [0.06,  0.34,  0.14],    ← 行和≈0.54
      [0.26,  0.18,  0.10],    ← 行和≈0.54
      [0.22,  0.02,  0.30] ]   ← 行和≈0.54
```

**约束**：每行每列D和相等。中心格(值5)D=0.18(中等)——既不是最忙也不是最闲。值9(隅角)D=0.02——几乎不干活，作为冷备。

### 3.3 ROM寻址

```
ROM地址 = tile(r,c)位置 → Farey分数索引 → 相位值
9个tile × 8 bit = 72 bit ROM
```

**面积估计**(28 nm)：9×8 bit ≈ 50 μm²，可忽略。

## 四、本地TEC驱动器 (AC-Peltier)

### 4.1 AC-Peltier 换向时序

```
PWM状态:   N主导  │  P主导  │  N主导  │  P主导  │
           ───────┼─────────┼─────────┼─────────
TEC冷端:   贴N端   │  贴P端   │  贴N端   │  贴P端   │
逻辑区:    永远贴冷端（N和P交替吸热）
热端:      甩给分形微通道
```

**频率选择**：100 Hz ~ 10 kHz。Peltier增益 > Thomson损失在此窗口最优。太低→Peltier来不及建立温差，太高→Thomson效应损耗>Peltier收益。

### 4.2 驱动电路

MOSFET H桥驱动TEC，PWM信号经过死区插入→H桥上下管交替导通。死区Δt与Farey相位同步偏移，避免所有tile同时处于亚稳态。

## 五、借核接口："洛书借核"协议

### 5.1 幻方行列约束的借核

每tile维护三个负载值：
- `load_local`：本tile的ALU利用率
- `load_row_sum`：本行所有tile的负载趋势
- `load_col_sum`：本列所有tile的负载趋势

**借核触发**：当 `load_local > 幻和/n × 1.3` 时

**借入方向**：按幻方对角线方向甩给空闲的相邻格（对角线守恒）
- 方向优先级：↘ > ↗ > ↓ > →

### 5.2 为什么幻方借核比随机窃密好

| | 随机窃密(work-stealing) | 幻方借核 |
|:---|:---|:---|
| 方向 | 随机邻居 | 对角线优先(幻和守恒) |
| 缓存抖动 | 高(任务频繁跨行迁移) | 低(优先同行/同列) |
| 负载平衡 | 局部贪心，可能过冲 | 行列全局约束，收敛快 |

## 六、面积功耗估算 (28 nm)

| 模块 | 面积(μm²) | 功耗(μW/MHz) |
|:---|:---:|:---:|
| ALU核心(P/N对称) | 1200 | 12.0 |
| PWM发生器 | 500 | 3.5 |
| Farey ROM(9×8) | 50 | 0.2 |
| TEC驱动器(H桥) | 800 | 5.0 |
| 借核接口 | 150 | 0.8 |
| **单tile合计** | **~2700** | **~21.5** |
| **9 tile总计** | **~24,300** | **~193 μW/MHz** |

**100 MHz下**：9 tile总功耗 ≈ 19.3 mW（占空比加权后≈3.1 mW，比传统8-bit ALU阵列降40-60%）。
