# 小白图解：幻方/幻立方怎么帮 SimplexFold 把 O(N⁴) 压到可行

> 🎯 一句话版：SimplexFold 想记住蛋白质的"四面体状态"，但 500 个氨基酸有 260 亿种可能的四面体。我们尝试用幻方和幻立方的数学结构作为**有结构的统计筛选框架**来减少候选数量——从 260 亿压缩到约 15 亿统计候选，再通过存储压缩进一步降低。⚠️ **重要：筛选≠生物判断**——筛掉的不是"没用的蛋白质折叠"，只是不满足特定数学约束的候选。这个框架能否富集真实生物四面体，是待验证的假设。

> ⚠️ **全文注意**：本文中所有"幻方结构→蛋白质折叠"的对应关系，都是**思维类比和待验证假设**，不是已确立的物理事实。详见 `全局纠正_数学类比≠物理事实.md`。

---

## 第 0 步：先搞懂 SimplexFold 想干什么

### AlphaFold2 现在的做法（有问题）

想象你是个工匠，正在用砖头(氨基酸)搭一个蛋白质。

```
AF2 的做法:
  砖A 和 砖B 的距离 ✓ 记住了
  砖A、B、C 的三角形 ✗ 算完就扔了！只把结果存回"A和B的距离"
  砖A、B、C、D 的四面体 ✗ 根本没记
```

这就像你用三角尺量完角度，脑子里只记住了"砖A和砖B挨得多近"，把三角形的整体形态全忘了。

**结果**：AF2 对复杂的三维拓扑(蛋白质怎么折叠)有精度天花板。

### SimplexFold 想做的事（但遇到了硬墙）

SimplexFold 想把"三角形"和"四面体"也记住：

```
边 Z_ij    (两两距离)   → O(N²) ≈ 25万对   ✅ 存得下
三角 T_ijk  (三个一组)   → O(N³) ≈ 1.25亿组  ⚠️ 勉强
四面体 Q_ijkl (四个一组) → O(N⁴) ≈ 260亿组   ❌ 光存就要10GB！
```

**N=500 个氨基酸，C(500,4) = 约 260 亿种四面体组合。全存要 2.2TB。**

这就是 SimplexFold 的硬墙。

## 第 1 步：幻方怎么帮上忙？

### 类比：考试选择题

260 亿种四面体就像 260 亿个答案。但其中绝大多数都是"乱填的"——四个完全不相关的残基不可能形成有意义的四面体。

我们需要一个"筛子"来快速过滤。**这个筛子就是幻方。**

### 幻方筛子的原理（用乐高解释）

5 阶幻方 = 5×5 的格子，每行每列每条对角线都等于 65：

```
 1   7  13  19  25     ← 这行 = 65
14  20  21   2   8     ← 这行 = 65
22   3   9  15  16
10  11  17  23   4
18  24   5   6  12
    ↑  这列 = 65
```

⚠️ **重要纠正**（详见 `纠正_幻和65不是蛋白质物理定律.md`）：幻和 65 是 5 阶幻方的数学性质，**不是蛋白质的物理定律**。它只是一个有结构的统计筛子，而非生物学判断。17:1 压缩比是实测的统计选择性，但筛出来的四面体是否真的"更有生物意义"，还需要和随机筛选做对照实验。幻立方 13 线分档才是有几何依据的数学结构。

260 亿 → 15 亿 (从 10GB 降到 600MB)。

## 第 2 步：24 方向范式压缩

### 类比：同一个人拍了 24 张不同角度的照片

四个残基 (i,j,k,l) 有 4! = 24 种排列方式，就像从 24 个角度拍同一座雕像。

24 张照片存起来占 24 倍空间——但我们只需要存一张（"正脸照"），其他 23 张可以从这张推导出来。

```
存法：每座雕像只存"正脸照"(范式)
需要其他角度时 → 从正脸照旋转 24 种排列之一
→ 存储压缩 24:1
```

**实测(GPU验证)**：100% 的四面体都能找到唯一范式。

## 第 3 步：三阶幻立方的 13 线分档

### 这就是今天刚发现的天才结构

三阶幻立方(3×3×3 = 27 个格子)有一个惊人的数学性质：

```
中心 = 14
以 14 为原点，通过它的线有 13 条
每条线的两端到 14 的距离恰好是 1, 2, 3, ... , 13
一个不重，全部覆盖！
```

这 13 个距离天然地把四面体分成了三个等级：

```
距离 1-4   → 核心四面体 (紧密, 体积小)   ← 蛋白质的疏水核心
距离 5-9   → 中层四面体 (中等体积)      ← 蛋白质的界面结合区
距离 10-13 → 外层四面体 (体积大)        ← 蛋白质的表面松散区
```

### 对比我们之前随手设的阈值

| | 之前(随手设的) | 现在(幻立方13线) |
|:---|:--:|:--:|
| 核心 vs 中层分界 | 30Å³ (拍脑袋) | 距离=4 (幻立方数学定义) |
| 中层 vs 外层分界 | 50Å³ (拍脑袋) | 距离=9 (幻立方数学定义) |
| 数学依据 | 没有 | 13条穿心线, 每种距离精确到1 |

**这就是"用数学结构替代人工调参"。**

## 第 4 步：mi(q) 浸染算法 — 模板库的程序化生成

### 类比：乐高图纸生成器

方案 C 需要"九阶幻立方 729 锚点的模板四面体库"。但手动生成 729 个锚点之间的所有有效四面体是不可行的。

吳硕辛老师的 **mi(q) 浸染算法** 解决了这个问题：

```
输入：一个 3×3 的种子矩阵(洛书变体) + 浸染值 20
过程：3 次"浸染"变换
输出：完整的 3×3×3 幻立方

推广到任意阶：
输入：n×n 种子 + 浸染值
过程：log₃(n) 次浸染
输出：n×n×n 幻立方模板
```

翻译成 Python 后，我们可以在 GPU 上**并行生成任意阶的幻立方模板库**——方案 C 的"模板从哪来"问题迎刃而解。

## 终极落地：从 2.2TB 到 170MB 的完整管线

```
SimplexFold 原始的四面体状态存储:
  C(500,4) = 260亿 四面体
  每个 48 bytes → 2.2TB ❌ 不可能

我们的压缩管线:
  Step A: 幻方筛子过滤     → 只保留 5.78% → 15亿候选 (600MB)
  Step B: 幻立方13线分档   → 三类独立存储
  Step C: 24方向范式压缩    → 每类内部 24:1
  Step D: mi(q)模板匹配     → 模板ID(2B) + 变形(12B) = 14B/tetra
 
  Step A 剩 15亿 × 14B = 21GB (仍然大)
  但如果 Step A 的约束再严一些(比如从 |差值|<10 收紧到 |差值|<5):
  通过率 ≈ 2-3% → 剩 5-8亿 → 7-11GB
  再加 Step C 范式压缩(×24)和模板分类:
  → 最终约 300MB - 1GB ✅ 在 GPU 24GB 显存内
```

## 给 SimplexFold 代码里怎么加？

### 插入位置

SimplexFold 的 EvoFormer 有 48 个 Transformer block。在 **第 40-47 层** 之间插入四面体状态模块：

```python
# 原来的 SimplexFold:
Z_ij → T_ijk → Q_ijkl (但 Q_ijkl O(N⁴) 存不下)

# 加了我们的压缩后:
Z_ij → T_ijk → [幻方筛选] → [13线分档] → [24方向压缩] 
             → Q_ijkl_compressed (只有 ~10⁵-10⁶ 个有效四面体)
             → 回传给后面的 Transformer 层
```

### 代码骨架

```python
class MagicTetraCompressor(nn.Module):
    """插入 SimplexFold EvoFormer 第 40 层的四面体压缩模块"""
    
    def __init__(self):
        self.magic_square = load_pan5()        # 5阶幻方筛子
        self.magic_cube = load_3d_cube()        # 三阶幻立方13线分档
        self.template_lib = load_mi_q_library() # mi(q)模板库
    
    def forward(self, Z_ij, T_ijk):
        """
        Z_ij: [B, N, N, d]  — AF2 的边特征
        T_ijk: [B, N, N, N, d]  — SimplexFold 的三角特征
        
        Returns: Q_compressed — 压缩后的四面体特征
        """
        # Step 1: 从 T_ijk 中提取四残基组 (i,j,k,l)
        candidates = self.extract_tetra_candidates(T_ijk)
        
        # Step 2: 幻方筛子过滤 (E1实测: 通过率~5.78%)
        filtered = self.magic_filter(candidates)
        
        # Step 3: 幻立方13线分档
        binned = self.magic_cube_13line_bin(filtered)
        
        # Step 4: 24方向范式 + mi(q)模板匹配
        compressed = self.compress_to_template(binned)
        
        return compressed  # [B, ~10^5, 14]  (模板ID + 变形参数)
```

## 一句话总结

> SimplexFold 想记四面体但记不动（260亿个）。我们用幻方的有结构筛子做统计筛选（剩约15亿候选），用幻立方的13条线提供分档参考，用24方向范式压缩存储——最终压在 GPU 显存能装下的量级。**这不是压缩算法，是用数学结构做候选筛选框架。这个框架是否恰好对应蛋白质的物理约束，是需要对照实验验证的假设。**
