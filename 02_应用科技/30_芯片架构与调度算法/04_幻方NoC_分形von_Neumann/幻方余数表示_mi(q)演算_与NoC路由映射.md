# 幻方余数表示 (mi(q)演算) 与 NoC 路由映射

> 来源：Encyclopedia of Mathematics "Magic Square" + 元宝对话芯片架构
> 核心：幻方的代数结构 = (Z/n)² → (Z/n)² 的线性映射，恰好就是 NoC 的路由表压缩算法
> 日期：2026-07-19

---

## 一、mi(q) 是什么？

幻方中每个数字 a (1 ≤ a ≤ n²) 可以唯一地用一对 n 进制余数表示：

$$a-1 = \alpha + \beta \cdot n, \quad \alpha, \beta \in \mathbb{Z}/n$$

其中：
- α = (a-1) mod n（第一余数）
- β = ⌊(a-1)/n⌋（第二余数）

这里 q = n（阶数），mi(q) = 第 i 个方向上的余数分量。对于二维幻方，有两个分量 α 和 β。

**关键**：幻方的单元格坐标 (i, j) 也属于同一个空间 (Z/n)²。所以幻方本质上是：

$$f: (\mathbb{Z}/n)^2 \to (\mathbb{Z}/n)^2, \quad (i, j) \mapsto (\alpha(i,j), \beta(i,j))$$

就是把网格坐标映射到幻方数字的余数坐标。

## 二、线性幻方的 Siamese 参数化

如果 α 和 β 都是线性的，即：

$$\alpha(i,j) = i_0 + p \cdot i + q \cdot j \pmod{n}$$
$$\beta(i,j) = j_0 + \bar{p} \cdot i + \bar{q} \cdot j \pmod{n}$$

其中六个参数 (i₀, j₀, p, q, p̄, q̄) 满足特定条件后，产生的就是奇数阶标准幻方。

**Siamese 方法的参数**（最常见）：
```
i₀ = 0      （从第0行开始）
j₀ = n/2    （中间列）
p = -1      （向上）
q = +1      （向右）
p̄ = +1     （被占时向下）
q̄ = 0      （被占时不动）
```

**3阶幻方 Siamese 参数下的 α, β**：

| (i,j) | α = i₀+pi+qj mod 3 | β = j₀+p̄i+q̄j mod 3 | a = α+3β+1 | 幻方值 |
|:---|:---:|:---:|:---:|:---:|
| (0,1) | -1+1=0 | 1+0=1 | 0+3+1=4 | 1 不是？|

实际上 Siamese 的"数字 a 的位置"由反向映射决定：

$$a \mapsto (i, j) = (i_0 + p \cdot (\alpha + \beta n), j_0 + q \cdot (\alpha + \beta n)) \pmod{n}$$

这才是 Siamese 方法的本质：**从数字 a 的余数表示 (α,β) 反算出它在网格中的位置 (i,j)。**

## 三、与 NoC 路由表压缩的直接映射

### 3.1 传统 NoC 路由：全表存储

每个 tile 需要存储整个网络的路由表：大小为 O(N²)（N=tile总数）。

### 3.2 幻方路由：6参数 + 模运算

每个 tile 知道自己在本级幻方中的 (α,β) 坐标后，目标 tile 的位置 (i,j) 由线性公式即时算出：

```
输入: 目标 tile 的幻方值 a
输出: 目标 tile 的网格位置 (target_i, target_j)

Step 1: α = (a-1) mod n
        β = (a-1) // n

Step 2: target_i = f_i(α, β) = (i₀ + p·α + q·β) mod n
        target_j = f_j(α, β) = (j₀ + p̄·α + q̄·β) mod n
```

**6个参数替代了 n² 个表项。** 路由表从 O(N²) 降为 O(1)。

### 3.3 多级分形路由

```
L0 (3阶): 9个ALU tile, 参数{i₀=0, j₀=1, p=-1, q=+1, p̄=+1, q̄=0}
           → tile间路由：α,β∈Z/3, 3bit地址

L1 (5阶): 25个L0 core, 参数{i₀=0, j₀=2, p=-1, q=+2, p̄=+2, q̄=-1}
           → core间路由：α,β∈Z/5, 5bit地址

L2 (7阶): 49个L1 cluster, 参数{i₀=0, j₀=3, p=-1, q=+3, p̄=+3, q̄=-1}
           → cluster间路由：α,β∈Z/7, 7bit地址
```

**注意**：每级的 Siamese 参数(p, q, p̄, q̄)不同——不是简单的放大，而是独立选择的参数可以优化该级的路由局部性。

### 3.4 为什么比 Hilbert/Peano 好？

| | Hilbert/Peano | 幻方 Siamese |
|:---|:---|:---|
| 编码方式 | 空间填充曲线的几何坐标→1D地址 | 余数对(α,β)→模运算→(i,j) |
| 负载均衡 | 无（仅保证走线局部性） | **行列对角线三向均负载**（幻和） |
| 多级扩展 | 每级重新算曲线 | 每级改6个参数即可 |
| 路由表大小 | O(1) 查表 | O(1) 6参数计算 |
| 对角线路由 | 无 | 有（幻方对角线也守恒） |

## 四、幻方路由的"幻和偏差"寻路

**A* 启发函数**（Manhattan + 幻和偏差双权重）：

$$h(tile) = w_1 \cdot (|i_{cur} - i_{target}| + |j_{cur} - j_{target}|) + w_2 \cdot |S_{row} - S_{col}|$$

其中 S_row 是当前行所有 tile 的负载估值，S_col 同。

第二项保证了**路由途经不会压爆某一行/列**——如果某行负载已经接近幻和，路由自动避开该行。

## 五、mi(q) 的物理含义在 NoC 中

```
"mi" = modular index (模索引)
"q"  = the modulus (模数，即幻方阶数 n)
"mi(q)" = 数字 a 在模 q 下的第 i 个余数分量

在 NoC 路由中：
  m₀(q) = α = (a-1) mod n  → 行偏移余数
  m₁(q) = β = (a-1) // n   → 列偏移余数
```

这两个余数就是 tile 地址的天然编码——不需要单独分配地址，幻方值本身就是地址。

## 六、Medjig 方法：偶数阶幻方的 mi(q) 扩充

当阶数 n 为偶数时，Siamese 不工作。但可以用 **Medjig 方法**：

1. 取一个 k×k 的奇数幻方（用 Siamese）
2. 每个格子再分为 4×4=16 个子格
3. 用 Medjig 片（0,1,2,3 的排列）填充
4. 幻方值 a → a + Medjig(i)·k²

这等价于在 Z/k 基础上再加一个 Z/4 的余数层：

$$\alpha_{ext} = \alpha_k + \text{Medjig}_\alpha \cdot k$$
$$\beta_{ext} = \beta_k + \text{Medjig}_\beta \cdot k$$

多了一个"子格余数"维度——恰好对应 NoC 中 tile 内部的子单元（ALU/寄存器 bank）。

## 七、总结

| 幻方概念 | NoC 路由对应 |
|:---|:---|
| mi(q) = (α, β) 余数对 | tile 地址的模编码 |
| Siamese 六个参数 | 路由表的压缩密钥（6个整数替代n²个表项） |
| 幻和行列守恒 | 路由的负载均衡硬约束 |
| 对角线守恒 | torus 拓扑的对角线路由 |
| Medjig 扩充 | 多级子格的地址扩展 |
| 幻和偏差 A* | 路由避热启发函数 |
