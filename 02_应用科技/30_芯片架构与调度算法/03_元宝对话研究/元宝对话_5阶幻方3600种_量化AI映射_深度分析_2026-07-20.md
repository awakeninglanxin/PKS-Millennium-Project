## 二、核心技术精华

### 2.1 5 阶完美幻方（Pandiagonal/Panmagic）构造公式

#### 母构造法：骑士步（Margossian cavalière）

```
(r, c) ← (r+1 mod 5, c+2 mod 5)
从 1 在 (0,0) 开始填，得到 base 型
```

**5×5 输出示例**（每行/列/主对角/折断对角和=65，中心=13）：
```
 1  7 13 19 25
14 20 21  2  8
22  3  9 15 16
10 11 17 23  4
18 24  5  6 12
```

#### 双拉丁方叠加法（更代数化）

```python
A[i][j] = i                  # 行号
B[i][j] = (a*i + b*j) mod 5  # a,b 与 5 互素，a≠b
M[i][j] = 5*A[i][j] + B[i][j] + 1
```

骑士步 `(dr=1, dc=2)` 对应 `B = i + 2j mod 5`。

#### 3600 = 36 × 4 × 25 变换链

| 层 | 变换 | 数学含义 |
|----|------|----------|
| **36 essentially different** | 所有合法 `(dr,dc)` 组合 (与5互素, dr≠dc) | 基本型枚举 |
| **×4** | 1-3-5-2-4 重排行+列 / 对角换行列 | 群变换族 |
| **×25** | 5 行 × 5 列循环位移 | 模 5 周期性 |

### 2.2 文档中 64×13+16+128×21+32×2=3600 的真正含义

这不是构造公式——是 budshaw 站 `Groups5Center13.html` 中按**中心13的子模式分类**做的计数加法拆解。3600 个 pan5 中心全是 13。真正"造"的公式只有骑士步/拉丁方那套。

## 三、幻方→量化金融的映射（火线区）

### 3.1 关键纠错："幻方量化"不真用 Magic Square

- **幻方量化 (High-Flyer)**：2015 梁文锋创立，A 股 Alpha + CTA，2021 破千亿
- **DeepSeek**：2023.7 从幻方拆分，专做 AGI
- 实盘底层：LSTM(Level-2 订单流) + Transformer(中宏观时序) + PPO(策略工厂) + MoE(分层稀疏激活)
- "幻方"是浙大电子工程+奥赛背景的数学审美传承，不是算法本体

### 3.2 隐式同构映射表

| 5 阶 pan5 构造思想 | 量化里的对应 | 同构本质 |
|-------------------|-------------|----------|
| 双拉丁方 `5A+B+1` | 多因子双线性交互 `w1*f1+w2*f2+f1⊗f2` | 两个独立模结构叠成预测表 |
| 骑士步 `(dr,dc)` 与 5 互素 | 因子正交化约束 (PCA/ICA) | 线性无关才能叠出 panmagic/alpha |
| 1-3-5-2-4 重排仍保 pan | 时间轴 cyclic augmentation | 错位喂给模型仍是合法样本 |
| 3600 = 36×4×25 变换群 | 策略工厂：基模型 × 排列/对称变换 = 策略族 | PPO 一晚 10 万组合同理 |
| mod 5 绕回（周期性） | 星期效应/月度效应 Fourier 特征 | 周期因子用 mod 结构建模 |

> **最深同构**：幻方用"规则离散结构"让 1~25 每个数恰好出现一次且行列对角和相等；量化用"正则化+稀疏+风险预算"让因子库每个信号恰好贡献一点 alpha 且不撞车。本质都是**"在约束下排布元素"的组合优化思维**。

## 四、离散组合数学在量化中的真·应用（四条硬核线）

### 线一：组合设计 → 因子正交 & 实验设计

| 数学对象 | 量化应用 |
|----------|----------|
| **Latin square**（n×n，每行每列每个符号恰好一次） | 策略回测参数正交采样（A/B test 多因素排布） |
| **Hadamard 矩阵**（H·Hᵀ = nI） | 因子正交化（低秩近似）、LLM 量化中的旋转权重 |
| **正交阵列 OA(N,k,s,t)** | 多策略组合的压力测试参数覆盖 |

### 线二：图论 → 套利 & 选股 & 风控

| 图论对象 | 量化映射 |
|----------|----------|
| **三角套利**（负权环 Bellman-Ford） | 跨市场/跨品种价差异常检测 |
| **关系图谱聚类** | 产业链/供应链关联选股 |
| **最大独立集 (MIS)** | 持仓组合：选相关系数 < 阈值的资产集合 → 组合优化 |
| **最大流/最小割** | 流动性压力测试、极端行情传导路径 |

### 线三：组合优化 QUBO/Ising → 离散版 Markowitz

$$ \min_x \left( \lambda x^T \Sigma x - \mu^T x \right) \quad \text{s.t. } x_i \in \{0,1\}, \sum x_i = K $$

这是 NP-hard（背包+二次规划的组合爆炸），与幻方"在约束下填数"同一 DNA。工具链：D-Wave 量子退火 / Fujitsu 数字退火 / Gurobi 整数规划。

### 线四：幻方本身 → 密码学 S-box

- 素数幻方作为"带结构的素数阵"可设计新型 S-box
- 16 个 ultramagic（同时满足 associative + pandiagonal）可做 AES 候选常量

### 5.2 PyTorch 原型一：同余 → MoE 确定性哈希路由

**核心代码**（已在对话中实现并运行通过）：
```python
class HashRouter(nn.Module):
    def forward(self, token_ids):
        hashed = (token_ids * self.seed) % self.expert_num
        return hashed  # 硬均匀分配，无需 auxiliary loss
```

**运行结果**：Expert 0-3 各 4 tokens（16/4=4），严格均匀。

**价值**：当前 Switch Transformer 用 top-k router + auxiliary loss 需要调 α 超参，同余 hash 给出**硬约束替代**，与幻方中 `(r+dr, c+dc) mod 5` 保证每行每列每个剩余类恰好出现一次是同构的。

### 5.3 PyTorch 原型二：反幻 → Contrastive Diversity

**核心思想**：反幻方要求所有行和互不相同 → 最大化 batch 内样本相似度方差

```python
def anti_magic_diversity_loss(z):
    sim = torch.mm(z, z.t())  # batch × batch 相似度矩阵
    var = torch.var(sim_no_diag)
    return -var  # 最大化方差 → 鼓励差异化
```

## 六、与 PKS 项目 / 芯片架构的桥接

### 6.1 同余 → Farey 芯片调度

幻方 `mod 5` 保证"每行每列每个数恰好一次"的同余思想，与 Farey 树的 **`bc-ad=1` 邻居性质保证相邻任务不可冲突**是同源数学结构——都是**用互素/同余约束保证排布无冲突**。

### 6.2 36×4×25 变换群 → 幻方 NoC 路由

`1-3-5-2-4` 重排列 + 循环位移形成的**变换群**，与 `04_幻方NoC_分形von_Neumann/` 中的 Torus 拓扑 Chiplet 2.5D 路由直接对应——芯片布线本质上就是"在约束下排布信号路径"的组合优化。

### 6.3 反幻 → MoE 专家分化与芯片调度

反幻要求"各行和互不相同"（最大化差异）→ 对应芯片调度中"不同模块并行度最大化避免资源争抢"——这正是 `Farey-Hybrid_Chip_Scheduler` 在灾难恢复场景比 CPM 快 55.8% 的数学根源。

### 6.4 素数幻方 → PKS SPF 筛选

`(无/全)素数幻方` 的"在组合约束上叠加数论约束"——与 PKS 锚点 2 的 B-smooth 碰撞（在加法表示上叠加光滑性约束）共享同一数论×组合的交叉 DNA。
