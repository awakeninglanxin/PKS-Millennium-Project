## 二、按文件类型分类（无明确主题时）

| 文件类型 | 默认放哪 | 是否上传 GitHub |
|:---------|:---------|:---------------:|
| `.md` 分析文档 | 按主题 → 对应子目录 | ✅ 是 |
| `.py` 代码 | `03_验证工具/{对应子工具}/` | ✅ 是 |
| `.mm` 脑图 | 与相关 MD 同目录 | ✅ 是 |
| `.png / .jpg` 截图/可视化 | 与生成代码同目录 | ✅ 是（<5MB） |
| `.dxf` 工程图 | `01_基础理论/16_schauberger…/DXF生成程序集/` | ✅ 是 |
| `.pdf` 参考论文 | `research/papers/`（学术） 或 主题目录（Schauberger等） | ❌ 否 |
| `.docx / .ppt` 文档 | 主题目录 | ❌ 否 |
| `.mp4 / .zip / .exe` | 主题目录 | ❌ 否 |
| `.nb` Wolfram 笔记本 | `02_应用科技/24_searl_SEG/瑟尔讲解所需nb/` | ❌ 否 |
| `.csv` 数据 | 与消费它的 py 同目录 | ✅ 是 |

## 四、编号规则

对同一子目录下内容相关的多个文件，用数字前缀标明顺序：

```
01_总览.md
02_深度分析.md
03_代码对应.md
04_验证结论.md
```

**规则**：
- 只在"同一个子主题下多个文件有明确的阅读顺序"时才编号
- 不强制所有文件编号
- 编号避免与已有文件冲突（先看同目录下最大编号）
